- 其中 `忽视书` 必须作用于宠物 **普攻 / 连击追加 / 主动技能** 三类伤害

**代码落点**

- [PetSkillData.lua](</C:/Users/mahaizhe/Documents/New project/taptapmaker/rengouxiantu-docs/scripts/config/PetSkillData.lua:47>)
- [Pet.lua](</C:/Users/mahaizhe/Documents/New project/taptapmaker/rengouxiantu-docs/scripts/entities/Pet.lua:141>)

**执行要求**

1. 在 `PetSkillData.SKILLS` 中新增 3 个技能定义：
   - `ignoreDef_basic`
   - `atkPerLv_basic`
   - `bonusDmg_basic`
2. 数值按当前文档执行：
   - `忽视书`：`8 / 15 / 20 / 25`
   - `蕴灵·攻击书`：`每级 +0.5 / +1 / +1.5 / +2`
   - `加伤书`：`5 / 10 / 15 / 20`
3. 在 `PetSkillData.STAT_NAMES` 中补上这 3 个字段的显示名。
4. 在 `Pet:GetSkillBonuses()` 中补上：
   - `ignoreDefPct`
   - `bonusDmgPct`
   - `perLvBonuses.atk`
5. 在 `Pet:RecalcStats()` 中补上 `atkPerLv` 的成长计算，不能只支持 `maxHp / def`。
6. 将宠物伤害统一收口到一个公共结算点，再由：
   - `DoAttack()`
   - 主动技能 `CastSpiritBite()`
   - 连击追加伤害
   共同调用，避免三套逻辑分叉。
7. `忽视书` 的判定规则：
   - 每次伤害包各自独立判定
   - 触发时按“无视防御”计算该次伤害
   - 连击追加与主动技能必须吃到同一套忽视逻辑
8. `加伤书` 的判定规则：
   - 作为宠物最终伤害乘区处理
   - 普攻、连击、主动技能都要生效

- 当前 `Pet.lua` 里虽然缓存了 `doubleHitPct / critDmgPct / lifeStealPct`，但战斗侧并没有完整消费。做这一步时，至少要把“连击追加”补到真正的伤害流程里，否则 `忽视书` 无法满足“连击可触发”的要求。
