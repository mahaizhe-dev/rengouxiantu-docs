- `wisdom`：悟性。
- `constitution`：根骨。
- `physique`：体魄。

| 属性 | 每格 | 9格合计 | 全激活额外 | 最终满额 |
|------|-----:|---------:|-----------:|---------:|
| 悟性 | +10 | +90 | +10 | +100 |
| 根骨 | +10 | +90 | +10 | +100 |
| 体魄 | +10 | +90 | +10 | +100 |

运行时写入：

```lua
player.artifactCh5Wisdom
player.artifactCh5Constitution
player.artifactCh5Physique
```

写入后调用 `player:InvalidateStatsCache()`，最终进入：

- `Player:GetTotalWisdom()`
- `Player:GetTotalConstitution()`
- `Player:GetTotalPhysique()`

## 七、被动技能：四剑诛灭

### 7.1 当前数值

| 项目 | 当前代码 |
|------|----------|
| 解锁 | 9格全激活并击败太虚宗主 |
| 触发入口 | `SkillSystem.HandleSkillCastPassives()` |
| 触发条件 | 施放非被动主动技能 |
| 目标 | 6格内最近的存活怪物 |
| 无目标 | 不触发、不进入冷却 |
| 内置冷却 | 60秒 |
| 目标区域 | 半径2格 |
| 初始落剑延迟 | 0.25秒 |
| 段间隔 | 0.18秒 |
| 四段倍率 | 1.0 / 1.5 / 2.0 / 2.5 |
| 总倍率 | 攻击力×7.0 |
| 暴击 | 每段独立调用 `player:ApplyCrit()` |

### 7.2 结算流程

```text
玩家施放主动技能
  ↓
被动已解锁且60秒冷却完成
  ↓
搜索6格内最近怪物
  ↓ 无目标：结束，不消耗冷却
锁定目标当前坐标
  ↓
四把剑按0.18秒间隔依次落下
  ↓
每段独立暴击，对半径2格内全部存活怪物造成伤害
```

锁定的是触发时坐标。后续怪物离开该区域会显示 `MISS`，不会追踪新位置。

### 7.3 运行时状态

以下字段不存档：

```lua
ArtifactCh5._passiveCdTimer
ArtifactCh5._passiveReady
```

读档时只要被动已解锁，就会重置为可用状态。

## 八、账号级与存档

### 8.1 当前双层结构

第五章神器并非“只写账号档”，实际同时写入：

```text
save_{slot}.artifact_ch5
account_atlas.artifact_ch5
```

角色槽位结构：

```lua
artifact_ch5 = {
    activatedGrids = { boolean × 9 },
    bossDefeated = boolean,
    passiveUnlocked = boolean,
}
```

账号档结构相同，位于 `account_atlas.artifact_ch5`。

### 8.2 保存

激活成功后，系统发出：

```lua
EventBus.Emit("save_request")
```

正常保存时：

1. `ArtifactSystem_ch5.Serialize()` 写入角色槽位镜像。
2. `AtlasSystem.Serialize()` 将当前进度并入账号数据。
3. `SaveWriteService` 把账号数据写入独立 `account_atlas`。

### 8.3 读取

1. 先读取角色槽位 `artifact_ch5`。
2. 再读取独立 `account_atlas`。
3. `AtlasSystem.PostLoadSync()` 将账号格子、Boss状态和被动状态按并集同步回当前角色。
4. `ArtifactCh5.RecalcAttributes()` 重算三项属性。

### 8.4 当前风险

1. 激活是异步弱保存，保存失败没有回滚消耗。
2. Boss胜利分支没有发 `save_request`，存在首胜后未及时落盘窗口。
3. `account_atlas` 当前由客户端提交，服务端没有第五章神器专用结构规范化。
4. 账号并集只增不减，普通 `ArtifactCh5.Reset()` 不能清理账号档。
5. `AtlasSystem.SaveImmediate()` 的 `clientCloud:SetScore` 接口需要真实环境验证，不能替代主存档链作为唯一保障。

## 十、完整自测方案

### 10.1 碎片

- 9个指定Boss均只掉对应残符，概率0.1%。
- 黑市1~3号为30/60，4~9号为25/50。
- 背包锁定碎片不能被激活流程误扣。

### 10.2 激活

- 资源不足时不扣除任何内容。
- 正常激活准确扣除2000万金币、10000灵韵、1个对应残符。
- 重复激活同一格失败且不扣费。
- 9格满额为悟性、根骨、体魄各100。
- 属性缓存失效后面板即时更新。

### 10.3 账号共享

- 角色A激活一格并成功保存。
- 切换角色B后自动拥有该格与对应属性。
- 角色B旧槽位镜像不能让账号进度倒退。
- 账号档高进度、角色档低进度时取并集。
- 多次切换角色属性不重复叠加。

### 10.4 Boss

- 8格不能挑战，9格可以挑战。
- 进入竞技场后原世界数据完整备份。
- 战败后回到原场景、满血复原、不解锁被动。
- 首胜后解锁Boss状态与被动，显示觉醒弹窗。
- 首胜保存失败时应回滚，当前代码尚未满足这一条。

### 10.5 被动

- 未解锁时永不触发。
- 被动技能、普攻和持续伤害不能触发。
- 主动技能有目标时触发并进入60秒冷却。
- 无目标时不消耗冷却。
- 四段时间为0.25、0.43、0.61、0.79秒附近。
- 四段倍率为100%、150%、200%、250%。
- 每段可独立暴击，范围外目标显示MISS。
- 冷却结束后恢复可用。
- 读档后已解锁被动重置为可用。

### 10.6 存档故障

- 激活请求超时、失败、断线时不应出现“资源已扣但格子未落盘”。
- Boss首胜后立即退出也不应丢失解锁。
- 旧档缺 `artifact_ch5` 时应补默认结构。
- 非法格数、非布尔值、伪造Boss/被动关系应被服务端拒绝或修正。
