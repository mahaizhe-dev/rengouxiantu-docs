> 状态：重评整合版

## 1. 文档目标

这份规格书只解决四件事：

1. 第五章地图整体应该长什么样。
2. 第五章地图应该怎样画，才能明显优于前四章现有地图。
3. 第五章地图怎样在手机上默认可承受，而不是靠额外性能开关补救。
4. 其他 AI 拿到这份文档后，应该按什么顺序改哪些文件，做到什么程度算完成。

## 2. 重评后的核心结论

### 2.1 方案结论

第五章仍然应该从“单层 tile 直接决定最终画面”升级为分层方案，但收敛成下面五层：

1. `logicGrid`
   负责碰撞、寻路、封印、交互语义。
2. `materialGrid`
   负责地表主材质。
3. `edgeGrid`
   负责自动拼边结果。
4. `decalGrid`
   负责静态裂纹、剑痕、灰烬、残剑等细节。
5. `ambientZones`
   只作为可选加分项，不作为第五章试验成功的前置条件。

### 2.2 性能结论

这次重评后，**不把“低功耗模式”或“第五章专属性能开关”作为正式方案组成部分**。

原因：

1. 如果第五章必须靠专门开关才能跑稳，说明默认实现本身就不合格。
2. 地图表现层不应该给项目继续引入一套新的质量分叉维护成本。
3. 对执行 AI 来说，“先做高成本版，再靠开关降级”会放大实现发散和返工。

因此这份规格书只定义一条正式路径：

1. 默认路径就要可上线。
2. 性能控制靠架构约束，不靠玩家可见的章节专属开关。
3. 开发期可以有临时调试开关，但它们不写进正式验收目标。

### 2.3 第五章真正要验证的东西

第五章试验真正要验证的是：

1. 自动拼边能否把断崖、岸线、墙边、转角画准。
2. 静态材质分层和贴花能否把“太虚废墟感”做出来。
3. 这套表现能否在默认路径下控制住手机发热。

**第五章试验不以“特效多不多”作为成功标准。**

## 3. 第五章地图设计基线

### 3.1 章节基线

| 项目 | 内容 |
|------|------|
| 章节名称 | 第五章 · 太虚之殇 |
| 进入条件 | 合体中期（Lv.90）+ 完成第四章主线 |
| 地图规模 | `80x80` |
| 地图母结构 | `前营 -> 双路 -> 剑宫 -> 深渊 -> 回廊` |
| 核心终局 | 四把入魔神剑 |
| 主题 | 修真界第一宗门废墟、剑气残留、魔化侵蚀、终局深渊 |

### 3.2 主线推进基线

第五章主线推进固定如下：

1. 玩家进入 `太虚遗址前营`。
2. 玩家确认 `裂山门遗址` 正路彻底损毁。
3. 玩家被迫左右分流，推进左路与右路。
4. 左右两路都完成后，解开 `太虚剑宫` 外层封印。
5. 玩家进入 `太虚剑宫`，并进一步推进 `镇魔深渊`。
6. 玩家以 `妖魔之血` 激活四座封剑台。
7. 玩家完成四把神剑终局。
8. 玩家进入连接 `栖剑别院` 与 `藏经书阁` 深层终点的 `万剑回廊`，承接特殊 Boss 与后续强化线。

### 3.3 空间设计硬规则

第五章地图必须遵守以下空间硬规则：

1. `太虚剑宫` 固定在地图中后部中心，不通过移动核心区制造变化。
2. `裂山门遗址` 必须明确告诉玩家“正路已断”，而不是只做一个路障。
3. 左右两路都不是支线，都是必须完成的主线空间。
4. 左右两路不能做机械镜像，只能做“母结构对应、细节非对称”。
5. `镇魔深渊` 必须挂在 `太虚剑宫` 背后，不提前放在左右路线里。
6. `万剑回廊` 必须放在最深处，并明确连接左侧 `栖剑别院` 与右侧 `藏经书阁` 的深层终点。
7. `万剑回廊` 可以在中段或附入口与 `镇魔深渊 / 剑宫终局层` 解锁后连通，但它不能被画成只服务深渊中轴的一条后场路。

### 4.4 空间组织规则

执行 AI 画图时必须遵守下面这些组织规则：

1. 左右路线宽度可以不等，但两条线都必须完整、可读、可回溯。
2. 中轴在 `裂山门遗址` 段必须封死，不能被玩家直接穿过去。
3. `太虚剑宫` 必须比前营更规整、更强秩序感。
4. `镇魔深渊` 必须比剑宫更破碎、更下沉、更危险。
5. `万剑回廊` 必须明显比其他区域更窄，并且悬空感更强。
6. `万剑回廊` 不是单点终点，它必须同时回应左深层终点和右深层终点的空间延伸。
7. 左路与右路应在视觉上形成对照：左路偏“战斗 / 锻造 / 生活残骸”，右路偏“寒池 / 传承 / 典籍焚毁”。

## 5. 推荐实现架构

### 5.1 运行流程

第五章地图渲染按下面的顺序组织：

1. `Chapter5.lua` 生成章节布局与逻辑格。
2. 逻辑格映射为 `materialGrid`。
3. `materialGrid` 生成 `edgeGrid`。
4. `materialGrid + edgeGrid + decalGrid` 编译成静态 chunk。
5. 运行时只绘制可视 chunk。
6. 如果后续补 `ambientZones`，它只能叠加在静态 chunk 之上。

### 5.2 数据职责

| 数据 | 责任 | 是否必须 |
|------|------|------|
| `logicGrid` | 通行、障碍、断崖、液面、封印等语义 | 必须 |
| `materialGrid` | 青石、裂石、白石、血石、焦岩等主材质 | 必须 |
| `edgeGrid` | 自动拼边后的边、角、端头、断崖边、岸线边 | 必须 |
| `decalGrid` | 剑痕、裂纹、灰烬、血迹、残剑、书页等静态细节 | 必须 |
| `ambientZones` | 冷雾、黑雾、血池脉冲、剑气余辉等区域级附加效果 | 可选 |

### 5.3 为什么这样拆

这样拆的好处是：

1. 地形逻辑仍然由网格控制，设计层能继续维护。
2. 视觉准确度主要提升在 `materialGrid -> edgeGrid` 这一段。
3. 性能成本主要压在一次性构建，而不是帧循环。
4. 即使最后完全不做 `ambientZones`，第五章画面也已经能比前四章明显提升。

## 6. 绘制规则

### 6.1 语义格最小集合

第五章逻辑格至少需要下面这些语义：

1. `walk_floor`
2. `walk_path`
3. `solid_wall`
4. `cliff_edge`
5. `abyss_void`
6. `ritual_platform`
7. `corrupted_floor`
8. `seal_surface`
9. `bridge_floor`
10. `water_or_blood`

### 6.2 主材质最小集合

第五章材质至少要能区分下面这些主地表：

1. `camp_dirt`
2. `camp_flagstone`
3. `ruin_bluestone`
4. `ruin_cracked_stone`
5. `forge_blackstone`
6. `forge_molten_crack`
7. `courtyard_moss_stone`
8. `cold_pool_jade`
9. `cold_pool_ice_edge`
10. `stele_pale_stone`
11. `library_burnt_floor`
12. `palace_white_stone`
13. `palace_corrupted_vein`
14. `blood_ritual_stone`
15. `abyss_charred_rock`
16. `abyss_flesh_rock`
17. `corridor_dark_stone`
18. `corridor_sword_metal`

### 6.3 自动拼边最低要求

第五章自动拼边至少覆盖下面四类交界：

1. 地表对地表
2. 地表对墙体
3. 地表对断崖
4. 地表对液面

规则如下：

1. 以四向邻接决定主轮廓。
2. 以对角关系修正内角和外角。
3. 变体在构建时确定，不在每帧重新随机。
4. `cliff_edge -> abyss_void` 必须单独处理，不允许只靠颜色变暗。

### 6.4 静态贴花最低要求

贴花必须是静态层，不得做成逐格动态特效。最低支持：

1. 剑痕
2. 小裂纹
3. 大裂纹
4. 灰烬
5. 血迹
6. 烧痕
7. 残剑
8. 书页或玉简残片

### 6.5 氛围效果规则

`ambientZones` 不是必须项。如果实现，只能满足以下条件：

1. 按区域挂，不按 tile 挂。
2. 数量极少，只给关键区域。
3. 不影响地形可读性。
4. 一旦成本超标，优先删除氛围效果，而不是牺牲自动拼边或静态贴花。

### 7.2 前营与山门

`太虚遗址前营` 和 `裂山门遗址` 的任务是先把“前线废墟感”和“正路已断”建立起来。

执行重点：

1. 前营中央地面尽量平整，方便 NPC 和功能点落位。
2. 山门必须是“被撕裂的结构”，不是“路中间放一堵墙”。
3. 山门中轴必须封死，左右各给一个可读的绕行口。

### 7.3 左路

左路承担“战斗遗迹 -> 锻造失控 -> 生活区沦陷”的叙事。

执行重点：

1. `问剑坪` 做开阔试剑场。
2. `铸剑地炉` 做热裂纹与焦黑炉区。
3. `栖剑别院` 做回廊、院墙、灵泉遗迹和生活中断感。

### 7.4 右路

右路承担“寒池污染 -> 传承断裂 -> 书阁焚毁”的叙事。

执行重点：

1. `洗剑寒池` 优先解决岸线准确度。
2. `悟剑碑林` 强调碑与地裂节奏，不堆随机碑。
3. `藏经书阁` 重点是焚毁后残留禁制，而不是普通废墟。

### 7.5 剑宫、深渊、回廊

这三段共同构成第五章后半场的视觉高点。

执行重点：

1. `太虚剑宫` 要规整、纯净、威严，但被侵蚀。
2. `镇魔深渊` 要破碎、下沉、失控，形成强反差。
3. `万剑回廊` 要在宽度和悬空感上与前面所有区域拉开差异，同时在空间上收住 `栖剑别院` 和 `藏经书阁` 两个深层终点。

## 8. 性能设计，改为默认路径约束

### 8.1 默认路径的硬约束

执行 AI 必须把实现限制在以下边界内：

1. 静态地表只在构建期生成，不在移动时逐格重算。
2. `chunk size` 固定优先 `16x16`。
3. 运行时只绘制可视 chunk，不重新拼边，不重新生成贴花。
4. 不新增全可视区逐格扫描的第二套特效循环。
5. 不新增全屏半透明叠层作为常驻效果。
6. 随机变体只在构建期决定一次。

### 8.2 允许的失效时机

静态 chunk 只允许在以下事件触发时失效：

1. 进入地图
2. 切换章节
3. 封印状态变化
4. 桥梁、机关、关键地形发生确定性变化

玩家普通移动、转向、战斗摄像机变化都不应该触发静态重建。

### 8.3 氛围效果的预算规则

如果最后实现 `ambientZones`，也只能遵守以下规则：

1. 只给少数关键区域上氛围效果。
2. 同屏氛围发射器数量必须很低。
3. 氛围效果必须比静态地表更容易被裁掉。
4. 只要测试显示发热明显增加，就先删氛围效果，不保它。

### 8.4 需要观测的数据

不新增产品开关，但必须保留开发期观测项。至少要能看到：

1. 静态地表构建耗时
2. chunk 重建次数
3. 可视 chunk 数量
4. 如果实现了 `ambientZones`，则额外记录可视发射器数量
