## 1. 变更摘要

将黑市卖出客户端门禁从 **"仓库 dirty + 3 项敏感道具白名单"** 升级为 **基于操作类型的统一未同步门禁**。

### 核心设计

| 维度 | BM-01A (旧) | BM-S2 (新) |
|------|------------|-----------|
| 消耗品覆盖 | 3 项白名单 (`_BM_SENSITIVE_CONSUMABLES`) | 全量消耗品自动覆盖 |
| 消耗路径标记 | 设 `WarehouseSystem._dirty` + 发 `save_request` | 设 `BlackMarketSyncState._dirtyConsume` |
| 仓库路径标记 | 设 `WarehouseSystem._dirty` + 发 `save_request` | 附加 `BlackMarketSyncState._dirtyWarehouse` |
| 卖出前置检查 | `WarehouseSystem.IsDirty()` | `BlackMarketSyncState.IsBlocked()` (三合一) |
| 新增商品维护 | 手动加白名单 | 无需维护，走 ConsumeConsumable 即覆盖 |

### 三合一阻断条件 (`BlackMarketSyncState.IsBlocked()`)

1. `_dirtyConsume == true` — 消耗操作未同步 → reason: `consume_unsync`
2. `_dirtyWarehouse == true` — 仓库操作未同步 → reason: `warehouse_unsync`
3. `SaveSession.IsDirty()` — 延迟合并窗口活跃 → reason: `save_session_active`

### 清除时机

- `game_saved` 事件 → `ClearAll()`
- `game_loaded` 事件 → `ClearAll()`

## 3. InventorySystem 消耗路径变更详情

**旧 (BM-01A)**:
```lua
-- 白名单命中 → 设仓库脏 + 发 save_request
if InventorySystem._BM_SENSITIVE_CONSUMABLES[consumableId] then
    WarehouseSystem.MarkDirty()
    EventBus.Emit("save_request")
end
```

**新 (BM-S2)**:
```lua
-- 全量消耗品 → 标记消耗脏标记（不发 save_request）
local _okBMS, _BMS = pcall(require, "systems.BlackMarketSyncState")
if _okBMS and _BMS then
    _BMS.MarkConsumeUsed(consumableId)
end
```

关键变化:
- 惰性 `pcall(require)` 确保 BMS 不可用时消耗路径不中断
- 不再发 `save_request`（消耗路径本身没有仓库操作）
- 门禁覆盖全量消耗品，新增黑市商品无需维护白名单

## 4. BlackMerchantUI 卖出守卫变更

**旧**:
```lua
if WarehouseSystem.IsDirty() then
    SetStatus("背包数据未同步，请稍后再试", C.textError, 3)
end
```

**新**:
```lua
local blocked, reason = BlackMarketSyncState.IsBlocked()
if blocked then
    SetStatus("背包数据未同步，请稍后再试", C.textError, 3)
    print("[BlackMerchantUI] SELL BLOCKED: " .. tostring(reason))
end
```

## 6. T11 跨文件 EventBus 实例不匹配修复

**问题**: T11 在 test runner 顺序执行时失败(`game_saved` 后 `BMS.IsBlocked()` 仍返回 true)。

**根因**: BM-01 测试先于 BM-01A 运行，加载了 BMS 模块并在 BM-01 的 EventBus 实例上注册了 `game_saved` handler。BM-01A 的 `clearModuleCache()` 清除了 EventBus 但未清除 BMS，导致 BMS 的 handler 残留在旧 EventBus 实例上，新 EventBus 实例上无 handler。

**修复**: `clearModuleCache()` 中追加 `systems.BlackMarketSyncState` 和 `systems.save.SaveSession`，确保跨测试文件时 BMS 随 EventBus 一起获得干净实例。
