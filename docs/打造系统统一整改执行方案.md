## 三、现状问题清单

### P0. 测试体系边界错误

现状：

- `test_jiefeng_forge_pipeline.lua`
- `test_sword_forge_comprehensive.lua`

已经不再顶层自动污染，但执行时仍然会：

- 改 `package.loaded`
- 注入 `InventorySystem.SetManager(mock)`

结论：

- 这两份测试必须永久保留为 `lupa_only`
- 绝不允许在 GM / 真实引擎进程中执行

### P1. 第五章装备生成语义错误

现状：

1. `CreateSaintEquipment()` 把圣衣/披风统一转成 `saintStat`
2. `CreateJiefengSword()` 清空来源古剑 `spiritStat`
3. `SwordForgeUI.DoForgeSword()` 原地变换时也会覆盖掉旧 `spiritStat`

设计目标：

1. `dizun_saint_ring`：固定 `saintStat`
2. `daozang_saint_armor`：随机 `spiritStat`
3. `saint_cape_ch5`：随机 `spiritStat`
4. `jiefeng_*`：保留原 `spiritStat`，新增 `saintStat`
5. `fabao_longhunling`：保留灵器链路，不注入 `saintStat`

### P1. 第四章随机套装池错误

现状：

- `LootSystem.ForgeRandomSetLingqi()` 直接扫 `EquipmentData.SetBonuses`

结果：

- 当前不是只出四套制式套装
- 会把非目标套装也抽出来

### P1. 执行层分叉严重

现状：

- 第四章靠 `DragonForgeUI`
- 第五章靠 `SwordForgeUI`
- 规则、扣费、生成器、结果写回分散在两个 UI 中

后果：

- 新功能接入成本高
- 测试覆盖分裂
- 错误容易重复制造

### P2. 提交流程非原子

现状：

- 先扣金币
- 先删材料/装备
- 再生成产物
- 再写入背包/装备栏

后果：

- 生成失败会吞资源
- 加包失败会吞资源

## 八、阶段 3：统一打造配置与执行器

### 目标

把第四章、第五章从“两套 UI + 两套执行逻辑”收束成“一套配方模型 + 一套执行器”。

### 任务 1：建立统一配置

新增统一配置结构：

```lua
EquipmentData.FORGE_STATIONS = {}
EquipmentData.FORGE_SET_POOLS = {}
EquipmentData.FORGE_RECIPES = {}
EquipmentData.FORGE_RECIPE_ORDER = {}
```

### 任务 2：建立统一执行器

新增：

- `scripts/systems/ForgeSystem.lua`

职责：

1. 读取统一配方
2. 校验金币、材料、背包、输入装备
3. 根据 `generator.type` 调生成器
4. 根据 `outputMode` 写回：
   - 装备栏原地替换
   - 背包新增
5. 统一发：
   - `save_request`
   - `equipment_changed`

### 任务 3：UI 降级为展示层

`DragonForgeUI.lua` 和 `SwordForgeUI.lua` 只保留：

1. 配方列表展示
2. 缺料提示
3. 调 `ForgeSystem.Execute(recipeId)`
4. 成功弹窗与刷新

不再各自维护完整执行链路。

### 任务 4：统一生成器协议

把当前生成逻辑统一成参数化调用，例如：

```lua
LootSystem.CreateRandomLingqi(spec)
LootSystem.CreateRandomSetLingqi(spec)
LootSystem.CreateSetEquipment(spec)
```

### 涉及文件

- `scripts/config/EquipmentData.lua`
- `scripts/systems/ForgeSystem.lua`
- `scripts/systems/LootSystem.lua`
- `scripts/ui/DragonForgeUI.lua`
- `scripts/ui/SwordForgeUI.lua`

### 完成标准

1. 新配方无需再复制新 UI 执行逻辑。
2. 第四章、第五章都可接到统一执行器。
3. 套装池、输入模式、输出模式都可配置化。

## 九、阶段 4：新增第五章 T10 随机灵器打造

### 目标

把第五章随机灵器打造接入地炉系统。

### 规则

1. 站点：`sword_forge`
2. 消耗：
   - `immortal_essence_blood x4`
   - `1000000` 金币
3. 产物：
   - `T10`
   - `quality = cyan`
4. 概率：
   - `70%` 普通 `T10` 随机灵器
   - `30%` 制式套装 `T10` 随机灵器
5. 制式套装池：
   - `xuesha`
   - `qingyun`
   - `fengmo`
   - `haoqi`

### 实现要求

1. 不新增硬编码专用函数名如 `ForgeRandomLingqiT10Ch5()`
2. 必须走统一生成器参数化方案
3. 必须走统一 `ForgeSystem`
4. 必须是背包模式，不是装备栏替换模式

### 建议配方定义

```lua
t10_random_lingqi_ch5 = {
    stationId = "sword_forge",
    label = "T10随机灵器",
    inputMode = "bag_forge",
    outputMode = "add_to_bag",
    gold = 1000000,
    consumables = {
        { id = "immortal_essence_blood", count = 4 },
    },
    generator = {
        type = "random_lingqi_mixed",
        tier = 10,
        quality = "cyan",
        setChance = 0.30,
        setPoolId = "lingqi_t10_standard",
    },
}
```

### 涉及文件

- `scripts/config/EquipmentData.lua`
- `scripts/systems/LootSystem.lua`
- `scripts/systems/ForgeSystem.lua`
- `scripts/ui/SwordForgeUI.lua`

### 完成标准

1. 地炉可打造 `T10` 随机灵器。
