## 3. 断线判定新旧对比

| 维度 | 旧方案 (W1) | 新方案 (N1) |
|------|-------------|-------------|
| **采样间隔** | 5 秒 | 3 秒 |
| **断线判定** | 单次 `GetServerConnection()==nil` → 立即 `connection_lost` | 连续 3 次失败 → `connection_lost`（9 秒宽限） |
| **恢复判定** | 单次 `GetServerConnection()~=nil` → 立即 `connection_restored` | 连续 2 次成功 → `connection_restored`（滞回防抖） |
| **中间态** | 无 | `connection_unstable`（首次失败即触发，轻提示） |
| **状态模型** | 二态：connected / disconnected | 三态：connected / unstable / disconnected |
| **状态存储** | SaveSystem._lastConnState (布尔) | NetworkStatus 模块（全局单例，可被任意模块查询） |

**状态转移图**：

```
CONNECTED ──首次失败──→ UNSTABLE ──连续3次失败──→ DISCONNECTED
    ↑                       ↑                        │
    └──连续2次成功────────────┴───────连续2次成功───────┘
```

## 4. 强提示触发条件说明

| 层级 | 事件 | 触发条件 | UI 表现 |
|------|------|----------|---------|
| 轻提示 | `connection_unstable` | 首次采样失败（CONNECTED → UNSTABLE） | 黄色底部横幅："网络波动中，请稍候..." |
| 全屏阻断 | `connection_lost` | 连续 3 次采样失败（UNSTABLE → DISCONNECTED） | 半透明遮罩 + 居中弹窗：标题"网络连接已中断"，描述"当前进度可能无法同步/高风险操作已暂停"，指引"请检查网络后重新连接/或重新登录游戏" |
| 恢复清除 | `connection_restored` | 连续 2 次采样成功 | 清除所有断线 UI（横幅+遮罩） |

**防降级保护**：当全屏阻断弹窗已显示时，`connection_unstable` 事件不会将其降级为轻提示横幅。

## 6. 哪些只读请求增加了容错

| 模块 | API | 容错策略 |
|------|-----|----------|
| CloudStorage.lua | `GetRankList()` | 当 `NetworkStatus.IsUnstable()` 且非 `IsDisconnected()` 时，延迟 2s 重试 1 次（通过 `CloudStorage.Tick()` 驱动的延迟任务队列实现） |
| SaveSystemNet.lua | `FetchSlots()` / `Load()` | 已有 MAX_RETRY=3, RETRY_INTERVAL=1.5s（不变） |

**实现细节**：
- 在 CloudStorage.lua 中新增 `_delayedTasks` 队列和 `ScheduleDelayedLocal()` 函数
- 延迟任务由已有的 `CloudStorage.Tick()`（每帧调用）驱动执行
- 重试次数通过 `events._n1RetryCount` 标记，最多重试 1 次防止无限循环
