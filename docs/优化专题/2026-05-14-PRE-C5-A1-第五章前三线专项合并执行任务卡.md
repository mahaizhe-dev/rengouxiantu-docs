## 5. 分步执行要求

## Step 1. 黑市最小闭环 + 自动回收

### 黑市必须达成的行为

1. 买入后，只有该 `itemId` 进入锁定  
2. 其他未锁商品仍可进入出售确认  
3. 成功存档后，该商品真实解锁  
4. 重登后，解锁状态保持正确  
5. 商品卡不再出现“整个黑市一起同步中”  
6. 商品卡、顶部提示、确认出售三层语义一致

### 黑市 UI 规则

- 商品卡只显示：
  - `出售`
  - `🔒锁定`
- 不显示：
  - `⏳同步中`

### 黑市新增商品规则

如果本轮碰到新增黑市商品，完成报告必须写清：

1. 属于 `A 类` 还是 `B 类`
2. 如果是 `B 类`，本地先变化路径是什么
3. 是否已接入保护锁、操作闸和测试

### 自动回收必须达成的行为

1. 同日只执行一次
2. 次日会再次执行
3. 跨月不出错
4. 跨年不出错
5. 长停服再启动不出错

## Step 2. N1

### N1 必须达成的行为

1. 短暂抖动不立刻断线
2. 持续断连进入强提示
3. 只读请求有短容错
4. 高风险写操作不自动重发

## 6. 测试要求

### 黑市真实路径

至少提交以下验证：

1. `A` 锁住，`B` 未锁
   - `A` 不能卖
   - `B` 能进确认框
2. `A` 存档后解锁
3. `A` 重登后仍解锁
4. 不再出现整市场 `同步中`
5. 如果本轮新增黑市商品，必须验证其 `A/B` 分类与对应行为是否成立

### 自动回收真实路径

至少提交以下验证：

1. 同日不重复
2. 次日触发
3. 跨月触发
4. 跨年触发

### N1 真实路径

至少提交以下验证：

1. 瞬时波动不立刻 lost
2. 持续断连强提示出现
3. 只读请求短容错
4. 写请求不自动补发

必须提交：

- `python3 scripts/tests/_run_via_lupa.py`
- `python3 scripts/tests/_run_via_lupa.py --all`

并附原始输出。
