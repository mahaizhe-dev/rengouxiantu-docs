## 8. 架构说明

### 输出格式示例

```
[TEST] save_optimization
[PASS] save_optimization (81/81 passed)
[TEST] prison_tower
[FAIL] prison_tower error=7/442 sub-test(s) failed
[TEST] event_system
[SKIP] event_system reason=engine_required: config.EventConfig, systems.EventSystem
[SUMMARY] total=18 passed=3 failed=1 skipped=14
[FAILED CASES]
  - prison_tower: 7/442 sub-test(s) failed
```

### TestRegistry 模式

| mode | 说明 | 执行方式 |
|------|------|---------|
| `run_file` | 纯 Lua 文件，dofile 执行 | dofile(path) → 检查返回的 {passed,failed,total} |
| `require_runall` | 模块化测试 | require(entry).RunAll() → 适配多种返回值格式 |
| `skip` | 引擎依赖，暂跳过 | 仅记录 skip_reason |

### TestRunner 返回值适配

Runner 内部支持 3 种 RunAll() 返回模式，覆盖现有全部 engine_required 测试的导出习惯：

| 返回模式 | 适用测试 | 处理方式 |
|---------|---------|---------|
| `(pass, fail)` 两个数字 | 11 个测试 | fail > 0 则判定失败 |
| `(bool)` 单个布尔值 | test_p0_regression | false 则判定失败 |
| `(bool, pass, fail)` 三个值 | test_batch_consumable | 第三个值 > 0 则判定失败 |
