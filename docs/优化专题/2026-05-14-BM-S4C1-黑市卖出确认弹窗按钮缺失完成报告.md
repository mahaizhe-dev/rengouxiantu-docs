## 3. 直接根因说明

### 根因

`ShowConfirmDialog` 函数（BlackMerchantUI.lua 原 ~L361）在构建 `children` 数组时，使用了 Lua 三元表达式模拟：

```lua
children = {
    UI.Label { ... },           -- [1] 标题
    UI.Panel { ... },           -- [2] 分隔线
    UI.Label { ... },           -- [3] 交易详情
    UI.Label { ... },           -- [4] 附加信息
    isBuy and UI.Label { ... } or nil,  -- [5] 保护期提醒
    UI.Panel { ... buttons },   -- [6] 按钮行
}
```

当 `isBuy = false`（卖出路径）时，`[5]` 位置为 `nil`，导致数组中间出现空洞。

### 截断机制

UI 框架 `urhox-libs/UI/Core/Widget.lua:715` 使用 `for i, child in ipairs(props.children)` 遍历子元素。Lua 的 `ipairs` 遇到 `nil` 即停止迭代，因此 `[6]` 处的按钮行 Panel 永远不会被渲染。

- **buy 路径**：`isBuy = true` → `[5]` = UI.Label 实例（非 nil）→ `ipairs` 正常遍历到 `[6]` → 按钮行显示 ✅
- **sell 路径**：`isBuy = false` → `[5]` = `nil` → `ipairs` 在 `[5]` 停止 → 按钮行丢失 ❌

## 4. 修复前后行为说明

### 修复前

| 路径 | 弹窗标题 | 交易信息 | 保护提醒 | 按钮行 |
|------|---------|---------|---------|--------|
| buy  | ✅ 显示 | ✅ 显示 | ✅ 显示 | ✅ 显示 |

### 修复后

| 路径 | 弹窗标题 | 交易信息 | 保护提醒 | 按钮行 |
|------|---------|---------|---------|--------|
| buy  | ✅ 显示 | ✅ 显示 | ✅ 显示 | ✅ 显示 |
| sell | ✅ 显示 | ✅ 显示 | — 不适用 | ✅ **显示** |

### 修复方法

将 `children` 数组从内联字面量改为预构建本地表 `dialogContent`，用 `if isBuy then table.insert(...)` 条件插入保护期提醒，最后 `table.insert(dialogContent, buttonRowPanel)` 确保按钮行始终追加到末尾，不产生 nil 空洞。
