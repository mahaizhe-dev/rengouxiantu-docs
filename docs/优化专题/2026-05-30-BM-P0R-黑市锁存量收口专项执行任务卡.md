# 2026-05-30 BM-P0R 黑市锁存量收口专项执行任务卡

## 1. 任务目标

只做一件事：

**把当前已接入黑市的 `B 类` 商品，本地消耗口逐个收干净。**

本卡不是黑市新功能卡，不是黑市整体模型重构卡，不是自动回收/N1/服务端全权威合并卡，也不是第五章内容开发卡。

---

## 2. 背景与结论

### 2.1 当前真实问题

黑市锁已经存在，但仍然可以出现以下恶性路径：

1. 黑市买入道具进入锁定期
2. 某个本地系统先用总量判断“材料足够”
3. 实际扣除时，锁定堆叠不会被消费
4. 上层代码又没有检查扣除返回值
5. 结果继续生效
6. 玩家仍可把这批锁定道具卖回黑市

这不是“黑市卖出 UI 问题”，而是：

**黑市锁的语义没有真正收口到所有本地消耗口。**

### 2.2 当前代码基线

- `scripts/systems/BlackMarketTradeLock.lua`
  - 规则目标已经写明：保护期内阻断一切本地状态变更
- `scripts/systems/InventorySystem.lua`
  - `CountConsumable` 统计总量
  - `CountUnlockedConsumable` 统计未锁定量
  - `ConsumeConsumable` 只消费未锁定堆叠

当前风险不在底层“锁不存在”，而在上层业务口仍然大量沿用：

`先 CountConsumable 判定 -> 再 ConsumeConsumable -> 不检查返回值 -> 继续发收益`

### 2.3 为什么过去几轮黑市优化没有最终落地

不是单一原因，而是两段偏移叠加：

1. 前期方案太大
   - 目标同时包含黑市、自动回收、存档、服务端校验、章节护栏
   - 执行面过宽，容易失焦
2. 后期虽然收窄了
   - 但重心逐步偏到“黑市卖出卡片状态 / 顶部提示 / UI 语义”
   - 没有持续盯住真正危险的“所有本地消耗口”

所以这次不再沿用“黑市优化”这个笼统目标，而是改成：

**按已上线黑市 `B 类` 商品，逐个审计并封住真实本地消耗口。**

### 2.4 第五章现状对方案的影响

旧文档里很多判断建立在“第五章前”的前提上，但这个前提现在已经失效。

第五章已经把高风险物品接进黑市，包括但不限于：

- `tiandi_fragment_*`
- `lingyu_*`
- `sword_intent_crystal`
- `abyss_seal_shard`
- `immortal_essence_blood`
- `taixu_jianling_box`
- 全部 `book_*_2/3/4`

因此本轮方案必须按：

**存量章节回收 + 高危商品止血 + 分批收口**

来执行，不能再写成“第五章前预防性护栏”。

## 3. 本次执行原则

### 原则 A：只修真实失守口，不做黑市大重构

本轮唯一目标是关掉“锁定物品本地消耗不扣货但结果生效”的真实口子。

不允许顺手扩成：

- 黑市整体模型重建
- 存档体系重构
- 服务端全权威库存
- 第五章系统返工

### 原则 B：按“入口”修，不按“模块抽象”修

执行单位不是“炼丹系统”“宠物系统”“第五章系统”。

执行单位是：

**某个黑市 `B 类` 商品在哪个本地入口会被消费。**

### 原则 C：展示可看总量，可执行必须看未锁定量

允许：

- UI 展示总持有量
- 图鉴展示总持有量

不允许：

- 用总量直接判定是否可执行
- 用总量替代可消费资格

### 原则 D：消耗失败，后续结果一律不得推进

凡是：

- 加属性
- 学技能
- 升技能
- 炼丹成功
- 打造成功
- 解锁成功
- 激活成功
- 发金币
- 发灵韵
- 发奖励

都必须以“实际扣除成功”为硬前置。

### 原则 E：本轮没审完的 `B 类` 商品，不准带病保留

如果来不及全修：

- 可以临时下线
- 不允许“先上线继续卖，后面再补”

## 8. 统一修补规则

以下规则必须在所有入口保持一致。

### 规则 1：展示和执行分离

- 展示层可读总量
- 执行层必须读未锁定可用量

### 规则 2：锁优先于“数量不足”

如果：

- 总量足够
- 但未锁定量不足

必须返回黑市锁提示，而不是普通“材料不足”。

### 规则 3：扣除成功是唯一推进条件

允许的顺序只有：

1. 校验可执行
2. 实际扣除
3. 判断扣除成功
4. 发放结果
5. 记脏/存档/刷新 UI

不允许：

1. 校验总量
2. 先发结果
3. 再假设扣除应该成功

### 规则 4：技能书和材料列表不等于可执行资格

尤其是：

- `GetSkillBookList`
- 背包材料展示列表
- 合成面板展示数量

这些只能做显示，不得直接等价为“可消费”。

### 规则 5：锁定失败不能转化为其他收益

锁定物品扣除失败时，不允许出现任何替代收益路径，包括：

- 先扣金币再推进
- 先扣灵韵再推进
- 先写成长再提示失败
