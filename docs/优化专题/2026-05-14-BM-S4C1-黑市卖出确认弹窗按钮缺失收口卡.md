# BM-S4C1：黑市卖出确认弹窗按钮缺失收口卡

## 1. 任务定位

- **任务 ID**：`BM-S4C1`
- **任务类型**：`超小 UI 收口`
- **目标**：修复黑市卖出确认弹窗中“确认/取消按钮行缺失”的直接 UI 问题。

本卡只处理一个问题：

- 商品卡上的 `出售` 按钮是亮的
- 点击后能弹出“确认出售”弹窗
- 但弹窗底部按钮行不显示，导致无法继续操作

这一步**不处理**黑市锁模型、全局未同步门、服务端卖出校验，只修这个直接 UI 缺陷。

---

## 2. 当前判断

根据现有代码，最可疑路径在：

- `scripts/ui/BlackMerchantUI.lua`
- `ShowConfirmDialog(action, itemId)` 内部 `children` 结构

当前实现里，买入提示使用了：

```lua
isBuy and UI.Label { ... } or nil
```

这会在 `sell` 路径下把中间子节点变成 `nil`。  
如果 UI 框架对子节点按顺序遍历并在中间 `nil` 处截断，后面的按钮行就会丢失。

**本轮优先假设**：
- 根因是 `children` 数组中间出现 `nil`
- 不是锁状态问题本身
- 不是服务端状态问题本身

---

## 3. 本轮目标

只做 3 件事：

1. **修复卖出确认弹窗按钮行缺失**
   - `sell` 路径下必须稳定显示：
     - `取消`
     - `确认出售`

2. **保证买入路径不被修坏**
   - `buy` 路径下原有提示和按钮仍然正常

3. **补一个最小回归测试或结构验证**
   - 至少验证 `ShowConfirmDialog("sell", itemId)` 的按钮行仍然存在
