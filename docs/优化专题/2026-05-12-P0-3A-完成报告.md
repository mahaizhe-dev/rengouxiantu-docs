## 4. 关键结论

### main_character_save 边界

- **唯一主 key**: `save_{slot}` (v11 统一格式)
- **包含 22 个字段域**: version, player, equipment, backpack, skills, collection, pet, quests, shop, titles, challenges, sealDemon, trialTower, artifact, fortuneFruits, seaPillar, prisonTower, warehouse, yaochi_wash, event_data, openedXianyuanChests, bossKillTimes
- **透传字段**: atlas, accountCosmetics (embedded 但归属 account_level_meta)
- **服务端旁路**: pendingXianyuanRewards (Load 时附加)
- **legacy 读兼容**: save_data_{slot}, save_bag1/2_{slot} (v10 格式)

### slot_index_meta 边界

- `slots_index`: 全局唯一，包含所有槽位元数据摘要
- `player_level`: 等级整数副本，排行辅助

### backup_recovery_meta 边界

- 7 个 key 家族: login, prev, deleted, premigrate, premigrate_bag1/2, save_migrate_backup, migration_backup_done
- 全部是备份/恢复链，不参与正常读写流程
- P0-5 strict mode 已阻止 login_/prev_ 在网络模式写入

### account_level_meta 边界

- 3 个 key: account_bulletin, account_atlas, account_cosmetics
- 全部账号级，跨角色共享
- atlas 双写问题已确认（E1 歧义点）

### server_authority_side_data 边界

- v2 排行榜: rank2_score/info/kills/time (服务端 RankHandler.ComputeRankFromSaveData 计算)
- 试炼/镇狱排行: trial_floor/info, prison_floor/info (服务端从 coreData 计算)
- 服务端旁路: pending_redeem_s{slot}, dungeon_pending_sishoudao/reward_{slot}
- 版本守卫: _code_ver
- v1 排行 (已弃): rank_score/info/time, boss_kills (standalone 仍写入)
