## 7. 自动回收不受影响的真实验证说明

### 代码级验证（非文档推断）

1. **SellGuard 接口审计（T20 / I9）**：
   - `SellGuard` 仅导出 3 个函数：`CheckSellAllowed`, `GetSafeItemIds`, `GetHighRiskItemIds`
   - 不存在 `OnRecycle`, `HandleRecycle`, `RecycleTick`, `ExecuteRecycle` 等接口
   - 这是代码级断言，不是架构推断

2. **Handler 回收代码审计（I10）**：
   - `BlackMerchantHandler` 导出 `RecycleTick` 和 `ExecuteRecycle` 方法（回收功能）
   - I10 通过读取 Handler 源码，验证 RecycleTick/ExecuteRecycle 代码块**不引用 SellGuard**
   - 回收路径通过 `MoneyCost(SYSTEM_UID, stockKey)` 直接操作 serverCloud.money
   - 回收路径不经过 `HandleSell()` → 不触发 `SellGuard`

3. **路径隔离总结**：
   - **卖出路径**：`C2S_BMSell → HandleSell() → SellGuard.CheckSellAllowed() → 拒绝/放行`
   - **回收路径**：`ScheduledUpdate → RecycleTick() → ExecuteRecycle() → MoneyCost()` ← 不经过 HandleSell
   - 两条路径在代码层完全独立
