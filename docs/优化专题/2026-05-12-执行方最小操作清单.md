## 1. 先做什么

第一件事只能做 `P0-1 建立统一测试入口`。

不允许直接开始以下任务：

- 存档 DTO
- 迁移入口收敛
- 服务端存档服务化
- `main.lua` 拆分
- `server_main.lua` 拆分

原因：

- 没有测试入口，后面所有重构都缺回归护栏。

模板：

```md

变更类型:

新增测试:
失败判据:

```

如果写不出来，不允许动代码。

## 4. 高风险任务必须满足的 5 个条件

凡是涉及 `存档 / 网络 / 服务端写入`，必须同时满足：

1. 保留旧路径
2. 有开关
3. 有自动化回归
4. 有失败判据
5. 有回滚方式

缺一个就停止。

凡是涉及 `存档 / 网络 / 服务端写入`，还必须额外对照：

- [rules.md](../架构文档/rules.md) 的线上存档红线
- 旧 key 是否保持兼容
- 旧迁移函数是否保留
- 是否引入了必须配对同步的新字段

---

## 5. 遇到这些情况必须停手

1. 同一个存档字段有两个权威来源
2. 客户端和服务端槽位号含义不一致
3. 发现“假成功”但调用方依赖这个假成功
4. 新旧 serializer 关键字段对不上
5. 无法证明奖励发放是否幂等
6. 无法证明 `requestId` 是否贯穿链路
7. 需要一次性删除旧逻辑才能继续
8. 一个任务要改超过 8 个核心运行文件

出现任一项，停止编码，改为输出问题说明。

## 7. 每次结束时必须交付的东西

模板：

```md
任务ID:
本次结论: 完成 / 部分完成 / 停止

1. 做了什么
2. 修改了哪些文件
3. 新增了哪些测试
4. 测试怎么跑
5. 测试结果
6. 还剩什么风险
7. 怎么回滚
8. 建议下一步
9. 对 `rules.md` 的符合性说明
```

没有这份交付说明，等于没完成。

## 8. 绝对禁止

1. 直接编辑历史快照文件
2. 跳过 `P0-1`
3. 没有测试就改存档主路径
4. 没有开关就改线上高风险逻辑
5. 一次同时改存档结构、迁移入口、服务端事务写入
6. 用“我感觉没问题”代替回归结果
