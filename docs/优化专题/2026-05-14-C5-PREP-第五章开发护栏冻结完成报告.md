## 3. 步骤 2：黑市新增商品准入最终规则

> 规则文档已独立存在：`docs/优化专题/2026-05-14-黑市新增商品准入规则.md`
>
> 以下为最终冻结版摘要。

### 3.1 A 类：纯库存型（可直接接入配置）

同时满足以下**全部**条件：

1. 不能直接使用
2. 不能直接卖钱/转资源
3. 不会被炼丹、打造、神器、附灵等本地系统消耗
4. 不会因仓库存取影响黑市卖出一致性
5. 无已知"客户端先变化、服务端后结算"风险路径

### 3.2 B 类：受保护接入型（必须额外接入保护）

满足以下**任一**条件即为 B 类：

1. 可直接使用
2. 可直接卖钱/转资源
3. 可开盒
4. 会进入仓库或因仓库存取影响卖出一致性
5. 会被本地系统消耗
6. 本地先变化路径暂时无法明确

**B 类必须同时完成**：
- 接入交易保护锁（BlackMarketTradeLock）
- 接入统一操作闸（BlackMarketSyncState.MarkConsumeUsed/MarkWarehouseOp）
- 新增对应测试

### 3.3 默认规则

**判不清 → 默认 B 类。**

不允许"先上配置，后面再看"。

### 3.4 执行 AI 判断流程

```
新增黑市商品
  ↓
检查 5 项 A 类条件是否全部满足
  ↓
├─ 全满足 → A 类，直接接入配置
└─ 任一不满足或不确定 → B 类
      ↓
    ├─ 说明本地先变化路径
    ├─ 接入保护锁
    ├─ 接入操作闸
    └─ 新增测试
```

## 4. 步骤 3：高风险写入口最终规则

### 4.1 什么是高风险写入口

凡是第五章新增 UI 中包含以下**任一**操作的入口，即为高风险写入口：

| # | 操作类型 | 典型函数 |
|---|---------|---------|
| 1 | 购买 | `C2S_*Buy`, `C2S_SkinShopBuy` |
| 2 | 兑换 | `C2S_*Exchange`, `C2S_RedeemCode` |
| 3 | 领奖 | `C2S_*Claim`, `C2S_TrialClaimDaily` |
| 4 | 开箱 | `C2S_*OpenFudai`, `C2S_*OpenBox` |
| 5 | 抽取 | `C2S_*Pull`, `C2S_*Draw` |
| 6 | 合成确认 | `C2S_*Craft`, `C2S_*Forge` |
| 7 | 激活确认 | `C2S_*Activate`, `C2S_*Use` |
| 8 | 卖出 | `C2S_*Sell`, `C2S_BlackMerchantSell` |
| 9 | 提交 | `C2S_*Submit`, `C2S_DaoTreeComplete` |
| 10 | 重置 | `C2S_*Reset`, `C2S_GMResetDaily` |

### 4.2 必须接入的阻断逻辑

所有高风险写入口**必须**在 `SendRemoteEvent` 之前接入：

```lua
if NetworkStatus.IsDisconnected() then
    -- 显示提示（可选）
    return false
end
```

### 4.3 推荐实现模式

参考 BlackMerchantUI.lua 的统一 SendToServer 封装：

```lua
local function SendToServer(eventName, data)
    if NetworkStatus.IsDisconnected() then
        print("[WARN] 网络断连，阻断 " .. eventName)
        return false
    end
    connection:SendRemoteEvent(eventName, true, data or VariantMap())
    return true
end
```

### 4.4 执行 AI 判断流程

```
第五章新增 UI 文件
  ↓
搜索 SendRemoteEvent / SendToServer
  ↓
├─ 无远程调用 → 无需处理
└─ 有远程调用
      ↓
    判断是否属于上述 10 类操作
      ↓
    ├─ 是 → 必须接入 IsDisconnected() 阻断
    └─ 否（纯查询/只读） → 建议接入，非强制
```

### 4.5 例外说明

- GMConsole.lua：GM 工具在线上关闭（GM_ENABLED=false），不计入玩家侧风险
- 纯查询类请求（如 C2S_*Query）：建议接入但不强制，可在 unstable 时延迟重试

### 第五章新增系统存档影响检查表

每个新增系统在开发前**必须**逐项回答以下问题。无法回答的项目不允许直接上线。

| # | 检查项 | 回答（是/否） | 如果"是"的处理要求 |
|---|--------|-------------|-------------------|
| 1 | 是否修改玩家资源（金币/灵韵/仙石/金条）？ | | 必须接 save_request + 服务端 V3/V4 校验覆盖 |
| 2 | 是否修改背包/仓库内容？ | | 必须接 save_request + 服务端 V6 仓库校验覆盖 |
| 3 | 是否修改装备属性？ | | 确认 fillEquipFields 配对同步 + 不碰已有装备公式 |
| 4 | 是否修改章节/挑战进度？ | | 确认进度字段在 SaveDTO.DTO_FIELDS 中已注册 |
| 5 | 是否接 save_request 触发存档？ | | 确认 EventBus.Emit("save_request") 在状态变更后调用 |
| 6 | 是否进入 SaveSession（延迟合并窗口）？ | | 确认 flush 时机正确，不遗漏脏数据 |
| 7 | 是否新增存档字段（DTO / coreData）？ | | 必须：递增 CURRENT_SAVE_VERSION + 写迁移函数 + 旧字段保留默认值 |
| 8 | 是否影响旧存档兼容？ | | 禁止删除/重命名已有字段；新字段必须 nil 兼容 |
| 9 | 是否新增可消耗品？ | | 必须在 GameConfig.CONSUMABLES 注册 + AddConsumable 返回值检查 |
| 10 | 是否涉及黑市交易？ | | 按 §3 准入规则判断 A/B 类 |
