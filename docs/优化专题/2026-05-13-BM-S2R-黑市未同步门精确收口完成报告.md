## 1. 问题描述

BM-S2 将 **所有** `ConsumeConsumable` 调用统一纳入黑市未同步门禁，导致非黑市消耗品（`exp_pill`、`lingyun_fruit`、`item_guardian_token` 等）也会误触发门禁，阻断黑市卖出操作。

**根因**：`InventorySystem.ConsumeConsumable` 中的 `MarkConsumeUsed` 不区分物品是否属于黑市可卖物，所有消耗操作一律置脏。

---

## 2. 修复方案

**收窄消耗门禁判定**：仅当消耗物品存在于 `BlackMerchantConfig.ITEMS` 时才调用 `MarkConsumeUsed`。

设计原则：
- **BMConfig.ITEMS 驱动**：以黑市商品配置表为唯一过滤源
- **fail-closed**：`BMConfig` 加载失败时保守置脏，不漏过
- **非侵入**：仓库路径（存取操作）不受影响，保留 BM-01/BM-01A 止血效果

## 4. 核心代码差异

### 4.1 InventorySystem.lua (lines 712-723)

```lua
-- BM-S2R: 黑市可卖物消耗才置脏（非黑市消耗品不触发门禁）
-- BMConfig 加载失败时保守置脏（fail-closed）
local _okBMS, _BMS = pcall(require, "systems.BlackMarketSyncState")
if _okBMS and _BMS then
    local _okBMC, _BMC = pcall(require, "config.BlackMerchantConfig")
    if (not _okBMC) or (not _BMC) or (_BMC.ITEMS and _BMC.ITEMS[consumableId]) then
        _BMS.MarkConsumeUsed(consumableId)
    end
end
```

**判定逻辑**：
1. `BMConfig` 加载失败 → 保守置脏（fail-closed）
2. `BMConfig.ITEMS` 存在且包含 `consumableId` → 置脏
3. 其他情况（非 BM 消耗品）→ 不置脏，不触发门禁

### 4.2 BlackMarketSyncState.lua (设计原则更新)

```diff
-- 设计原则：
--   1. 不依赖手写物品名称白名单 — 消耗路径由 BMConfig.ITEMS 驱动（BM-S2R 收口）
--   2. 新增黑市商品自动进入门禁 — 只要在 BMConfig.ITEMS 中且走 ConsumeConsumable 就被覆盖
+--   5. 非黑市消耗品（exp_pill、lingyun_fruit 等）不误拦黑市卖出
```

## 8. 误拦修正说明

### 修正前（BM-S2）

所有 `ConsumeConsumable` 调用均触发 `MarkConsumeUsed`，导致：
- `exp_pill`（经验丹）消费 → 门禁触发 → 黑市卖出被阻断
- `lingyun_fruit`（凌云果）消费 → 门禁触发 → 黑市卖出被阻断
- `item_guardian_token`（守护令）消费 → 门禁触发 → 黑市卖出被阻断

### 修正后（BM-S2R）

仅 `BMConfig.ITEMS` 中的物品触发门禁：
- `wubao_token_box`（乌堡令盒）消费 → 门禁触发 ✅
- `gold_brick`（金砖）消费 → 门禁触发 ✅
- `lingyu_xuesha_lv1`（附灵玉）消费 → 门禁触发 ✅
- `exp_pill`（经验丹）消费 → **不触发** ✅
- `lingyun_fruit`（凌云果）消费 → **不触发** ✅
- `item_guardian_token`（守护令）消费 → **不触发** ✅
