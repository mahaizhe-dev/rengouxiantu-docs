## 1. 目标摘要

BM-S4A 的核心偏差是：黑市买入消耗品时**先并入旧未锁堆，再整堆加锁**，导致玩家原有未锁库存被一起冻结。本次收口任务解决 4 个问题：

| # | 问题 | 解决方案 |
|---|------|---------|
| 1 | 买入消耗品没有强制分堆 | 服务端 `AddLockedNewStack` / 客户端 `AddConsumableLockedNewStack` |
| 2 | 旧未锁库存被一起锁住 | 新入口绝不合并已有堆，锁元数据在创建时预置 |
| 3 | 玩家无法从 UI 感知锁定堆 | `ImageItemSlot` 右上角 🔒 锁图标 |
| 4 | 黑市 NPC 面板无事前说明 | NPC 对话气泡常驻保护提示 + 买入确认弹窗保护提醒 |

## 3. 核心实现说明

### 3.1 服务端如何实现"强制新建锁堆"

**入口**: `BackpackUtils.AddLockedNewStack(backpack, consumableId, amount, itemName, batchId, duration)`

```lua
-- 关键逻辑：
-- 1. 遍历背包 1..60 找空槽
-- 2. 每个空槽创建新堆，锁元数据在 table 构造时直接写入：
backpack[key] = {
    category     = "consumable",
    consumableId = consumableId,
    count        = addCount,
    name         = itemName,
    bmLockUntil  = os.time() + duration,    -- 创建即锁
    bmLockSource = "black_market",
    bmLockBatchId = batchId,
}
-- 3. 绝不搜索已有堆，绝不合并
-- 4. 超过 MAX_STACK(9999) 自动拆多堆
-- 5. 背包满时返回剩余未放入的数量
```

**调用处**: `BlackMerchantHandler.HandleBuy` 直接调用 `BackpackUtils.AddLockedNewStack`，替代旧路径 `AddToBackpack` + 循环 `TradeLock.ApplyLock`。

### 3.2 客户端如何实现"强制新建锁堆"

**入口**: `InventorySystem.AddConsumableLockedNewStack(consumableId, amount, itemName, batchId, duration)`

```lua
-- 与服务端逻辑完全对齐：
-- 1. 遍历背包找空槽
-- 2. 创建新堆时直接写入锁元数据
-- 3. 不调用 AddConsumable（它会合并已有堆）
-- 4. 调用 EventBus.Emit("backpack_changed") 通知 UI 刷新
```

**调用处**: `BlackMerchantUI.HandleBMResult` 的买入成功路径，替代旧 `AddConsumable` + `TradeLock.ApplyLock`。

### 3.3 旧路径是否仍存在"先并老堆再加锁"

**否**。旧路径已被完全替代：

| 位置 | 旧路径 | 新路径 |
|------|--------|--------|
| 服务端 HandleBuy | `AddToBackpack` → 循环 `ApplyLock` | `AddLockedNewStack` |
| 客户端 HandleBMResult | `AddConsumable` → `ApplyLock` | `AddConsumableLockedNewStack` |

`TradeLock.ApplyLock` 函数保留（其他场景可能复用），但黑市买入路径已不再调用。

### 3.4 锁图标放在哪些界面、怎样展示

**界面**: `ImageItemSlot`（背包物品格子渲染组件，所有使用该组件的界面均生效）

**展示方式**:
- 右上角绝对定位 `Label`，内容为 🔒 emoji
- 尺寸: `max(14, floor(slotSize * 0.3))` 像素，字号: `max(9, lockSize - 4)`
- 颜色: `{255, 180, 80, 255}`（橙金色）
- 显隐逻辑: `UpdateDisplay()` 末尾判断 `TradeLock.IsLocked(item)`

### 3.5 黑市主面板提示放在哪、文案是什么

**位置 1 — NPC 对话气泡**（`BlackMerchantUI.lua` `_buildDialogueBubble` / NPC 面板对话区）:
```
⚠ 购入商品将暂时冻结，成功存档后自动解锁
```

**位置 2 — 买入确认弹窗**（`BlackMerchantUI.lua` 买入确认对话框）:
```
⚠ 购入后该商品将暂时冻结（保护期），存档成功后自动解锁
```
