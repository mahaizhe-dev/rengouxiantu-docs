## 1. 目标摘要

在黑市买入消耗品后，对新增物品施加 **5 分钟交易保护锁**（或下次存档成功自动解锁），锁定期间禁止一切本地操作（使用、消耗、本地卖出、仓库操作、炼制、激活、锻造、黑市再卖出），并在服务端做逐堆叠锁态校验替代 BM-S3R 的 SellGuard 全拦截。

### 核心设计要点

| 维度 | 方案 |
|------|------|
| 锁字段 | `bmLockUntil`(unix timestamp) / `bmLockSource`("black_market") / `bmLockBatchId`("bm_{t}_{r}") |
| 保护时长 | 300 秒（`LOCK_DURATION`） |
| 解锁条件 | ① `os.time() > bmLockUntil`  ② `game_saved` 事件触发 `ClearAllOnSaveSuccess` |
| 堆叠隔离 | 锁定栈 ≠ 未锁定栈；不同 batchId 也不合并 |
| 客户端门禁 | `IsOperationBlocked()` 统一闸；UI 卖出前 `CountUnlockedConsumable` 检查 |
| 服务端校验 | HandleSell §4.5 `BackpackUtils.CountUnlockedItem()` 替代 SellGuard 全拦截 |
| 存档持久化 | `SaveSerializer.SerializeItemFull` 序列化 3 字段；反序列化自动透传 |
| 版本升级 | `v23 → v24`（noop 迁移，nil-safe 新字段） |

## 5. 安全与风险评估

| 风险项 | 缓解措施 |
|--------|---------|
| 旧存档无锁字段 | v24 noop 迁移 + nil-safe 判定（`bmLockUntil` 为 nil → not locked） |
| 锁定物品堆叠溢出 | 锁定栈独立，不与未锁定栈合并；背包满时正常返回 AddConsumable 失败 |
| 服务端绕过 | HandleSell §4.5 在读取存档后做 `CountUnlockedItem`，客户端锁可被绕过但服务端拦截 |
| 循环依赖 | BlackMarketSyncState 通过 `pcall(require, ...)` 引用 TradeLock |
| SellGuard 退化 | SellGuard 模块保留完整，仅 HandleSell 不再调用；未来可回滚 |

## 6. 数据流示意

```
买入:
  Client HandleBMResult → AddConsumable → TradeLock.ApplyLock(batchId)
  Server HandleBuy      → AddToBackpack → TradeLock.ApplyLock(batchId)

操作闸:
  Client 各操作入口 → TradeLock.IsOperationBlocked → blocked? return

卖出:
  Client UI → CountUnlockedConsumable → unlocked < 1 ? block : confirm
  Server HandleSell §4.5 → CountUnlockedItem → unlockedHeld < amount ? reject

存档:
  SaveSerializer → 写入 bmLockUntil/bmLockSource/bmLockBatchId
  game_saved → ClearAllOnSaveSuccess → 清除全部锁

排序:
  SortBackpack → mergeGroupKey = consumableId|unlocked 或 consumableId|locked|batchId
```
