## 3 层门最终职责表

| 层级 | 名称 | 检查位置 | 职责 | 粒度 | 提示文案 |
|------|------|---------|------|------|---------|
| **L1** | 同类锁门 | 客户端: `TradeLock.HasAnyLockedConsumable`<br>服务端: `BackpackUtils.HasAnyLockedItem` | 同一 `itemId` 存在任意活动锁堆 → **整类禁售** | itemId 级别 | "该商品处于黑市交易保护期，请稍后再试" |
| **L2** | 全局未同步门 | 客户端: `BlackMarketSyncState.IsBlocked()` | `_dirtyConsume` / `_dirtyWarehouse` / `SaveSession.IsDirty()` 任一命中 → **阻断所有卖出** | 全局 | "背包数据未同步，请稍后再试" |
| **L3** | 服务端最终门 | 服务端: `HandleSell` 内 `held < amount` | L1 通过后，按总持有量判断是否足够 | itemId 级别 | "持有量不足" |

**检查顺序**：L1 → L2 → L3（锁门优先于未同步门）

## 根因分析："之前为什么会表现成全部不能卖"

### 原有问题拆解

1. **UI 优先级倒置**：客户端确认弹窗 onClick 先检查 L2（全局未同步），再检查 L1（同类锁）。由于 `_dirtyConsume` 在任何消耗品操作后都为 `true`，导致：
   - 玩家打开黑市时，全局未同步门已触发
   - 所有商品点击卖出后统一显示"背包数据未同步"
   - 真正因锁被禁售的商品，其保护期提示被 L2 覆盖

2. **服务端判定不一致**：服务端 HandleSell 使用 `CountUnlockedItem()` 判断，允许卖出同一 `itemId` 中未锁定的堆叠。而客户端产品规则是"整类禁售"（`HasAnyLockedConsumable`）。这导致：
   - 客户端显示不可卖
   - 如果绕过前端直接发请求，服务端可能放行

3. **脏标记范围过宽**：`_dirtyConsume` 对所有消耗品操作一视同仁，无论该消耗品是否在黑市商品列表中。L2 门一旦触发就阻断所有卖出，不区分具体 itemId。

### S4C 修复

| 问题 | 修复 |
|------|------|
| L2 覆盖 L1 提示 | 调换检查顺序：L1 先于 L2 |
| 服务端允许部分卖出 | `CountUnlockedItem` → `HasAnyLockedItem`（整类禁售） |
| 持有量校验复杂化 | 消耗品过 L1 锁门后，直接用 `held < amount` 简单判断 |

## 具体代码变更

### 1. BackpackUtils.HasAnyLockedItem（新增）

```lua
--- BM-S4C: 判断指定消耗品是否存在任何锁定堆叠（服务端整类禁售判定）
function BackpackUtils.HasAnyLockedItem(backpack, consumableId)
    if not backpack then return false end
    for _, item in pairs(backpack) do
        if type(item) == "table"
            and item.category == "consumable"
            and item.consumableId == consumableId
            and TradeLock.IsLockedServerSide(item) then
            return true
        end
    end
    return false
end
```

### 2. HandleSell 锁校验（修改）

**旧代码**（允许部分卖出）：
```lua
local unlockedHeld = BackpackUtils.CountUnlockedItem(backpack, consumableId)
if unlockedHeld < amount then ...
```

**新代码**（整类禁售）：
```lua
if not isEquip and BackpackUtils.HasAnyLockedItem(backpack, consumableId) then
    -- 整类拒绝
    SendError(connection, ..., "该商品处于黑市交易保护期，请稍后再试")
    return
end
if held < amount then
    SendError(connection, ..., "持有量不足")
    return
end
```

### 3. 客户端确认弹窗路径（修改）

**旧顺序**：L2 (SyncState) → L1 (CountUnlockedConsumable)
**新顺序**：L1 (HasAnyLockedConsumable) → L2 (SyncState)
