## 一、属性公式总览

### 1.1 主属性公式

```
主属性值 = 基础值(BASE) × 阶级倍率(TIER_MULT) × 品质系数(qualityMult)
```

- **百分比属性**（critRate/speed/dmgReduce）使用独立倍率表 `PCT_TIER_MULTIPLIER`
- **普通属性**（atk/def/maxHp/hpRegen 等）使用 `TIER_MULTIPLIER`

### 1.2 副属性公式

```
副属性值 = 基础值(baseValue) × 副属性阶级倍率(SUB_TIER_MULT) × 随机波动(rand)
```

- **百分比副属性**（critRate/critDmg）使用 `PCT_SUB_TIER_MULT`
- **普通副属性**使用 `SUB_STAT_TIER_MULT`
- **线性成长副属性**（killGold）：`value = tier × qualityMult`，无随机波动
- 随机波动区间受品质影响（见 1.5 节）

|------|------|-----------|-----------|---------|-------------|

| green 良品 | 2 | 1.1 | 1 | — | 0.8 ~ 1.2 |
| blue 精品 | 3 | 1.2 | 2 | — | 0.8 ~ 1.2 |
| purple 极品 | 4 | 1.3 | 3 | — | 0.8 ~ 1.2 |
| orange 稀世 | 5 | 1.5 | 3 | — | 0.9 ~ 1.3 |
| cyan 灵器 | 6 | 1.8 | 3 | ×1（半值）| 1.0 ~ 1.4 |
| red 圣器 | 7 | 2.1 | 3 | ×1（半值）| 1.0 ~ 1.4 |
| gold 仙器 | 8 | 2.5 | 3 | ×1（半值）| 1.1 ~ 1.5 |
| rainbow 神器 | 9 | 3.0 | 3 | ×1（半值）| 1.2 ~ 1.6 |

### 4.2 品质波动偏移规则

副属性随机波动区间宽度固定为 0.4，但下限随品质提升：

```
shift = max(0, qualityOrder - 4) × 0.1
波动区间 = [0.8 + shift, 1.2 + shift]
```

- 白/绿/蓝/紫（序号 1~4）：shift = 0 → 0.8 ~ 1.2
- 橙色（序号 5）：shift = 0.1 → 0.9 ~ 1.3
- 灵器（序号 6）：shift = 0.2 → 1.0 ~ 1.4
- 圣器（序号 7）：shift = 0.3 → 1.0 ~ 1.4（注：代码中序号偏移按 QUALITY_ORDER 计算）
- 仙器（序号 8）：shift = 0.4 → 1.1 ~ 1.5
- 神器（序号 9）：shift = 0.5 → 1.2 ~ 1.6

此偏移仅影响**随机生成**的装备，独特装备使用确定值（见第五节）。

### 4.3 副属性条数规则

- 副属性条数由品质决定：白 0 条 / 绿 1 条 / 蓝 2 条 / 紫及以上 3 条
- 副属性从可用池中随机抽取，排除与主属性同类型的属性

### 4.4 灵性属性（青·灵器及以上品质专属）

**概念**：灵器及以上品质的**随机装备**额外附带 1 条"灵性属性"——本质是一条**半值副属性**。

#### 4.4.1 生成规则

- **触发条件**：品质 ≥ cyan（灵器），仅限随机装备（独特装备不生成灵性，独特装备有 specialEffect）
- **数量**：固定 1 条
- **属性池**：从 `SUB_STATS` 全池中随机抽取，**仅排除主属性**
- **数值公式**：与副属性公式相同，但最终值 **×0.5**（半值）

```
灵性属性值 = 副属性值 × 0.5
         = baseValue × SUB_TIER_MULT × rand × 0.5（普通属性）
         = baseValue × PCT_SUB_TIER_MULT × rand × 0.5（百分比属性）
         = floor(tier × qualityMult) × 0.5（线性成长属性，如福缘/悟性）
```

#### 4.4.2 四属性互斥/重复规则

| | 主属性 | 副属性×3 | 洗练×1 | 灵性×1 |
|--|--------|---------|--------|--------|
| **主属性** | — | 互斥 | 互斥 | 互斥 |
| **副属性** | 互斥 | 互斥（彼此不重复） | 互斥 | ✅ 可重复 |
| **洗练** | 互斥 | 互斥 | 替换（重洗） | ✅ 可重复 |
| **灵性** | 互斥 | ✅ 可重复 | ✅ 可重复 | — |

**核心总结**：灵性属性**仅排斥主属性**，与副属性和洗练都**允许重复**。

#### 4.4.3 数据结构

```lua
item.spiritStat = {
    stat = "atk",       -- 属性类型
    name = "攻击力",    -- 显示名
    value = 3.75,       -- 半值数值
}
```

与 `forgeStat` 结构相同，存档中新增此字段。

#### 4.4.4 示例

T9 青·灵器 武器（合体剑）：
- 主属性：atk = 5 × 17.0 × 1.8 = 153
- 副属性①：def = 1.2 × 11.0 × rand = 13.2（中值）
- 副属性②：maxHp = 6 × 11.0 × rand = 66（中值）
- 副属性③：critRate = 0.01 × 6.0 × rand = 0.06（中值）
- **灵性**：atk = 1.5 × 11.0 × rand × **0.5** ≈ 8.25（与副属性 atk 无关，可重复）

洗练后新增 def → 此时 def 出现在副属性和洗练各一次，灵性 atk 与它们独立共存。

## 五、独特装备设计规范

### 5.1 核心原则

1. **属性确定性**：独特装备所有属性为确定值，无随机波动
2. **符合品质规范**：属性值严格按公式计算，不额外增加倍率（无 BOSS 溢价）
3. **副属性取中值**：紫色品质中值系数 = 1.0，橙色品质中值系数 = 1.1
4. **副属性条数遵循品质规则**：绿色 1 条，紫色 3 条，橙色 3 条
5. **特殊效果是核心优势**：独特装备的差异化来自技能效果和精选副属性组合，而非数值溢出

### 5.2 独特装备主属性计算

```
主属性 = BASE_MAIN_STAT × TIER_MULT × qualityMult
```

百分比属性用 PCT_TIER_MULTIPLIER。

### 5.3 独特装备副属性计算

```
紫色品质: 副属性 = baseValue × SUB_TIER_MULT × 1.0（中值）
橙色品质: 副属性 = baseValue × SUB_TIER_MULT × 1.1（中值偏移）
```

百分比副属性用 PCT_SUB_TIER_MULT。

## 八、随机装备与独特装备对比

| 维度 | 随机装备 | 独特装备 |
|------|---------|---------|
| 主属性 | 公式值（确定） | 公式值（确定） |
| 副属性数值 | 随机波动（±20%起） | 确定值（中值） |
| 副属性种类 | 随机抽取 | 精选组合 |
| 品质系数 | 影响主属性 + 副属性条数 + 波动偏移 | 影响主属性 + 副属性条数 |
| 特殊效果 | 无 | 可附带技能/特效 |
| 获取方式 | 击杀怪物随机掉落 | BOSS 固定掉落 |

独特装备的核心优势：
1. **确定性** — 无随机波动，保底中值
2. **精选副属性** — 针对装备定位的最优属性搭配
3. **特殊效果** — 独有技能（酒葫芦治愈、虎王血珀流血）
4. **套装加成** — 虎王三件套可触发套装效果

## 九、装备特效系统

### 9.1 流血 DOT（bleed_dot）— 虎王血珀

| 参数 | 值 | 说明 |
|------|-----|------|
| triggerChance | 35% | 每次普攻触发概率 |
| cooldown | 3s | 触发后进入内置 CD，CD 期间不可再触发 |
| damagePercent | 8%/s | 每秒造成玩家总攻击力 8% 的伤害 |
| duration | 3s | 持续 3 秒，总计 24% 攻击力 |
| 防御减免 | 无 | DOT 伤害无视目标防御 |

**刷新逻辑**：
- CD 期间：不可触发（即使命中也跳过判定）
- CD 结束后：每次攻击按 35% 概率判定
- 已有流血的目标再次触发：**重置持续时间**，更新伤害快照（取当前 ATK），不叠加层数
- 目标死亡：立即清除该目标的 DOT

**触发周期分析**（攻速 1次/s）：
- 平均触发周期 = CD(3s) + 等待触发(1/0.35 ≈ 2.86s) ≈ 5.86s
- 但 duration(3s) 内的攻击也可能在 CD 结束后刷新，实际 uptime 更高

**T4 阶段 DPS 占比验算**（ATK≈80, 怪物 DEF≈22）：
- 普攻 DPS ≈ 65（含重击、暴击期望）
- 流血单次总伤 = 80 × 8% × 3 = 19.2
- 流血 DPS ≈ 19.2 / 5.86 ≈ 3.3
- 占比 ≈ 3.3 / (65 + 3.3) ≈ **4.8%**
- 考虑刷新 uptime 提升，实际约 **5~7%**

| 3 | critDmg 副属性 ×0.5 | 普通装备副属性 + 洗练 | `value *= 0.5` |
| 4 | 独特装备覆写 | `isSpecial=true` 的装备 | 从 SpecialEquipment 模板覆写 mainStat + subStats |
| 5 | 法宝主属性查表 | `slot=="treasure"` | `mainStat.hpRegen = GOURD_MAIN_STAT[tier]` |

**处理顺序**：独特装备覆写 → 洗练修正 → 法宝主属性查表覆写（确保升级后的葫芦用正确 tier 值）

**安全措施**：
- 独特装备走覆写逻辑，跳过乘法，避免双重处理
- 升级后葫芦的副属性来自 GOURD_UPGRADE（maxHp/def/killHeal），不含受影响属性，无需修正
- 所有乘法结果保留2位小数，保底 0.01，不产生负值或零值
