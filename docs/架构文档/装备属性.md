## 一、属性公式总览

### 1.1 主属性公式

```
主属性值 = 基础值(BASE) × 阶级倍率(TIER_MULT) × 品质系数(qualityMult)
```

- **百分比属性**（critRate/speed/dmgReduce）使用独立倍率表 `PCT_TIER_MULTIPLIER`
- **普通属性**（atk/def/maxHp/hpRegen 等）使用 `TIER_MULTIPLIER`

### 1.2 副属性公式

```
副属性值 = 基础值(baseValue) × 副属性阶级倍率(SUB_TIER_MULT) × 随机波动(rand)
```

- **百分比副属性**（critRate/critDmg）使用 `PCT_SUB_TIER_MULT`
- **普通副属性**使用 `SUB_STAT_TIER_MULT`
- **线性成长副属性**（fortune/wisdom/constitution/physique）：`value = floor(tier × qualityMult)`，无随机波动
- 随机波动区间受品质影响（见第四节）

## 四、品质系统

### 4.1 品质总表

| 品质 | 序号 | 主属性系数 | 副属性条数 | 随机装备额外属性 | 随机波动区间 |
|------|------|-----------|-----------|------------------|-------------|
| white 普通 | 1 | 1.0 | 0 | — | 0.8 ~ 1.2 |
| green 良品 | 2 | 1.1 | 1 | — | 0.8 ~ 1.2 |
| blue 精品 | 3 | 1.2 | 2 | — | 0.8 ~ 1.2 |
| purple 极品 | 4 | 1.3 | 3 | — | 0.8 ~ 1.2 |
| orange 稀世 | 5 | 1.5 | 3 | — | 0.9 ~ 1.3 |
| cyan 灵器 | 6 | 1.8 | 3 | 灵性 ×1（半值） | 1.0 ~ 1.4 |
| red 圣器 | 7 | 2.1 | 3 | 随机路径仍按灵性；圣器打造使用圣性 | 1.1 ~ 1.5 |
| gold 仙器 | 8 | 2.5 | 3 | 随机路径仍按灵性 | 1.2 ~ 1.6 |
| rainbow 神器 | 9 | 3.0 | 3 | 随机路径仍按灵性 | 1.3 ~ 1.7 |

说明：

- `GameConfig.QUALITY` 定义了 white 到 rainbow 全品质。
- 当前普通随机装备权重只投放到 cyan；red/gold/rainbow 主要由特殊装备、打造或后续系统控制。
- `GenerateRandomEquipment` 的额外属性触发条件是 `qualityOrder >= 6`，生成 `spiritStat`。
- `CreateSaintEquipment` / `CreateJiefengSword` 会清除 `spiritStat` 并生成或保留 `saintStat`。

### 4.2 品质波动偏移规则

副属性随机波动区间宽度固定为 0.4，下限随品质提升：

```text
shift = max(0, qualityOrder - 4) × 0.1
波动区间 = [0.8 + shift, 1.2 + shift]
```

| 品质 | qualityOrder | shift | 波动区间 |
|------|-------------:|------:|----------|
| white/green/blue/purple | 1-4 | 0.0 | 0.8 ~ 1.2 |
| orange | 5 | 0.1 | 0.9 ~ 1.3 |
| cyan | 6 | 0.2 | 1.0 ~ 1.4 |
| red | 7 | 0.3 | 1.1 ~ 1.5 |
| gold | 8 | 0.4 | 1.2 ~ 1.6 |
| rainbow | 9 | 0.5 | 1.3 ~ 1.7 |

此偏移影响随机生成的普通副属性、灵性属性、圣性属性；固定属性（如天诛）不参与波动。

### 4.3 副属性条数规则

- 副属性条数由品质决定：白 0 条 / 绿 1 条 / 蓝 2 条 / 紫及以上 3 条。
- 副属性从 `EquipmentData.SUB_STATS` 中随机抽取，排除与主属性同类型的属性。
- 当前标准副属性池为 12 种：`atk / def / maxHp / hpRegen / killHeal / critRate / critDmg / heavyHit / fortune / wisdom / constitution / physique`。
- `fortune / wisdom / constitution / physique` 为线性成长属性，公式为 `floor(tier × qualityMult)`，不乘副属性倍率，不参与随机波动。

### 4.4 灵性属性（cyan 灵器及以上随机路径）

**概念**：灵性属性是 `spiritStat`，本质是一条半值副属性，并可从扩展池中抽到天诛属性。

#### 4.4.1 生成规则

- **触发条件**：`qualityOrder >= 6` 的随机装备、套装灵器、法宝动态掉落等随机路径。
- **数量**：固定 1 条。
- **属性池**：`EquipmentData.SPIRIT_STATS`，等于标准副属性池 + `tianzhuChance / tianzhuDamage`。
- **排除规则**：仅排除主属性类型；与副属性、洗练属性允许重复。
- **独特装备**：以模板为准。模板可直接配置 `spiritStat`，也可通过 `hasSpiritStat` 走生成器补一条灵性。

#### 4.4.2 数值公式

```text
普通灵性 = baseValue × SUB_STAT_TIER_MULT[tier] × rand × 0.5
百分比灵性 = baseValue × PCT_SUB_TIER_MULT[tier] × rand × 0.5
线性灵性 = floor(tier × qualityMult) × 0.5
固定灵性 = baseValue
```

`tianzhuChance / tianzhuDamage` 标记为 `fixed=true`，因此完全固定，不乘 tier、不乘品质、不乘随机波动。

#### 4.4.3 四属性互斥/重复规则

| | 主属性 | 副属性×3 | 洗练×1 | 灵性×1 |
|--|--------|---------|--------|--------|
| **主属性** | — | 互斥 | 互斥 | 互斥 |
| **副属性** | 互斥 | 互斥（彼此不重复） | 互斥 | 可重复 |
| **洗练** | 互斥 | 互斥 | 替换（重洗） | 可重复 |
| **灵性** | 互斥 | 可重复 | 可重复 | — |

#### 4.4.4 数据结构

```lua
item.spiritStat = {
    stat = "atk",
    name = "攻击力",
    value = 8.25,
}
```

### 4.5 圣性属性（red 圣器打造路径）

**概念**：圣性属性是 `saintStat`，当前用于圣器打造、解封古剑和第五章圣器等路径。它与灵性互斥，生成圣性时会清除 `spiritStat`。

#### 4.5.1 生成规则

- **生成入口**：`LootSystem.CreateSaintEquipment`、`LootSystem.CreateJiefengSword`。
- **属性池**：`EquipmentData.SAINT_STATS`，等于标准副属性池 + 更高档天诛属性。
- **排除规则**：仅排除主属性类型。
- **固定圣性**：若模板已配置 `saintStat`，直接保留模板值。
- **随机圣性**：若模板未配置 `saintStat`，按 `GenerateSaintStat` 随机生成。

#### 4.5.2 数值公式

```text
普通圣性 = baseValue × SUB_STAT_TIER_MULT[tier] × rand
百分比圣性 = baseValue × PCT_SUB_TIER_MULT[tier] × rand
线性圣性 = floor(tier × qualityMult)
固定圣性 = baseValue
```

天诛固定值：

| 属性池 | 天诛概率 | 天诛伤害 |
|--------|---------:|---------:|
| 灵性 SPIRIT_STATS | 1% | 5% |
| 圣性 SAINT_STATS | 2% | 10% |

#### 4.5.3 数据结构

```lua
item.saintStat = {
    stat = "fortune",
    name = "福缘",
    value = 21,
}
```

## 五、独特装备设计规范

### 5.1 核心原则

1. **模板确定性**：装备 ID、槽位、品质、Tier、主属性、特效、套装关系由模板确定
2. **主属性确定性**：主属性严格按 `BASE_MAIN_STAT × 对应Tier倍率 × qualityMult` 计算或由模板固定
3. **非线性副属性可波动**：`atk/def/maxHp/hpRegen/killHeal/critRate/critDmg/heavyHit` 等副属性在创建时按 `SPECIAL_FLUCTUATION` 重算
4. **线性副属性保持模板值**：`fortune/wisdom/constitution/physique` 等线性属性通常由模板直接给定，不走随机波动
5. **特殊效果是核心优势**：独特装备的差异化来自技能效果和精选副属性组合，而非数值溢出
6. **额外属性以模板为准**：独特装备可在模板中固定 `spiritStat` / `saintStat`，或通过 `hasSpiritStat` 触发生成器补一条灵性；圣器打造路径会清除灵性并生成/保留圣性。

### 5.2 独特装备主属性计算

```
主属性 = BASE_MAIN_STAT × TIER_MULT × qualityMult
```

百分比属性用 PCT_TIER_MULTIPLIER。

### 5.3 独特装备副属性计算

当前代码通过 `loot/shared.lua:SPECIAL_FLUCTUATION` 控制特殊装备非线性副属性波动：

| 品质 | 波动起点 | 波动范围 | 实际区间 |
|------|---------:|---------:|----------|
| purple | 1.0 | 0.2 | 1.0 ~ 1.2 |
| orange | 1.1 | 0.2 | 1.1 ~ 1.3 |
| cyan | 1.1 | 0.3 | 1.1 ~ 1.4 |
| red | 1.2 | 0.3 | 1.2 ~ 1.5 |
| gold | 1.2 | 0.4 | 1.2 ~ 1.6 |
| rainbow | 1.3 | 0.4 | 1.3 ~ 1.7 |

```text
普通副属性 = baseValue × SUB_STAT_TIER_MULT[tier] × specialRand
百分比副属性 = baseValue × PCT_SUB_TIER_MULT[tier] × specialRand
线性副属性 = 模板值
```

百分比副属性用 PCT_SUB_TIER_MULT。

## 八、随机装备与独特装备对比

| 维度 | 随机装备 | 独特装备 |
|------|---------|---------|
| 主属性 | 公式值（确定） | 公式值（确定） |
| 副属性数值 | 按品质波动区间随机 | 非线性副属性按特殊波动区间随机；线性副属性按模板 |
| 副属性种类 | 随机抽取 | 精选组合 |
| 品质系数 | 影响主属性 + 副属性条数 + 波动偏移 | 影响主属性 + 特殊波动区间 + 线性属性模板值 |
| 特殊效果 | 无 | 可附带技能/特效 |
| 获取方式 | 击杀怪物随机掉落 | BOSS 固定掉落 |

独特装备的核心优势：
1. **模板确定性** — 装备身份、主属性、特效和关键线性属性稳定
2. **精选副属性** — 针对装备定位的最优属性搭配
3. **特殊效果** — 独有技能（酒葫芦治愈、虎王血珀流血）
4. **套装加成** — 虎王三件套可触发套装效果

## 九、装备特效系统

### 9.1 流血 DOT（bleed_dot）— 虎王血珀

| 参数 | 值 | 说明 |
|------|-----|------|
| triggerChance | 35% | 每次普攻触发概率 |
| cooldown | 3s | 触发后进入内置 CD，CD 期间不可再触发 |
| damagePercent | 8%/s | 每秒造成玩家总攻击力 8% 的伤害 |
| duration | 3s | 持续 3 秒，总计 24% 攻击力 |
| 防御减免 | 无 | DOT 伤害无视目标防御 |

**刷新逻辑**：
- CD 期间：不可触发（即使命中也跳过判定）
- CD 结束后：每次攻击按 35% 概率判定
- 已有流血的目标再次触发：**重置持续时间**，更新伤害快照（取当前 ATK），不叠加层数
- 目标死亡：立即清除该目标的 DOT

**触发周期分析**（攻速 1次/s）：
- 平均触发周期 = CD(3s) + 等待触发(1/0.35 ≈ 2.86s) ≈ 5.86s
- 但 duration(3s) 内的攻击也可能在 CD 结束后刷新，实际 uptime 更高

**T4 阶段 DPS 占比验算**（ATK≈80, 怪物 DEF≈22）：
- 普攻 DPS ≈ 65（含重击、暴击期望）
- 流血单次总伤 = 80 × 8% × 3 = 19.2
- 流血 DPS ≈ 19.2 / 5.86 ≈ 3.3
- 占比 ≈ 3.3 / (65 + 3.3) ≈ **4.8%**
- 考虑刷新 uptime 提升，实际约 **5~7%**

## 十一、天诛属性系统

### 11.1 概述

**天诛**是暴击后的二次判定属性。只有在成功暴击后，才会按天诛概率判定是否触发天诛，触发后对暴击伤害进行额外倍率加成。

**触发流程**：
```
普通攻击 → 暴击判定(critRate) → [暴击成功] → 天诛判定(tianzhuChance) → [天诛触发] → 天诛伤害加成
```

### 11.2 伤害公式

```
天诛伤害 = 暴击伤害 × 天诛伤害倍率
天诛伤害倍率 = 1.5(基础) + 装备天诛伤害加成
```

- **基础倍率**：1.5（即 150%），无任何装备时天诛触发即为暴击伤害 ×1.5
- **装备加成**：每件装备的天诛伤害属性直接累加到基础倍率上

### 11.3 属性来源

天诛属性**仅出现在灵性/圣性属性中**，不在标准副属性池（SUB_STATS）中：

| 属性池 | 天诛概率 | 天诛伤害 |
|--------|---------|---------|
| SPIRIT_STATS（灵性） | 1% (0.01) | 5% (0.05) |
| SAINT_STATS（圣性） | 2% (0.02) | 10% (0.10) |

### 11.4 完全固定数值（无任何波动）

**天诛属性标记为 `fixed = true`**，生成时直接使用 baseValue，无任何随机波动、无 T 级缩放。

```
天诛属性值 = baseValue（固定值，不乘任何系数）
```

- 不乘以 SUB_STAT_TIER_MULT 或 PCT_SUB_TIER_MULT
- 不受品质浮动影响（无 randBase 随机）
- T1 到 T20 任何等级装备的天诛值完全相同
- 设计意图：天诛是稀有的额外收益，数值绝对固定，不随装备等级或品质波动

**固定数值**：

| 属性池 | 天诛概率 | 天诛伤害 |
|--------|---------|---------|
| 灵性 | 1% | 5% |
| 圣性 | 2% | 10% |

### 11.5 角色面板显示

角色面板中天诛显示**总值**（非装备增量）：

- **天诛概率** = `player:GetTotalTianzhuChance() × 100`%（无装备时为 0%）
- **天诛伤害** = `player:GetTotalTianzhuDmg() × 100`%（无装备时为 150%，含基础倍率）

### 11.6 UI 颜色

天诛属性使用**灵器青色** `{0, 220, 220, 255}`，与灵器品质颜色一致。

### 11.7 战斗浮字

天诛触发时显示特殊浮字：
- 前缀 "天诛"
- 字体大小 24（比暴击 16 更大更醒目）
- 颜色：青色 `nvgRGBA(0, 220, 220, alpha)`
