# 角色帧动画制作规范

> 为其他角色（太虚、灵宠等）制作动画帧时的参考文档。

## 一、当前动作配置总览

| 动作 | 帧数 | FPS | 循环时长 | 循环 |
|------|------|-----|---------|------|
| idle（待机） | 1 | 1 | — | 是（单帧静止） |

## 二、帧序列定义

### 行走 (move) — 4帧

| 帧 | 描述 | 要点 |
|----|------|------|
| move_1 | 左脚迈出 | 步幅适中，重心略前移 |
| move_2 | 过渡收拢 | 双腿并拢，身体直立 |
| move_3 | 右脚迈出 | 与帧1镜像对称 |
| move_4 | 回程过渡 | 双腿收拢回位 |

### 攻击 (attack) — 4帧

| 帧 | 描述 | 要点 |
|----|------|------|
| atk_1 | 举刀蓄力 | 刀举在身前（眼前），不过头顶 |
| atk_2 | 挥砍中段 | 刀横于腰部高度，水平朝前 |
| atk_3 | 下劈到位 | 刀到身体前方下侧，斜向 |
| atk_4 | 收刀回位 | 恢复持刀站姿 |

## 三、精灵制作规范

### 画面规格
- **尺寸**: 256×256 像素
- **风格**: 像素风 Q版，大头小身
- **视角**: **微侧 3/4 视角**（必须能看到双眼，不是纯正面也不是平侧面）
- **锚定**: 以角色原始待机精灵 (`player_right.png`) 为视角/比例参考

### 绘制红线（必须遵守）
1. **头部完整** — 不可裁切头顶，角色整体居中于画布
2. **双眼可见** — 微侧3/4视角，双眼都能看到
3. **武器尺寸一致** — 所有帧的武器大小严格对齐原始精灵
4. **武器不过头顶** — 攻击动作幅度克制，刀在身前活动
5. **无特效/残影** — 纯净角色帧，不加刀光、残影、粒子
6. **背景透明** — 输出PNG带透明通道

### 出图后强制比对流程（每帧必做）

**每次生成资源后，必须逐帧与原始精灵图严格比对，全部通过后才能放入正式目录。**

| # | 检查项 | 判定标准 | 不合格处理 |
|---|--------|---------|-----------|
| 1 | **图片完整性** | 头部、武器、四肢均完整可见，无裁切、无消失 | 重新生成 |
| 2 | **背景透明** | 纯透明背景，不能出现任何有色背景/色块/渐变底 | 重新生成 |
| 3 | **微侧脸（3/4视角）** | 能清晰看到两只眼睛；不是正侧面（只见一眼）也不是纯正面 | 重新生成 |
| 4 | **武器大小一致** | 武器长度/宽度与原始精灵图严格一致 | 重新生成 |
| 5 | **整体风格一致** | 配色、身材比例、像素密度与原始精灵图一致 | 重新生成 |

**比对方法**: 打开原始精灵图（如 `class_swordsman_right.png`），逐帧目视对比上述5项。任一项不通过即判定不合格，必须重新生成。

### AI生图提示词模板

```
像素风Q版{角色名}，256x256，透明背景，
微侧3/4视角双眼可见，{动作描述}，
角色完整居中不裁切头部完整可见，
{武器}清晰可见，与参考图大小一致
```

## 四、程序化动画辅助参数

以下参数在 `PlayerAnimator.lua` 中统一管理，所有角色共享同一套效果。

### 待机 (idle)

| 参数 | 值 | 说明 |
|------|---|------|
| offsetY | `sin(t * 2.0) * 1.5` | 呼吸起伏，上下1.5px |
| scaleY | `1.0 + sin(t * 2.0) * 0.01` | 纵向微膨胀（1%） |
| scaleX | 1.0 | 无 |
| rotation | 0 | 无 |

### 行走 (move)

| 参数 | 值 | 说明 |
|------|---|------|
| offsetY | `-bounce * 0.4` | 向上弹跳0.4px（极轻微） |
| rotation | `sin(phase) * 0.006` | 身体微倾±0.3° |
| scaleX/Y | 1.0 | 无缩放 |

> bounce = `abs(sin(bouncePhase * π))`，bouncePhase 跟帧进度同步

### 攻击 (attack) — 逐帧配置

| 帧 | scaleX | scaleY | offsetY | rotation | 效果 |
|----|--------|--------|---------|----------|------|
| 1 | 0.92 | 1.06 | +1.0 | 0 | 蓄力压缩(squash) |
| 2 | 1.08 | 0.95 | -1.5 | -0.04 | 挥砍拉伸(stretch)+前倾 |
| 3 | 1.05 | 0.97 | -1.0 | -0.02 | 到位保持 |
| 4 | 回弹sin | 回弹sin | 0 | 0 | 收刀过冲回弹 |

### 变换应用方式
- **锚点**: 角色脚底中心 (`pivotY = sy + halfSize * 0.4`)
- **变换顺序**: Translate(pivot) → Rotate → Scale → Translate(-pivot)
- **offsetY**: 在最终绘制坐标 `imgY` 上直接加减

## 六、新角色接入步骤

1. **准备原始精灵** — `player_right.png` 等价物，确定角色外观/武器/视角
2. **生成帧图** — 按上述规范用AI生图，每个动作4帧，idle可仅1帧
3. **身体对齐** — 对所有帧执行身体中心X对齐（见第八节），确保偏差≤±8px
4. **放置文件** — 按目录结构放入 `assets/Textures/anim/{class}/`
5. **注册配置** — 在 `PlayerAnimator.lua` 的 `ANIM_DATA` 中添加对应角色数据块
6. **测试** — 程序化辅助效果自动生效，无需额外配置

### ANIM_DATA 配置模板

```lua
taixu = {
    idle = {
        frames = { "Textures/anim/taixu/idle_1.png" },
        fps = 1,
    },
    move = {
        frames = {
            "Textures/anim/taixu/move_1.png",
            "Textures/anim/taixu/move_2.png",
            "Textures/anim/taixu/move_3.png",
            "Textures/anim/taixu/move_4.png",
        },
        fps = 6,
    },
    attack = {
        frames = {
            "Textures/anim/taixu/atk_1.png",
            "Textures/anim/taixu/atk_2.png",
            "Textures/anim/taixu/atk_3.png",
            "Textures/anim/taixu/atk_4.png",
        },
        fps = 10,
        loop = false,
    },
},
```

## 七、已踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| 头部被裁切 | AI生图时角色未居中 | 提示词加"角色完整居中不裁切头部完整可见" |
| 双眼不可见/平侧面 | 视角描述不够精确 | 强调"微侧3/4视角双眼可见"，附参考图 |
| 武器消失 | 动作幅度大时AI遗漏 | 提示词加"{武器}清晰可见" |
| 武器大小不一致 | 未锚定参考 | 附原始精灵作为 reference_image |
| 修size时方位变了 | 只说了改大小没锁方位 | 加"与参考图方位完全一致" |
| 行走抖动过大 | 弹跳/旋转参数过高 | offsetY≤0.4px, rotation≤0.006rad |
| 动画切换时角色水平跳动 | 各帧身体X中心不一致 | 执行身体对齐流程（见第八节） |
| 对齐平移后武器尖端被裁 | 平移量大于边缘空间 | 扩展画布→复制末段纹理→缩放回256px |

## 八、身体对齐规范（Body Alignment）

### 8.1 为什么需要对齐

`PlayerAnimator` 按帧中心点渲染精灵（`imgX = sx - halfSize`）。如果各帧中角色身体的X位置不一致，切帧时角色会产生水平"抖动/跳跃"。

**容差**: 所有帧的身体中心X偏差不超过 **±8px**（相对于 idle_1 基线），超出此范围人眼可感知。

### 8.2 身体中心检测算法

```
输入: 256×256 RGBA 帧图像
输出: body_cx (整数，角色躯干在X轴上的重心)

步骤:
1. 计算内容包围盒 (content_top, content_bottom)
2. 取躯干区域: y ∈ [content_top + 35%*height, content_top + 75%*height]
   — 排除头顶装饰和腿部大幅运动干扰
3. 统计躯干区域每列(x)的不透明像素数 density[x]
4. 取 max_density = max(density[])
5. 过滤: 只保留 density[x] >= max_density * 0.3 的列
6. 连续性分组: 遍历保留列，gap > 5px 视为不同物体（分离武器/特效）
7. 取最大连续组（像素总量最多的组）= 角色躯干
8. body_cx = 该组内各列的加权平均 (权重=density[x])
```

### 8.3 对齐流程

```
1. 对角色全部帧执行身体中心检测
2. 选择基线: idle_1 的 body_cx 作为 baseline
3. 计算偏移: offset = frame.body_cx - baseline
4. 若 |offset| > 8: 需要平移修正
5. 平移修正: 将帧内容水平平移 -offset 像素
   - 若平移后无内容溢出(0~255范围): 直接平移
   - 若平移后内容超出画布边缘: 执行裁切修复（见 8.4）
6. 验证: 重新检测 body_cx，确认偏差 ≤ ±8px
```

### 8.4 边缘裁切修复（武器延伸过长时）

当帧中武器向右延伸较远，对齐平移后右端被裁切：

```
修复策略（等比缩放法）:
1. 扩展画布宽度（加上被裁切量 + 渐变收尖量）
2. 在扩展区域复制武器末段纹理（保持连续性）
3. 添加渐变收尖: 最后 2~3 列逐步收窄像素高度
4. 将全部内容等比缩放（NEAREST 插值）回 256px 宽度
5. 确保 body_cx 仍在目标位置

注意:
- 缩放比例建议 ≥ 0.9（<0.9 会明显模糊，需重新生图）
- 收尖模式: 从武器高度 h 逐步减少到 h*0.6 → h*0.3 → 0
- 垂直居中收尖，使尖端自然消失
```

### 8.5 各角色对齐记录

| 角色 | baseline (idle_1) | 最大偏差 | 特殊处理 |
|------|-------------------|---------|---------|
| 罗汉 (monk) | body_cx = 126 | ±1px | atk_1 平移-15px，atk_2 平移-32px |
| 太虚 (taixu) | body_cx = 118 | ±7px | atk_2 缩放97.2%，atk_3 缩放90.7%（武器超长） |
