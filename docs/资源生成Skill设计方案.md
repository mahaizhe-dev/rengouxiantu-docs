# 资源生成 Skill 设计方案

> 目标：把《人狗仙途》的资源生成约束固化为 skill 内置规则，让 AI 在生成瓦片、贴图、角色图、装备图、怪物头像、BOSS 全身图、特效资源和地图大型物件时自动遵守统一规范，而不是每次都重新强调尺寸、透明底、风格、视角。
>
> 依据：
> - `docs/架构文档/rules.md` 第 2 节与第 4.3 节
> - `docs/架构文档/npc-portrait-spec.md`
> - 本地可见精灵基线：
>   - `assets/Textures/player_right.png`
>   - `assets/Textures/class_swordsman_right.png`
>   - `assets/Textures/class_body_cultivator_right.png`
>   - `assets/Textures/class_formation_master_right.png`

---

## 1. Skill 定位

这个 skill 的职责不是“随便写提示词”，而是：

1. 先判断用户要的是哪一类资源。
2. 自动套用该类资源的默认规格。
3. 只把“本次变化项”暴露给用户，例如题材、颜色、身份、材质、阵营、章节感。
4. 生成图片后先给预览，等用户确认，再决定是否接入代码或配置。

这套 skill 应当默认服务以下资源：

- 瓦片
- 装饰物贴图
- 大型物件贴图（N×N）
- 地表贴花
- 角色图 / NPC 图
- 装备图标
- 小怪展示图
- 精英头像
- BOSS 全身图
- 特效资源

---

## 2. Skill 名称建议

推荐名称：

- `rengouxiantu-asset-generator`

原因：

- 触发语义直接，看到“asset / generator”就知道是资源生成。
- 带项目名，避免和通用图片 skill 混淆。
- 符合 skill 命名规则：小写、连字符、无空格。

---

## 3. 目录结构建议

建议先做成“轻量文档型 skill”，v1 不强依赖脚本：

```text
rengouxiantu-asset-generator/
├── SKILL.md
└── references/
    ├── asset-standards.md
    ├── naming-rules.md
    ├── front-view-spec.md
    └── prompt-examples.md
```

建议拆分方式：

- `SKILL.md`
  只保留触发说明、执行流程、分类判断、默认规则总表。
- `references/asset-standards.md`
  放尺寸、透明度、风格、构图等固定规范。
- `references/front-view-spec.md`
  单独定义“正视角”的语义，避免装饰物贴图不断跑偏。
- `references/naming-rules.md`
  放文件命名规范。
- `references/prompt-examples.md`
  放各类资源的 prompt 模板和示例。

v1 不必上脚本。等后续真的频繁重复拼 prompt，再补一个 `scripts/build_prompt.*`。

## 4. 常驻默认规则

这些规则应作为 skill 的“内置常量”，默认生效，不需要每次对用户重复。

| 维度 | 默认规则 |
|------|----------|
| 世界观 | 仙侠 / 修仙题材，和《人狗仙途》现有资源一致 |
| 画风 | 精细像素 / Hi-Bit Pixel Art；角色偏 Q 版；图标偏清晰读形；不要写实厚涂 |
| 输出格式 | PNG |
| 文字元素 | 禁止文字、数字、水印、UI 边框、血条、按钮、标签、表情符号 |
| 单体性 | **每张图只画一个主体**，禁止出现副本、缩略图、群体、多视角拼图 |
| 构图 | 主体清晰、单体居中，边缘不截断，留出安全边距 |
| 一致性 | 同一批资源保持同一视角、同一风格、同一级别细节密度 |
| 响应方式 | 默认不反复解释透明背景、PNG、无文字等常规项，除非用户要求查看完整规格 |

透明度默认规则：

- 瓦片：默认不透明。
- 装备图标：默认透明背景。
- 角色 / NPC 图：默认透明背景。
- 小怪展示图 / 精英头像 / BOSS 全身图：默认透明背景。
- 装饰物贴图：默认透明背景。
- 地表贴花：默认透明背景。

尺寸白名单：

- `64x64`：装备图标
- `128x128`：瓦片
- `256x256`：角色、NPC、小怪展示图、精英头像、BOSS 全身图、装饰物贴图、特效资源
- 多格物件：按占地 `N×N` 推导，基础宽度 = `N×128`

除非用户明确说明，否则 skill 不主动发散到其他尺寸。

### 4.1 本地精灵基线观察

基于当前本地能看到的四张角色精灵图，可以把《人狗仙途》的角色像素风格进一步收紧为：

- 统一 `256x256` 透明底。
- 主体完整、单体居中，周围保留明确留白，不贴边。
- 轮廓线干净，明暗分层清楚，但不是低细节 8-bit 粗颗粒风。
- 像素密度属于“中高密度像素风”，允许较完整的五官、衣褶、武器和肌肉结构。
- 角色默认以全身站立或全身战斗姿态呈现，而不是半身插画。
- 整体更接近“手游像素角色精灵”而不是“超复古像素块面”。

这条基线主要约束角色、NPC、怪物和大型立式物件，不用于限制瓦片的平铺逻辑。

## 5. 分类规则

skill 需要先做资源分类，再选默认规格。推荐固定成下面这张表。

| 资源类型 | 默认尺寸 | 背景 | 默认视角 | 风格重点 |
|----------|----------|------|----------|----------|
| 装备图标 | 64x64 | 透明 | 物品展示视角，以读形为先 | 纯物品渲染，居中占 70%-85% |
| 角色图 / NPC 图 | 256x256 | 透明 | 正身站立展示 | Q 版像素风，全身站立，简洁干净 |
| 小怪展示图 | 256x256 | 透明 | 正面或微正侧，根据体型选择构图 | 识别种族特征，轮廓简洁 |
| 精英头像 | 256x256 | 透明 | 正面或微正侧胸像 | 比小怪更强的压迫感和特征差异 |
| BOSS 全身图 | 256x256 | 透明 | 全身正面或微正侧展示 | 完整姿态、武器和首领感 |
| 特效资源 | 256x256 | 透明 | 叠加型自由视角，按用途控方向 | 读形清楚、便于叠加、不要场景背景 |
| 瓦片 | 128x128 | 不透明 | 俯视角 | 可无缝拼接，地表逻辑明确 |
| 装饰物贴图（立式） | 256x256 | 透明 | 正视角 | RPG Maker 风格地图物件 |
| 大型物件贴图（N×N） | 按占地推导 | 透明 | 正视角或俯视角，取决于类型 | 先固定逻辑占地，再决定构图 |
| 地表贴花（decal） | 128x128 或 256x256 | 透明 | 俯视角 | 用于裂纹、阵纹、污渍、灰烬等覆盖层 |

这里要特别强调分类边界：

- “装饰物贴图”默认指立起来的地图物件，例如宝箱、石碑、丹炉、树、旗帜、路牌、祭坛、门楼、机关柱。
- “大型物件贴图”默认指有明确逻辑占地的 `N×N` 地图物件，例如大型祭坛、山门、巨树、巨像、楼阁前景、巨型宝箱、阵台。
- “特效资源”默认指战斗或场景叠加用的独立特效元素，例如刀光、爆炸、护盾、火焰、灵气、冲击波、命中特效。
- “地表贴花”默认指贴在地面的覆盖物，例如裂纹、血迹、阴影、符阵、灰烬、拖痕。

不要把这些类型混在一起，尤其不要把立式物件、平铺大型物件、特效资源和地表贴花混为一类。

## 6. 各类资源的硬规范

### 6.1 装备图标

直接继承 `rules.md` 的现有规范：

- `64x64`
- PNG
- 透明背景
- 纯物品渲染
- 主体居中，占画面 `70%-85%`
- 禁止背景、文字、UI、人物

skill 对装备图标应默认理解为“已有规则沿用”，不需要每次重新声明。

推荐 prompt 骨架：

```text
{装备中文描述}，仙侠修仙风格，像素图标风格，纯物品渲染，无文字无UI，透明背景
```

### 6.2 角色图 / NPC 图

直接兼容 `npc-portrait-spec.md`：

- `256x256`
- PNG
- 透明背景
- Q 版卡通像素风
- 全身站立
- 主体完整
- 不要半身像
- 不要场景背景

推荐 prompt 骨架：

```text
{角色服饰描述}，{角色气质/动作}，只画一个,全身站立像，Q版卡通像素风，无背景，透明底图，简洁干净
```

### 6.3 怪物资源

怪物资源必须拆成三档，不再用一个“怪物图”统包。

怪物展示图根据体型选择构图：大型人形/兽型怪物可用胸像构图，小型兽类/虫类/鸟类等必须保留整体轮廓。

#### 6.3.1 小怪展示图

- `256x256`
- PNG
- 透明背景
- 根据体型选择构图：大型人形/兽型可用胸像，小型兽类/虫类/鸟类保留全身轮廓
- 默认正面或微正侧展示
- 不做俯拍，不做强透视
- 优先突出种族识别、攻击倾向、颜色识别
- 细节密度低于精英和 BOSS，避免画得过满

推荐 prompt 骨架：

```text
{小怪描述}，像素风，仙侠怪物展示图，只画一个,单体居中,根据体型选择构图,种族特征清晰，轮廓简洁，无背景，透明底图
```

#### 6.3.2 精英头像

- `256x256`
- PNG
- 透明背景
- 头像 / 胸像构图
- 默认正面或微正侧展示
- 不做俯拍，不做强透视
- 必须比同系小怪多一层“精英识别物”，例如角、甲、纹路、面具、异火、额外眼部特征
- 允许更强的威压感，但不要膨胀成 BOSS 全身图

推荐 prompt 骨架：

```text
{精英怪描述}，像素风，仙侠精英怪头像，只画一个,单体居中,胸像构图,特征强化，威压感更强，无背景，透明底图
```

#### 6.3.3 BOSS 全身图

- `256x256`
- PNG
- 透明背景
- 全身构图
- 默认正面或微正侧展示
- 主武器、核心肢体、关键装甲或异形结构必须完整可见
- 轮廓优先于局部花纹，避免因为细节过多导致缩小后不可读
- 默认仍控制在 `256x256`，不要随意单独放大尺寸
- 只有在用户明确特别注明时，才允许例外尺寸或更高画布，用于超高、超宽或结构特别复杂的 BOSS

推荐 prompt 骨架：

```text
{BOSS描述}，像素风，仙侠BOSS全身图，只画一个,单体居中,完整站姿或战斗姿态，轮廓强烈，主武器清晰，无背景，透明底图
```

### 6.4 瓦片

瓦片严格按地图资源处理：

- `128x128`
- PNG
- 不透明
- 俯视角
- 可无缝拼接
- 不带立面正视信息
- 先保证大面积平铺稳定，再追求细节

推荐 prompt 骨架：

```text
{地表材质描述}，128x128 像素瓦片，俯视角，像素风，可无缝拼接，无文字无UI
```

### 6.5 特效资源

这类资源是独立叠加的像素特效，不是地面贴花，也不是地图物件。

硬规则：

- 默认 `256x256`
- PNG
- 透明背景
- 主体居中
- 允许半透明发光、能量边缘、烟雾过渡
- 不带场景背景，不带 UI，不带角色本体
- 优先保证缩小后仍然能一眼看出特效类型
- 特效形状要便于叠加到角色、怪物或场景节点上

适用对象：

- 刀光
- 剑气
- 爆炸
- 火焰团
- 命中闪光
- 护盾
- 灵气
- 光环
- 冲击波
- buff / debuff 环绕特效

推荐 prompt 骨架：

```text
{特效描述}，像素风特效资源，256x256，透明背景，主体居中，便于叠加使用，无背景无UI
```

### 6.6 装饰物贴图（立式）

这是本 skill 需要额外固化的重点。

硬规则：

- 默认 `256x256`
- PNG
- 透明背景
- 正视角
- RPG Maker 风格地图物件
- 主体底边落地，可直接摆在俯视地图上
- 不要俯视顶面，不要三四分之三俯角，不要照片透视感

适用对象：

- 宝箱
- 石碑
- 丹炉
- 灯笼
- 路牌
- 小祭坛
- 树木
- 枯木
- 屏风
- 小型机关柱

推荐 prompt 骨架：

```text
{物件描述}，RPG Maker 风格地图装饰物，正视角，像素风，透明背景，主体完整，底边落地，无文字无UI
```

### 6.7 地表贴花（decal）

这类资源不要误用正视角。

硬规则：

- 默认 `128x128`，复杂组合可 `256x256`
- PNG
- 透明背景
- 俯视角
- 用于覆盖在地面之上
- 边缘允许半透明过渡，但主体逻辑仍应清晰

适用对象：

- 裂纹
- 剑痕
- 血迹
- 灰烬
- 地面符阵
- 焦黑污渍

推荐 prompt 骨架：

```text
{贴花描述}，俯视角地表贴花，像素风，透明背景，用于地图覆盖层，无文字无UI
```

### 6.8 大型物件贴图（N×N）

这类资源必须先定义“逻辑占地”，再定义画布。

这里的 `N×N` 指地图上的占地格数，不是随口描述的大概尺寸。

硬规则：

- 先声明占地：`1×1`、`2×2`、`3×3`、`4×4` 等
- 地图单格基准 = `128px`
- 基础画布宽度 = `N×128`
- 默认透明背景
- 先判断物件是“立式大型物件”还是“平铺大型物件”

立式大型物件规则：

- 默认正视角
- 适用于山门、巨树、祭坛主体、巨像、楼阁前景、塔形机关等
- 画布宽度 = `N×128`
- 画布高度默认 = `N×128`
- 如需表现上方立面，允许高度额外增加 `128px`
- 额外高度只能向上延展，底边必须严格对齐占地底边
- 不允许无说明地向左右超出占地宽度
- 只有在用户明确特别注明“允许超高”时，才允许继续上探更高画布

平铺大型物件规则：

- 默认俯视角
- 适用于大型法阵、平台、深坑口、地面祭台、整块残垣地基等
- 画布尺寸 = `N×128` × `N×128`
- 应优先保证边界和拼接关系清晰

推荐 prompt 骨架：

```text
{N}x{N} 占地的大型{物件描述}，像素风，{正视角或俯视角}，透明背景，底边或边界严格对齐占地，可直接用于俯视地图摆放，无文字无UI
```

## 7. “正视角”定义

这是装饰物贴图的核心约束，必须在 skill 里写成硬说明。

### 7.1 正确定义

这里的“正视角”不是角色立绘术语，而是 **RPG Maker 式地图物件正面视角**：

- 物件正面朝向镜头。
- 镜头基本水平，不俯拍、不仰拍。
- 以正立面为主，只允许极少量厚度暗示。
- 不出现明显顶部大面积可见面。
- 物件底边平稳落在地面，方便直接摆在俯视地图中。

### 7.2 错误情况

以下都不算正视角：

- 明显俯视 45 度
- 明显顶视
- 三四分之三俯角
- 顶面比正面还醒目的箱子 / 石碑 / 建筑
- 透视太强、像插画摆拍而不是地图物件

### 7.3 Skill 中的明确写法

建议直接写进 skill：

```text
当用户请求装饰物贴图时，默认按“立式地图物件”理解，使用 RPG Maker 风格正视角。
正视角指物件正面朝向镜头、底边落地、无明显俯视顶面。
如果用户请求的是地表裂纹、阵纹、污渍等覆盖层，则改判为地表贴花，使用俯视角而不是正视角。
```

## 8. 命名规则建议

沿用项目已有命名习惯，避免时间戳命名。

| 类型 | 命名建议 |
|------|----------|
| 装备图标 | `icon_{slot}.png` / `icon_t{tier}_{slot}.png` / `icon_{unique_id}.png` |
| NPC 图 | `npc_{id}.png` |
| 小怪展示图 | `monster_{name}.png` / `monster_{name}_small.png` |
| 精英胸像 | `monster_{name}_elite.png` |
| BOSS 全身图 | `monster_{name}_boss.png` / `monster_{name}_king.png` |
| 特效资源 | `fx_{theme}_{name}.png` |
| 瓦片 | `tile_{theme}_{variant}.png` |
| 装饰物贴图 | `deco_{theme}_{name}.png` 或 `prop_{theme}_{name}.png` |
| 大型物件贴图 | `prop_{theme}_{name}_{n}x{n}.png` |
| 地表贴花 | `decal_{theme}_{name}.png` |

命名原则：

- 优先稳定可检索，不要自动加时间戳。
- 同一系统统一前缀。
- 文件名只表达“资源身份”，不要混入临时状态词。
- 怪物命名必须对齐现有 60+ 文件的**后缀**模式 `monster_{name}[_{role}].png`，如 `monster_spider_elite.png`、`monster_bandit_boss.png`。不要使用 `monster_elite_{id}.png` 这种角色前缀模式。

## 9. Skill 运行流程

建议 skill 固定采用下面的执行流程：

1. 识别资源类型。
2. 自动套用该类型的默认规格。
3. 只提取本次变量：
   - 主题
   - 材质
   - 阵营
   - 章节气质
   - 颜色倾向
   - 特征物
   - 占地规格（如 `2×2`、`3×3`）
4. 生成 prompt。
5. 生成图片并展示预览。

## 10. 对外响应规则

这是解决“不要每次都强调”的关键。

skill 对外输出时，不要每次都写一长串：

- PNG
- 透明背景
- 无文字
- 无 UI
- 保持统一风格

这些应该被视为 skill 已经内置完成。

推荐输出风格：

```text
资源类型：装饰物贴图
本次变量：青铜宝箱 / 根骨系 / 仙缘封印感
默认规格：已按 skill 内置规则处理
文件名建议：deco_chest_constitution.png
```

默认就按这四行输出，不额外铺完整规格；只有在用户要求时再补充完整 prompt 或详细参数。

只有在下面几种情况，才显式展开完整规格：

- 用户明确问“规格是什么”
- 用户请求和默认规则冲突
- 需要解释为什么这次用了正视角或俯视角

```markdown
# 资源生成

先判断资源类型，再套用默认规格，不要把所有通用规范每次重复给用户。

## 默认规则

- 世界观：仙侠修仙
- 风格：像素风
- 输出：PNG
- 禁止：文字、水印、UI
- 同类资源保持统一视角和尺寸

## 分类

- 装备图标：64x64，透明背景
- 角色/NPC 图：256x256，透明背景，全身站立
- 小怪展示图：256x256，透明背景，根据体型选择构图
- 精英头像：256x256，透明背景，胸像构图
- BOSS 全身图：256x256，透明背景，全身构图
- 特效资源：256x256，透明背景，主体居中，便于叠加
- 瓦片：128x128，不透明，俯视角，可无缝拼接
- 装饰物贴图：256x256，透明背景，RPG Maker 风格正视角
- 大型物件贴图：按占地 `N×N` 推导，先定占地再定视角
- 地表贴花：128x128 或 256x256，透明背景，俯视角

## 正视角规则

装饰物贴图中的正视角，指立式地图物件的正面朝向镜头、底边落地、无明显俯视顶面。
如果请求的是裂纹、阵纹、污渍等地面覆盖物，则改判为地表贴花，使用俯视角。

## 多格物件规则

`N×N` 表示逻辑占地，不是模糊尺寸描述。
立式大型物件默认正视角，基础宽度 = `N×128`；平铺大型物件默认俯视角，画布 = `N×128 × N×128`。

## 工作流

1. 判断资源类型
2. 提取本次变量
3. 生成 prompt
4. 生成图片并展示预览
5. 等用户确认后再接入代码或配置
```

## 12. 结论

这份 skill 方案的核心不是“多写几个 prompt 模板”，而是把下面五件事固定下来：

1. 常规规格内置化，不再反复强调。
2. 怪物资源分层化，明确区分小怪展示图、精英头像和 BOSS 全身图。
3. 特效资源独立化，明确它不是地图贴花，也不是地图物件。
4. 地图物件规格化，明确单格物件和 `N×N` 大型物件的边界。
5. 把“装饰物贴图 = RPG Maker 风格正视角”写成硬规则，并明确它不等于俯视角。
