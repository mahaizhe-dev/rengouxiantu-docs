## 3. Logger 设计

### 等级定义

| 级别 | 函数 | 用途 | 默认状态 |
|------|------|------|----------|
| ERROR | `Logger.error(tag, msg)` | 必须修复的异常 | 始终开启 |
| WARN | `Logger.warn(tag, msg)` | 值得关注的非致命问题 | 始终开启 |
| INFO | `Logger.info(tag, msg)` | 关键生命周期事件 | 始终开启 |
| DIAG | `Logger.diag(tag, msg)` | 诊断/调试详情 | **默认关闭** |

### 输出格式

```
[LEVEL][Tag] message
```

示例: `[ERROR][FetchSlots] JSON decode failed`

### 默认策略

- error/warn/info 始终输出（不可关闭）
- diag 默认关闭，通过 `Logger.SetDiagEnabled(true)` 开启
- `SetDiagEnabled` 仅接受 `boolean true` 才开启，其他值（数字、字符串、nil）均视为 false

### 纯叶模块

- 零依赖，不 require 任何业务模块
- 可在纯 Lua 环境（lupa）加载和测试
- 公开 API 6 个：`error`, `warn`, `info`, `diag`, `SetDiagEnabled`, `IsDiagEnabled`
