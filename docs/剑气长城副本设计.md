## 剑气长城副本设计

### 1. 文档定位

本文只定义 `剑气长城` 副本、通关奖励、积分商店与执行边界。

技能、图标、坐标、具体站位、宝箱外观细节由执行AI按当前代码机制自行决定。

### 2. 玩法定位

`剑气长城` 是第五章内的门票型多波怪副本。

基础定位：

1. 参考 `声望挑战` 的门票进入、临时挑战怪、挑战后结算模式
2. 不进入异空间
3. 角色会在第五章内传送到 `剑气长城` 实战区域
4. 实战区域使用第五章当前无普通怪刷新的区域
5. 副本入口 NPC 位于 `别院` 靠近 `剑气长城` 入口处
6. 入口 NPC 同时提供 `进入副本` 与 `剑气积分商店`

### 3. 进入规则

进入条件：

1. 玩家与入口 NPC 交互
2. 每次进入消耗 `1个仙人精血`
3. 消耗成功后，角色传送到第五章 `剑气长城` 实战区域
4. 传送完成后刷新第 `1` 波怪物

进入失败时，不应扣除门票。

### 4. 副本流程

副本采用固定 `3波怪物 + 1个大型结算宝箱`。

流程：

1. 玩家消耗门票进入副本
2. 角色传送到剑气长城实战区域
3. 刷新第 `1` 波怪物
4. 第 `1` 波全部击杀后刷新第 `2` 波怪物
5. 第 `2` 波全部击杀后刷新第 `3` 波怪物
6. 第 `3` 波 BOSS 击杀后生成大型结算宝箱
7. 玩家开启宝箱后获得奖励
8. 奖励结算完成后返回入口 NPC

宝箱规则：

1. 宝箱对标 `福源宝箱` 的大型宝箱表现
2. 宝箱只能开启 `1次`
3. 开启后直接获得装备与积分
4. 不存在重复开启、重新随机、补开

### 5. 波次配置

第 `1` 波：

1. `强化血屠将`
2. 数量：`6只`
3. 技能沿用原怪，不额外设定副本专属技能

第 `2` 波：

1. `强化魔帅`
2. 数量：`3只`
3. 三名魔帅各 `1只`
4. 技能沿用原怪，不额外设定副本专属技能

第 `3` 波：

1. BOSS：`镇关魔王·斩劫罗睺`
2. 等级：`120级`
3. 境界：`谪仙`
4. 品级：`圣级BOSS`
5. 作为最终守关目标，击杀后触发大型结算宝箱

BOSS详细技能由执行AI按现有怪物技能系统自行配置。设计侧只要求其主题围绕四仙剑融合，不要求固定技能ID、技能数量或阶段结构。

### 6. 副本强化

副本内所有怪物使用专属强化：

- `剑气长城·魔劫强化`

效果：

1. 攻击提升 `30%`
2. 防御提升 `30%`
3. 生命提升 `30%`
4. 速度提升 `30%`
5. 技能冷却缩减 `30%`

该强化用于区分副本怪与野外同名怪。

### 7. 掉落与奖励

小怪规则：

1. 小怪无直接掉落
2. 小怪只承担波次推进作用

BOSS规则：

1. BOSS无直接掉落
2. BOSS击杀后只触发大型结算宝箱

宝箱奖励：

1. `10~30点剑气积分`
2. `1件随机T10青色灵器`
3. 随机灵器有概率成为套装件

随机T10灵器规则：

1. 阶位固定 `T10`
2. 品质固定 `青色/灵器`
3. `30%概率` 生成套装件
4. `70%概率` 生成普通T10青装
5. 不做职业定向
6. 不做部位保底
7. 不做连续重复保护

套装件规则：

1. 只从四种制式套装中抽取
2. 套装白名单：`xuesha`、`qingyun`、`fengmo`、`haoqi`
3. 沿用现有 `setId`
4. 不新增T10专属套装身份

### 8. 剑气积分

`剑气积分` 是剑气长城副本的保底货币。

规则：

1. 只在通关宝箱中产出
2. 每次成功通关产出 `10~30点`
3. 失败不产出
4. 小怪不产出
5. 不作为普通背包材料处理
6. 建议保存为独立角色货币字段，例如 `swordWallPoints`

### 9. 剑气积分商店

入口 NPC 同时提供积分商店功能。

商店规则：

1. 使用 `剑气积分` 兑换
2. 所有商品都是指定兑换，不走随机
3. 商店定价按 `20点剑气积分/次副本` 的期望值折算
4. 单次副本实际产出为 `10~30点`，所以折算次数是期望值，不是固定次数
5. 四把仙剑作为 `4个独立商品` 上架，玩家指定兑换其中一把

### 10. 失败与退出

失败规则：

1. 玩家死亡视为挑战失败
2. 使用副本内 `退出传送阵` 视为挑战失败
3. 主动离开副本区域视为挑战失败
4. 失败后返回入口 NPC
5. 失败不保留波次进度
6. 失败不生成宝箱
7. 失败不获得积分或随机装备

成功规则：

1. 击杀第 `3` 波 BOSS
2. 生成大型结算宝箱
3. 玩家开启宝箱
4. 获得 `剑气积分 + 随机T10青色灵器`
5. 奖励写入成功后返回入口 NPC

### 11. 防作弊与存档要求

`剑气积分` 和宝箱结算需要有基础防作弊能力。

最低要求：

1. 每次挑战生成唯一 `runId`
2. 门票消耗、挑战开始、宝箱结算绑定同一个 `runId`
3. 每个 `runId` 只能结算一次
4. 宝箱开启成功后立即标记已开启
5. 积分数值由权威流程生成或校验，客户端不能自行写入
6. 积分商店兑换必须校验余额
7. 积分扣除与商品发放必须同一次写入存档
8. 记录积分获得与兑换日志

如果第一版只能做本地弱校验，也必须满足：

1. 积分不放进普通背包
2. 单次通关积分新增不得超过 `30`
3. 兑换时立即校验余额并存档
4. 重复宝箱开启必须拒绝

### 12. 执行边界

由执行AI自行决定：

1. BOSS技能细节
2. 怪物技能ID
3. 图标与美术资源
4. 入口 NPC 精确坐标
5. 剑气长城实战区精确坐标
6. 怪物刷新点
7. 大型宝箱坐标与表现
8. 退出传送阵坐标与表现
9. UI细节与按钮布局

执行AI必须遵守：

1. 不使用多人副本系统
2. 不进入异空间
3. 不复用会重建地图的声望挑战 `_createArena`
4. 基于声望挑战的门票、临时怪、挑战状态、死亡事件、结算思路实现
5. 副本怪必须是临时怪
6. 副本怪必须屏蔽普通掉落
7. 宝箱只能结算一次
8. 退出传送阵等同失败

### 13. 执行前需要确认

以下问题需要在正式执行前确认：

1. `剑气积分` 第一版是否必须接入服务端权威结算，还是允许先做本地弱校验
2. 积分 `10~30点` 是否使用等概率整数随机
3. 随机T10灵器的部位池是否使用全标准装备槽
4. 积分商店商品是否不限购
5. 背包满时，随机T10灵器是进入 `pendingItems`，还是阻止开启宝箱
6. 玩家断线或重登时，进行中的副本是否统一按失败处理
7. 宝箱开启后是否立即自动传回入口 NPC，还是允许玩家看完奖励后手动确认离开
