## 二、设计原则

1. **不污染第一章**：第六章不复用第一章 NPC、装饰、刷怪点和任务链；功能相似时也必须使用第六章独立 ID。
2. **ID 前缀统一**：NPC 与交互对象使用 `ch6_` 前缀；主线链使用独立 `shadow_village_zone`，避免复用 `town_zone`。
3. **封印边界清晰**：第 8 步只解开中央两界村封印；虎王试炼封印是未来副本封印，不随主线、GM 常规解封、读档同步或任务完成自动开启。
4. **系统入口沿用成熟框架**：主线 NPC 沿用 `VillageChiefUI` + `QuestSystem`，封印沿用 `QuestData.SEALS`、`QuestSystem.ApplySeals/Unseal` 和地图独立阻挡层。
5. **天青白莲独立养成**：天青白莲用于青莲长生体获取，需使用第六章独立 ID、独立进度与独立消耗；交互流程可参考福源果，但不得复用福源果存档字段。

### 3.1 主线 NPC 背景

第六章主线 NPC 为村长。原来两界村的村长并非凡人，而是镇守仙凡交界之地的仙人。两界村原本是仙凡交界之地，有天兵镇守，秩序稳定且极为安全；但不知仙界出了什么状况，魔君来犯时迟迟没有任何援军。留守两界村的仙兵、仙将在绝望中放弃了自己的使命，拒绝反攻两界村，只维持残存防线与自保秩序。

| 项目 | 规则 |
|------|------|
| 交互对象 | 天青白莲 |
| 数量 | 9 颗，从原计划 4 颗调整为 9 颗 |
| 分布 | 呱大人区域莲湖周边 9 颗；4 颗在湖心 BOSS 岛，5 颗沿西岸 / 东岸池水外圈散布，形成水边与岸边交错站位 |
| 状态 | 每颗白莲有未激活 / 已激活两种状态 |
| 激活表现 | 已激活状态下，NPC 名字呈现渐变青白色，并带类似福源果的光效 |
| 激活消耗 | 莲池仙露 ×1 + 2万灵韵 |
| 莲池仙露来源 | 蛤蟆仙人 0.2% 掉落；呱大人 0.05% 掉落 |
| 道具品级 | 莲池仙露为圣器级消耗品 |
| 黑市口径 | 莲池仙露进入黑市-消耗品；定价 60 / 30 |

| 激活流程 | 交互过程与界面参考福源果；每次激活界面显示当前激活进度，例如 3/9 |
| 全部激活奖励 | 9 颗全部激活后，弹出获取展示框，表现形式参考节日活动抽奖获得展示 |
| 最终获得 | 新仙体：青莲长生体 |
| 仙体属性 | 成长模板：生命 +30/级，生命恢复 +1/级，攻击 +3/级，防御 +2/级 |
| 解锁口径 | 只解锁青莲长生体，不自动切换当前角色仙体 |
| 仙体资源 | 需要新增仙体整卡资源；参考 `body_immortal_1` 的物象剪影、卡背、艺术字规格 |

实现边界：

- 已采用系统 ID：仙体 `immortal_body_qinglian`，交互类型 `qinglian_body_lotus`，消耗品 `lotus_pool_dew`。
- 9 颗天青白莲已使用稳定编号 `ch6_qingbai_lotus_01` 至 `ch6_qingbai_lotus_09`，当前分布为 4 颗湖心 BOSS 岛、5 颗池水外圈西岸 / 东岸水岸，避免后续位置调整导致存档错位。
- 每颗白莲只能激活一次；已激活白莲再次交互只显示已激活状态和当前进度，不扣除莲池仙露 / 灵韵，不重复给经验。
- 激活状态必须持久化；单颗经验奖励、最终仙体解锁和消耗扣除都需要防重复领取。
- 激活流程需要处理掉线 / 保存失败边界：参考仙体切换事务，先检查存档可用，再执行本地事务变更并调用 `SaveSystem.Save(callback)` 强保存；保存成功后确认激活，保存失败时回滚本次激活、消耗、经验和仙体解锁状态，避免未落盘状态悬在内存里。
- 9 颗全部激活时，只调用账号级仙体解锁，不自动切换当前角色；玩家后续仍通过仙体界面手动切换。
- 莲池仙露是新圣器级消耗品，已注册道具、图标、怪物掉落和黑市条目；暂不接入打包、锻造或其他后置系统。
- 青莲长生体作为新仙体，已接入 `AscensionConfig.GROWTH_PROFILES`、`ImmortalBodySystem` 账号级解锁、仙体面板卡片、属性预览、属性汇总和存档兼容。
- 仙体卡资源不是普通图标；已生成整张竖版仙体卡，包含青莲 / 长生意象剪影、青白卡背光效和底部“青莲长生体”艺术字，并加入仙体面板贴图映射。

存档与掉线口径：

| 场景 | 处理 |
|------|------|
| 正常激活 | 校验存档可用、白莲未激活、莲池仙露数量和灵韵；建立本地快照；扣除莲池仙露与灵韵；写入白莲激活状态；发放 1000万经验；若达到 9/9 则解锁账号级青莲长生体；随后强保存 |
| 重复交互 | 若该白莲已激活，只显示已激活和当前进度，不再扣消耗、不再给经验 |
| 连续点击 | 激活保存中锁定按钮 / 读条，防止同一白莲重复提交 |
| 保存失败 | 使用本地快照回滚白莲状态、背包、灵韵、经验 / 等级和账号仙体解锁缓存；提示失败，玩家可重新激活 |
| 保存成功前掉线或关闭 | 重新登录后以云存档为准；若激活未保存，则该白莲仍未激活，消耗和经验也不应持久化，玩家可重新激活 |
| 保存成功但回包丢失 | 重新登录后云存档已有激活状态，不能重复激活 |
| 9/9 完成 | 写入白莲进度和账号级仙体解锁；弹出获得展示；不切换当前角色仙体 |
| 已拥有仙体 | 账号已解锁青莲长生体时，9/9 完成只补齐本系统完成状态和展示提示，不重复创建账号资产 |

### 4.1 封印落地边界

| 封印 ID | 绑定任务链 | 作用 | 当前状态 |
|---------|------------|------|----------|
| `ch6_shadow_village` | `shadow_village_zone` | 中央两界村四门封印 | 第 8 步可解除，解除后恢复 `TOWN_ROAD` |
| `seal_ch6_tiger_trial` | `ch6_tiger_trial_future` | 虎王试炼未来副本入口 | `manualOnly = true`、`futureLocked = true`，当前没有任何解封方式 |

实现约束：

- `VillageChiefUI` 查找 `questChain = "shadow_village_zone"`，第 8 步调用 `QuestSystem.Unseal("ch6_shadow_village")`。
- `seal_ch6_tiger_trial.questChain` 不得绑定 `shadow_village_zone`。
- `QuestSystem` 对 `futureLocked` 封印执行拒绝解封、读档重锁、GM 常规切换阻断和地图阻挡层保底。
- 地图封印不只依赖瓦片不可通行，还会写入 `GameMap:SetSealBlockedTiles`，防止视觉或地形覆盖后穿过封印。
