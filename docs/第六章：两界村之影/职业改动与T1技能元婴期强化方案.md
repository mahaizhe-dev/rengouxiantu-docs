> 状态：执行稿

> 范围：罗汉、太虚、镇岳
> 说明：本稿只记录最终确认的技能改动口径，不包含未改项、对比、评估、测算、建议稿。

---

## 1. 执行范围

1. 本稿只覆盖三职业：
   - 罗汉
   - 太虚
   - 镇岳
2. 阵法师不在本稿范围内。
3. 三职业 `1技能` 的 `元婴初期` 强化，统一通过 `SkillData.RealmEnhancements.yuanying_1` 配置。

## 2. 通用执行要求

### 2.1 动态取值

所有改动都必须读取实时面板值，不允许写死结果值。

| 动态值 | 读取来源 | 用途 |
|------|----------|------|
| 总攻击 | `Player:GetTotalAtk()` | 太虚技能说明与结算 |
| 总防御 | `Player:GetTotalDef()` | 罗汉地印说明与结算 |
| 总最大生命 | `Player:GetTotalMaxHp()` | 御剑诀、裂山说明与结算 |
| 总击杀回血 | 统一 helper，口径与 `GameEvents.monster_death` 一致 | 一剑开天触发回血 |

### 2.2 总击杀回血统一口径

统一 helper 返回：

```text
floor(
  equipKillHeal
  + titleKillHeal
  + collectionKillHeal
  + pillKillHeal
  + artifactTiandiKillHeal
  + minggeKillHeal
)
```

该数值只作为一剑开天触发回血的计算基数；若巨剑造成击杀，击杀回血仍按原击杀规则单独结算。

### 2.3 通用限制

1. handler 必须走通用字段扩展，不按技能 ID 写死职业规则。
2. 技能面板与真实结算必须读取同一组动态值。
3. 技能文案、真实结算、判定范围、表现效果必须同步落地。

### 2.4 特效链路

当前技能特效链路如下：

1. 主动技能释放：
   - `casting_a.lua`
   - `casting_b.lua`
   - `passives.lua`
2. 短时爆发特效：
   - 通过 `CombatSystem.AddSkillEffect()` 注册到 `CombatSystem.skillEffects`
   - 由 `rendering/EffectRenderer.lua` 按 `skillId` 分发到：
     - `rendering/effects/skills_a.lua`
     - `rendering/effects/skills_b.lua`
3. 持续地面区域：
   - 通过 `CombatSystem.AddZone()` 注册到 `CombatSystem.activeZones`
   - 由 `rendering/effects/zones.lua` 统一渲染

### 2.5 特效字段边界

`CombatSystem.AddSkillEffect()` 当前只透传以下字段：

```text
shape
coneAngle
rectLength
rectWidth
targetAngle
zones
zoneSize
centerOffset
duration
```

本批特效落地前，补充透传以下通用字段：

```text
effectVariant
swordCount
outerShockwaveScale
outerShockwaveDelay
zoneVisualKey
showZoneLabel
tickFlashColor
healFlashColor
```

`CombatSystem.AddZone()` 当前只有通用圆形区域表现。本批如需做金刚印地面，不直接复用默认圆区表现，需补：

```text
zoneVisualKey
showZoneLabel
tickFlashColor
```

### 2.6 角色面板展示要求

1. `yuanying_1` 的 `1技能强化`，在角色面板技能页中必须作为 `1技能` 下方的独立条目显示。
2. 展示位置放在 `CharacterTab2` 的技能配置区，紧跟 `slotIndex = 1` 的技能卡片之后，不并入 `1技能` 的基础描述文本。
3. 展示样式统一对齐当前 `CharacterTab2` 里 `金丹强化` 的显示方式：
   - 先有分隔线
   - 再显示强化标签
   - 再显示强化描述
4. 展示数据统一读取 `SkillData.GetRealmEnhancement("yuanying_1", player.classId)`，不新增第二套面板配置。
5. 仅当以下条件同时满足时显示：
   - 存在 `SkillData.RealmEnhancements.yuanying_1[player.classId]`
   - `enhancedSkill` 对应 `1技能`
   - 玩家境界达到 `元婴初期` 及以上
6. 展示内容至少包含：
   - `元婴强化` 标签
   - 强化名称
   - 强化描述
7. `BreakthroughCelebration` 继续沿用当前大境界突破强化展示；角色面板只补充常驻展示，不新增独立系统。
8. `SkillData.GetDynamicDescription()` 只负责基础技能说明，不把元婴强化文案拼接进 `1技能` 主描述。

## 3. 罗汉

### 3.1 `yuanying_1` -> `ice_slash` 金刚掌强化

执行口径：

1. 本体命中后，在命中区域留下 `1个` 金刚印。
2. 同次施法不生成第二块地面。
3. 金刚印范围与金刚掌本体命中范围一致。
4. 金刚印伤害为普通伤害，可暴击。

执行字段：

```text
followupGroundZone = {
  tickInterval = 0.5,
  duration = 3.0,
  damageSource = "def",
  damageCoeff = 0.25,
  usesCalcDamage = true,
  canCrit = true,
  shape = "same_as_hit_area"
}
```

面板要求：

1. 显示 `每0.5秒：防御 × 0.25`
2. 显示 `持续3秒`
3. 显示 `可暴击`
4. 显示当前单跳估值

表现要求：

1. 先出现掌击命中表现。
2. 命中后衔接同一枚地印。
3. 不允许额外生成第二套无关联地面特效。

特效实现建议：

1. `renderIceSlash` 当前已经有：
   - 身前圆形冲击波
   - 掌印
   - 金色气浪
   - 地裂纹
2. 金刚掌强化不重写本体 renderer，继续复用 `rendering/effects/skills_a.lua` 里的 `renderIceSlash` 作为起手爆发。
3. 金刚印持续 `3秒`，不应继续挂在 `skillEffects` 上做长驻留；应改为：
   - 本体仍走 `AddSkillEffect()`
   - 地印改走 `AddZone()`
4. 金刚印地面表现不要使用 `zones.lua` 的默认圆区样式，需新增 `zoneVisualKey = "monk_seal_zone"`，在 `rendering/effects/zones.lua` 加独立 renderer。
5. `monk_seal_zone` 建议表现：
   - 外圈金色法印圆环
   - 中心掌印残像
   - 每 `0.5秒` 触发一次短脉冲亮边
   - 不显示区域名称标签
6. 金刚印位置必须与 `renderIceSlash` 的掌击落点一致；不要重新以玩家脚下或目标脚下重新取中心。

## 4. 太虚

### 4.1 `yuanying_1` -> `three_swords` 破剑式强化

执行口径：

1. `20%` 概率触发大破剑式。
2. 大破剑式：
   - 三剑改五剑
   - 伤害系数改为 `250%`
   - 扇形角度改为 `90`
   - 命中上限改为 `5`
3. 大破剑式继续吃连击。

执行字段：

```text
upgradeChance = 0.20
upgradeVariant = {
  swordCount = 5,
  damageMultiplier = 2.5,
  coneAngle = 90,
  maxTargets = 5,
  keepCombo = true
}
```

面板要求：

1. 显示 `20%概率触发大破剑式`
2. 显示 `大破剑式：250%攻击力`
3. 显示 `五剑 / 宽扇形 / 可连击`

表现要求：

1. 大破剑式与普通破剑式必须有明显区分。
2. 五剑轨迹、扇面宽度、覆盖范围都要同步放大。

特效实现建议：

1. `renderThreeSwords` 当前硬编码为：
   - `3把剑`
   - `60度扇面`
2. 大破剑式要和判定一致，必须把 `renderThreeSwords` 改成读取：
   - `se.swordCount`
   - `se.coneAngle`
3. 大破剑式建议通过 `effectVariant = "great_break_sword"` 分支表现：
   - 剑数量读取 `5`
   - 扇面读取 `90`
   - 剑体尺寸放大
   - 剑尖高亮更强
   - 增加更宽的青白色残影弧线
4. 不要只放大已有三剑贴图，否则会出现判定五剑、表现三剑的错位。

### 4.2 `sword_shield` 御剑诀

执行口径：

1. 每消耗 `1柄` 灵剑，立即回复 `8%当前最大生命`。

执行字段：

```text
breakHealPercent = 0.08
```

面板要求：

1. 显示 `单剑回复 = 8%最大生命`
2. 显示当前单剑回复值
3. 战斗内显示 `剩余灵剑 X/3`

表现要求：

1. 灵剑破碎时同步出现回血浮字。
2. 灵剑破碎反馈与回血反馈同时触发。

特效实现建议：

1. 当前 `renderSwordShield` 已有：
   - 环形护盾闪光
   - 三段护盾弧
   - 剑影粒子
2. 破剑回血不新建单独技能 renderer，继续复用 `renderSwordShield`，但增加 `effectVariant = "break_heal"` 分支。
3. `break_heal` 分支建议补两层反馈：
   - 现有蓝白护盾破碎外闪
   - 角色脚下绿色短脉冲回命环
4. 回血浮字继续走 `CombatSystem.AddFloatingText()`，不要把回血数值画进 renderer 贴图。

### 4.3 `sword_formation` 一剑开天

执行口径：

1. 移除原 `triggerKillHealMultiplier = 2` 的旧口径。
2. 改为：

```text
巨剑触发时，立即回复 floor(当前总击杀回血 × 8)
```

3. 本次触发回血与命中目标数量无关。
4. 若巨剑造成击杀，击杀回血继续按原击杀回复属性单独结算。

执行字段：

```text
triggerOnHitHealFromKillHealMultiplier = 8
```

面板要求：

1. 显示 `计数 N/8`
2. 显示 `触发回复 = 当前总击杀回血 × 8`
3. 显示当前单次触发实际回复值

表现要求：

1. 巨剑触发时结算一次回血反馈。
2. 不按命中目标数量拆分多次回血浮字。

特效实现建议：

1. 当前 `renderSwordFormation` 已经有：
   - 巨剑下落
   - 落地白闪
   - 三层冲击波
2. 巨剑本体继续复用当前 `renderSwordFormation`，沿用现有落地冲击主表现。
3. 触发回血不要复用当前巨剑落地点表现，因为当前 renderer 的中心点在玩家前方落点，不在玩家自身。
4. 回血反馈建议单独补一个玩家身上的短时效果，推荐两种实现方式任选其一：
   - 在 `renderSwordFormation` 内增加 `effectVariant = "trigger_heal"` 分支，改为以玩家当前位置为中心渲染绿色回命环
   - 或新增单独 renderer key，例如 `sword_formation_heal`
5. 无论采用哪种方式，都只结算并表现 `1次` 回血，不随命中目标数重复播放。

## 9. 执行前代码基准评估

> 评估日期：2026-06-30
> 评估口径：以 `scripts/` 当前代码为准，本文档方案为目标口径。

### 9.1 总体结论

本方案可执行，但不是纯配置改动，整体风险为 **中高**。

风险主要集中在：

1. 战斗数值放大与多段触发叠加。
2. `HitResolver`、`player_deal_damage`、击杀回血、血怒、套装 AOE 等运行时事件联动。
3. 太虚御剑诀回血与一剑开天回血的重复结算风险。
4. 副本 BOSS C/S 伤害汇报与本地表现伤害的一致性。
5. 角色面板、突破弹窗、技能真实结算的动态数值一致性。

执行策略必须采用：

1. 先补测试。
2. 再补通用 helper 与配置字段。
3. 再改结算链路。
4. 最后补 UI 与表现。

不得直接修改基础技能字段来承载元婴强化，否则未达到元婴的玩家也会吃到强化。

### 9.3 数值战斗风险

#### 罗汉金刚印

`CombatSystem.AddZone()` 当前地面区域伤害是：

```text
atk × damagePercent -> GameConfig.CalcDamage -> Monster:TakeDamage
```

当前 zone 不走 `HitResolver`，因此默认不支持暴击，也不会保留标准命中事件语义。

金刚印要求“防御 × 0.25、普通伤害、可暴击”，执行时必须扩展 zone 通用字段，例如：

```text
damageSource = "def"
damageCoeff = 0.25
usesCalcDamage = true
canCrit = true
isDot = true
```

并通过 `HitResolver.Hit()` 结算。建议将 tick 标记为 `isDot = true`，避免每 0.5 秒触发血怒、套装 AOE 等非预期连锁。

#### 太虚大破剑式

大破剑式必须是“本次施法快照”，不能修改 `SkillData.Skills.three_swords` 原表。

风险点：

1. 若直接改原表，触发一次后后续普通破剑式会被污染为 5 剑 / 90 度 / 250%。
2. 连击必须复用同一次大破剑式参数，否则会出现首段 5 剑、连击 3 剑的结算不一致。
3. 表现必须读取 `swordCount` 和 `coneAngle`，不能判定 5 剑但只渲染 3 剑。

#### 御剑诀回血

御剑诀当前通过 `Player:AddPreDamageHook("sword_shield", ...)` 在 `Player:TakeDamage()` 的受伤前钩子里执行。

方案要求“每消耗 1 柄灵剑，立即回复 8% 当前最大生命”。这意味着默认实现顺序会是：

```text
受击 -> 消耗灵剑 -> 立即回血 -> 返回减伤后的伤害 -> 后续继续承伤
```

该顺序会提升低血量容错，是本批太虚生存增强的主要风险点。必须用测试固定：

1. 每消耗 1 剑只回血 1 次。
2. 回血以 `Player:GetTotalMaxHp()` 动态值计算。
3. 回血不超过最大生命。
4. 剩余灵剑数正确。

#### 一剑开天回血

当前一剑开天通过 `triggerKillHealMultiplier = 2` 设置 `_killHealMultiplier`，只在击杀事件中影响击杀回血。

新口径要求：

```text
巨剑触发时，立即回复 floor(当前总击杀回血 × 8)
```

风险点：

1. 不能随命中目标数量重复回血。
2. 不能随击杀目标数量重复触发该“触发回血”。
3. 巨剑造成击杀时，原击杀回血仍应按 `monster_death` 正常结算。
4. `_killHealMultiplier` 不应再参与一剑开天。

#### 镇岳裂山

裂山元婴强化不能直接覆盖基础技能字段。应在施法时按玩家境界生成运行时参数：

```text
range = 2.0
hpCost = 0.06
hpCoefficient = 0.12
```

血神累计应按实际 HP 消耗累加。若实现为 `0.05 + 额外 0.01`，需避免重复累计；若实现为运行时 `hpCost = 0.06`，则 `feedsAccumulator` 自然累计 6%。

### 9.4 C/S 架构风险

当前多人副本 BOSS 伤害链路为：

```text
本地 CombatSystem / SkillSystem
  -> Monster:TakeDamage()
  -> dungeon boss _onDamageTaken()
  -> DungeonClient pendingDamageReport_
  -> C2S_DungeonAttack
  -> DungeonHandler.HandlePlayerAttack()
```

只要新增伤害最终经过 `Monster:TakeDamage()`，服务端能收到总伤害汇报。

主要风险不是协议字段，而是数值峰值：

1. 服务端 `MAX_DAMAGE_PER_REPORT = 50000`，客户端 0.3 秒内汇总伤害超过上限会被截断。
2. 大破剑式 + 连击 + 一剑开天可能造成 0.3 秒 burst 增大。
3. 若本地飘字显示完整伤害，但服务端截断，BOSS 血量、排行、奖励占比会与本地感知不一致。

上线前必须压测元婴太虚在副本中 0.3 秒伤害包峰值。

### 9.5 存档系统与线上数据风险

本批目标不应新增持久字段，不需要迁移。

当前相关状态：

| 字段/状态 | 当前是否存档 | 风险 |
|------|------|------|
| `classId` | 是 | 不改字段名 |
| `realm` | 是 | 不改境界顺序和值 |
| `_bloodAccumulator` | 是，字段名 `bloodAccumulator` | 镇岳扣血累计会影响已有玩家状态 |
| `_swordShield` | 否 | 加载后仍按当前逻辑重新初始化 |
| `_killHealMultiplier` | 否 | 一剑开天应停止使用该临时字段 |
| `pillKillHeal` | 是 | 只读，不改含义 |
| 装备/称号/图录/天地法宝/命格击杀回血 | 多来源运行时聚合 | 只读，不改存档 key |

禁止事项：

1. 不新增“灵剑剩余数量”存档。
2. 不改 `bloodAccumulator` 字段名。
3. 不改 `pillKillHeal`、装备 `killHeal`、称号/图录/命格击杀回血字段含义。
4. 不改 `REALMS` 顺序和值。

线上影响：

1. 元婴及以上玩家会立即获得 T1 技能强化。
2. 太虚练气/筑基技能改动会影响已解锁御剑诀、一剑开天的玩家。
3. 副本排行、推图速度、生存续航会变化，需要观察上线后伤害排行和战斗时长分布。
