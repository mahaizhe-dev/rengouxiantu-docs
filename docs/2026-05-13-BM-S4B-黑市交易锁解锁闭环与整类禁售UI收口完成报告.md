## 一、问题背景

BM-S4A 为黑市引入了交易保护锁（`bmLockUntil` / `bmLockSource` / `bmLockBatchId`），但落地后发现两个遗留问题：

### 问题 A — 解锁闭环缺陷（无限锁循环）

- `SerializeItemFull()` 将锁字段写入序列化输出 → 云存档
- `game_saved` 事件触发 `ClearAllOnSaveSuccess()` 清除内存锁
- 下次登录加载存档 → 锁字段从云存档恢复 → 锁重新出现
- **结果**：锁永远无法解除，玩家被永久锁定

### 问题 B — 整类禁售 UI 缺失

- S4A 的锁检查仅在确认对话框的确认按钮 onClick 内（深层拦截）
- 卖出按钮本身可点击、不变色、不提示 → 玩家不理解为什么卖不了
- 且按 `itemId` 只要有任一堆叠被锁就应整类禁售，但 UI 未体现

---

## 二、修复方案

### 问题 A：SerializeItemFull 去锁字段

**策略**：锁是临时态，不应落盘。从序列化输出中移除三个锁字段。

**改动**：`scripts/systems/save/SaveSerializer.lua`

```diff
-        -- BM-S4A: 交易保护锁字段
-        bmLockUntil = item.bmLockUntil,
-        bmLockSource = item.bmLockSource,
-        bmLockBatchId = item.bmLockBatchId,
+        -- BM-S4B: 交易保护锁字段不落盘（锁为临时态，存档成功即清除）
+        -- bmLockUntil / bmLockSource / bmLockBatchId 故意省略
+        -- 见 BlackMarketTradeLock.ClearAllOnSaveSuccess()
```

- 不需要存档版本号升迁（v24 不变）——仅移除字段，旧存档中残留字段被 `DeserializeItem` 忽略
- `game_saved` → `ClearAllOnSaveSuccess` 保留，作为内存清理的冗余保障

### 问题 B：整类禁售 UI 前置拦截

**新增 API**：

| 文件 | 函数 | 作用 |
|------|------|------|
| `BlackMarketTradeLock.lua` | `HasAnyLockedConsumable(getItemFn, maxSlots, consumableId)` | 遍历背包判断指定消耗品是否存在任何锁定堆叠 |
| `InventorySystem.lua` | `HasAnyLockedConsumable(consumableId)` | 门面包装，供 UI 层直接调用 |

**UI 改动**（`BlackMerchantUI.lua` → `BuildItemCard`）：

1. 计算 `hasLockedStacks = InventorySystem.HasAnyLockedConsumable(itemId)`
2. `canSell` 追加 `and (not hasLockedStacks)` 条件
3. 卖出按钮文本：锁定时显示 `"🔒锁定"`，正常时显示 `"出售"`
4. onClick 内新增实时检查门（items_ 可能在 `BuildItemCard` 之后变化）
5. 装备类物品（`itemType == "equipment"`）不受影响——仅消耗品适用

**防御纵深保留**：`ShowConfirmDialog` 内的 S4A 锁检查不删除，作为最后一道防线。

  ↓

  ↓

  ↓
