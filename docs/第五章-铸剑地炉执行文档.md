> 原则：复用第四章 `DragonForgeUI` 与 `龙极令系统` 的背包打造逻辑，不重开一套新框架。

## 一、执行范围

第五章在 `铸剑地炉` 新增装备锻造交互，统一走背包打造模式。

执行本文件前，先完成 `四把封印古剑` 的前置重定义：

- 保留现有 `主属性 + 副属性 + 特效`
- 在现有基础上，新增 `1条随机灵性属性`
- 该随机灵性属性按 `装备实例生成` 处理，不写成固定模板常量
- 参考并更新 [装备属性.md](/C:/Users/mahaizhe/Documents/New%20project/taptapmaker/rengouxiantu-docs/docs/架构文档/装备属性.md) 文档，新增 `圣性属性` 定义：
  - 机制与 `灵性属性` 相同
  - UI 显示为 `红色底`
  - 数值强度按 `100%副属性` 计算
  - 本条为执行前置，当前只记文档，不立即改系统
- 新增 `1件T10灵器披风` 作为披风合成材料：
  - 固定副属性：`攻击 / 暴击伤害 / 生命`
  - 移除 `灵性属性`
  - 新增 `1条防御向被动`
- `噬渊魔氅` 前置重定义：
  - 移除 `灵性属性`
  - 新增 `1条防御向被动`
- 新增 `1件T10灵器级戒指`：
  - 固定副属性：`防御 / 生命 / 生命回复`
  - 无 `灵性属性`
  - 固定特效：`你属性最低的仙缘属性获得100点加成`

本次纳入 5 条配方：

1. 一重解封古剑
2. 藏真圣衣
3. 圣器披风
4. 帝尊圣戒
5. 龙神令

#### 规则

- 参与打造的 `两件T10灵器披风` 需预先完成前置重定义：
  - 均移除 `灵性属性`
  - 均新增 `1条防御向被动`
- `T10 噬渊魔氅` 升级为新 `T10 圣器披风`

- 圣器披风固定副属性：`暴击 / 暴击伤害 / 生命`

### 4.4 帝尊圣戒

#### 规则

- `帝尊一~五戒` 合成为新 `帝尊圣戒`
- 产物品质为 `T10 圣器`
- 属性风格延续帝尊戒指路线
- 固定 `1条圣性属性：福缘`
- 固定副属性：`悟性 / 根骨 / 体魄`

|------|------|
| 圣性属性 | `福缘` |
| 副属性1 | `悟性` |
| 副属性2 | `根骨` |
| 副属性3 | `体魄` |

#### 配方

| 产物 | 材料 | 金币 |
|------|------|------|

---

### 4.5 龙神令

#### 规则

- 任意 `龙极令` 升级为 `龙神令`
- 产物品质为 `T10 灵器法宝`
- 固定主属性：`攻击`
- 固定特效，沿用 `龙神令` 独立特效稿
- 其余属性全部随机生成
- 带 `1条灵性属性`
- `太虚剑令 xN` 中的 `N` 当前保留为待定参数，执行前补值

#### 配方

| 产物 | 材料 | 金币 |
|------|------|------|
| `龙神令` | `任意龙极令 x1` + `仙人精血 x1` + `太虚剑令 xN` | `0` |

## 六、执行约束

1. 不新开独立锻造 UI，直接复用 `DragonForgeUI`
2. 不走装备栏替换逻辑，全部按背包打造处理
3. 圣器产物的 `随机灵性属性` 规则与第四章圣器武器一致
4. 所有“特效不变”的配方，不重写原特效脚本，只做品级升级和灵性追加
5. `圣器披风` 为例外：继承两件灵器披风统一追加的 `防御向被动`，并额外新增 `1条随机灵性属性`
6. `帝尊圣戒`、`圣器披风名称`、`龙神令所需太虚剑令数量 N` 当前仍是待补参数，不阻塞文档执行
