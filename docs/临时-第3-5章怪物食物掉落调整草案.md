## 仙1项链暴击异常排查与恢复方案（草案）

### 现象

玩家截图中出现：

- `10阶 [灵器] 项链` 主属性暴击约 `+43.0%`，表现正常。
- `仙1阶 [圣器] 项链` 主属性暴击约 `+0.6%`，明显异常。

按当前装备配置，仙1圣器项链主属性应为：

```text
BASE_MAIN_STAT.necklace.critRate = 0.03
PCT_TIER_MULTIPLIER[11] = 9.0
GameConfig.QUALITY.red.multiplier = 2.1

0.03 × 9.0 × 2.1 = 0.567 = 56.7%
```

因此 `+0.6%` 不是正常数值波动，而是被缩小了 100 倍。

### 根因定位

问题点在旧存档百分比归一化逻辑：

```text
scripts/systems/save/SaveMigrations.lua
NormalizePercentStats(item)
```

当前旧逻辑大意：

```lua
local PERCENT_STATS = { critRate = 0.5, critDmg = 2.0 }

if item.mainStat[stat] and item.mainStat[stat] > threshold then
    item.mainStat[stat] = item.mainStat[stat] / 100
end
```

该逻辑原本用于修复极早期旧存档格式，例如 `3` 表示 `3%`，应转为 `0.03`。

但仙1圣器项链的合法值是：

```text
0.567 = 56.7%
```

它大于 `critRate` 阈值 `0.5`，因此被误判为旧格式，再除以 100：

```text
0.567 / 100 = 0.00567
0.00567 × 100 = 0.567%
```

最终 UI 显示约 `+0.6%`。

### 影响范围

当前实际风险集中在：

```text
item.slot == "necklace"
item.mainStat.critRate
高阶 / 高品质项链主属性
```

已确认会被旧阈值误伤的典型范围：

- T10 red/gold/rainbow 项链
- T11 red/gold/rainbow 项链
- T12 cyan/red/gold/rainbow 项链
- T13 cyan/red/gold/rainbow 项链
- T14 orange+ 项链
- T15 purple+ 项链
- T16 blue+ 项链
- T17 blue+ 项链
- T18 green+ 项链
- T19 white+ 项链
- T20 white+ 项链

当前副属性暴击、洗练暴击、暴击伤害暂未达到旧阈值，不是本次主要事故点。

### 线上恢复目标

1. 阻止新装备继续被压坏。
2. 让已经被压坏的线上装备在玩家加载存档时自动恢复。
3. 不弹迁移界面，不要求玩家手动操作。
4. 不改装备配置、不改 UI 显示、不改品质倍率、不改 Tier 倍率。
5. 不升存档版本，避免旧代码因前向版本检查拒绝加载。

### 推荐代码修复方案

只修改：

```text
scripts/systems/save/SaveMigrations.lua
NormalizePercentStats(item)
```

不修改：

```text
scripts/systems/save/SaveState.lua
scripts/config/EquipmentData.lua
scripts/config/GameConfig.lua
scripts/ui/EquipTooltip.lua
```

核心逻辑：

1. 对主属性百分比，先根据 `slot / tier / quality` 计算当前合法值。
2. 合法值附近不处理，例如 `0.567` 保持不变。
3. 明显旧格式大数才除以 100，例如 `56.7 → 0.567`。
4. 已经被旧逻辑压坏的小数自动恢复，例如 `0.00567 → 0.567`。
5. 副属性和洗练属性只处理明显旧格式大数，避免误伤当前合法值。

建议判定伪代码：

```lua
local function GetExpectedPctMainValue(item, stat)
    if not item.slot or not item.tier or not item.quality then return nil end
    if not item.mainStat or item.mainStat[stat] == nil then return nil end
    if not EquipmentData.PCT_MAIN_STATS[stat] then return nil end

    local mainStatType = EquipmentData.MAIN_STAT[item.slot]
    if mainStatType ~= stat then return nil end

    local baseStat = EquipmentData.BASE_MAIN_STAT[item.slot]
    local baseValue = baseStat and baseStat[stat]
    if not baseValue then return nil end

    local qualityConfig = GameConfig.QUALITY[item.quality]
    local qualityMult = qualityConfig and qualityConfig.multiplier or 1.0
    local tierMult = EquipmentData.PCT_TIER_MULTIPLIER[item.tier] or 1.0
    return math.floor(baseValue * tierMult * qualityMult * 100 + 0.5) / 100
end

local function NormalizeMainStat(item, stat)
    local value = item.mainStat and item.mainStat[stat]
    if type(value) ~= "number" then return end

    local expected = GetExpectedPctMainValue(item, stat)
    if expected then
        if value > expected * 10 then
            item.mainStat[stat] = value / 100
        elseif expected > 0.5 and value > 0 and value < expected * 0.1 then
            item.mainStat[stat] = expected
        end
        return
    end

    if stat == "critRate" and value > 2.0 then
        item.mainStat[stat] = value / 100
    elseif stat == "critDmg" and value > 5.0 then
        item.mainStat[stat] = value / 100
    end
end
```

| 用例 | 输入 | 期望 |

### 发布恢复流程

1. 修改 `NormalizePercentStats`。
2. 跑 Lua LSP，要求 0 Error。
3. 用问题账号在预览服验证：
   - 登录前：仙1圣器项链暴击显示约 `+0.6%`。
   - 加载修复版后：应显示约 `+56.7%`。
4. 触发一次正常保存。
5. 重新登录确认仍保持 `+56.7%`。
6. 确认无误后再正式发布。
