# 2026-05-13 BM-S4AR 黑市交易保护锁强制分堆与 UI 提示收口任务卡

> 任务性质：`BM-S4A` 收口任务  
> 前置结论：`BM-S4A` 当前**未通过**。原因不是保护锁方向错误，而是**买入路径仍然先并老堆，再整堆加锁**，不符合产品要求。  
> 本卡目标：只收口黑市交易保护锁的**强制分堆**与**玩家可感知提示**，不扩写其他黑市逻辑。

---

## 1. 本次任务要解决的问题

当前 `BM-S4A` 的核心偏差是：

1. 黑市买入的消耗品没有强制拆成新堆；
2. 现有实现会先并入旧未锁堆，再把旧堆整体加锁；
3. 玩家无法从 UI 明确感知“这是黑市锁定堆”；
4. 黑市 NPC 面板没有事先说明“购入商品会短暂冻结，直到触发成功存档”。

本次任务的目的不是重做整套黑市，而是把这 4 个关键点收干净。

---

## 2. 明确的产品要求

### 2.1 强制分堆

黑市买入的消耗品必须：

- **强制新建锁定堆**
- **不能并入任何已有未锁定同类堆**
- **不能并入其他批次的已锁定同类堆**

也就是说：

- 玩家背包里原本有 `gold_brick x9`
- 黑市新买 `gold_brick x1`

结果必须是：

- 旧堆：`gold_brick x9`（未锁）
- 新堆：`gold_brick x1`（已锁）

而不是：

- 一个 `gold_brick x10` 的整堆锁

### 2.2 锁图标

锁定堆必须有**小图标标识**，玩家能一眼看出：

- 这是黑市保护锁商品
- 当前不能立即使用/消耗/存仓/回售

### 2.3 NPC 面板常驻提示

黑市 NPC 交互面板上必须有常驻提示，明确告诉玩家：

> 购入商品会进入短暂保护期，期间不可使用、消耗、存仓或回售；成功存档后自动解除。

### 2.4 二次提示

除常驻提示外，还必须在至少一个环节再次提示：

- 买入确认弹窗
- 或买入成功后的状态提示 / Toast

避免玩家只看到图标，不理解规则。

## 6. 实现要求

### 6.1 客户端买入路径必须强制新建锁堆

当前问题在于：

- 先 `AddConsumable`
- 后 `ApplyLock`

这会导致“并老堆再加锁”。

本次必须改成：

- 黑市买入消耗品走**专门的新增入口**
- 该入口直接创建新锁堆
- 不允许合并进旧未锁定堆

建议方向：

- `InventorySystem.AddConsumableLockedNewStack(...)`
- 或等价命名

要求：

- 普通掉落/普通奖励仍走原有 `AddConsumable`
- 只有黑市买入走“强制分堆”路径

### 6.2 服务端写回也必须强制新建锁堆

`HandleBuy()` 写 `saveData.backpack` 时也必须改成：

- 不走普通 `AddToBackpack()` 的未锁合并逻辑
- 黑市买入专用路径直接创建新锁堆

建议方向：

- `BackpackUtils.AddLockedNewStack(...)`
- 或给 `AddToBackpack()` 增加受控参数 `forceNewStack = true`

但要求很明确：

- 不能再出现“先合并，再打锁”

### 6.3 锁堆与未锁堆继续保持隔离

在完成“强制分堆”后，仍必须保留：

- 锁堆与未锁堆不合并
- 不同锁批次不合并
- 消耗时优先只吃未锁堆
- 黑市回售时只允许卖未锁堆

### 6.4 UI 锁图标

在背包/相关显示中，对锁定堆加明确标识。

最低要求：

- 小锁图标或等价视觉标记
- 图标不会误标未锁堆
- 图标不会因为和老堆混合而语义失真

如果当前 UI 能力有限，可接受：

- 图标 + 浮层提示
- 图标 + 名称角标

但不能只靠文案，不做视觉区分。

### 6.5 黑市 NPC 常驻提示

在黑市主面板加入常驻说明，建议文案：

`黑市购入商品会进入短暂保护期，期间不可使用、消耗、存仓或回售；成功存档后自动解除。`

要求：

- 文案不能藏得太深
- 打开黑市面板即可看到
- 不依赖玩家先点购买才知道

### 6.6 买入确认 / 成功后二次提示

至少补一处：

- 购买确认弹窗
- 或购买成功状态栏提示

建议优先两处都补，但至少要有一处。

2. 服务端 `saveData.backpack` 已有未锁同类，黑市买入后写回

5. 锁堆与旧堆并存时，消费/卖钱只允许扣旧堆
