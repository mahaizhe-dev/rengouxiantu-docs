更新时间：2026-07-11

## 说明

- 本文按当前工作树中的第三章代码配置核对整理。
- 本文只统计第三章主线区域与其实际掉落，不包含神器单独战斗房、仙缘宝箱房等旁支系统。

## 一、第二章 + 第三章联合 ROLL 掉落列表

### 观察口径

- 本节直接列出第二章和第三章的表内独立 ROLL，包含灵韵与食物，方便逐行调整。
- `dropTable` 中每个条目都是一次独立 ROLL；`world_drop` 或 `consumablePool` 命中后，再从池内等权随机 `1` 项。
- ROLL 编号按当前文档展示顺序编号，不等同于源码注释序号。
- `100%` 产出仍视为一个掉落槽位；随机装备的品质、部位和词条随机均归在同一个装备 ROLL 内。
- `表内 ROLL` 只统计 `dropTable`。`沙煞酒` 由 `WineData.lua` 在怪物死亡后额外独立判定，单独标为“外部 ROLL”，且玩家获得后不再继续判定。
- 2026-07-11 起，本节对 ROLL 做“类型归类”，但仍保留每个独立 ROLL 的 R1/R2/R3... 展示；分类用于读表和横向对比，不表示合并判定。

### ROLL 类型归类

| 分类 | 判定方式 | 二/三章常见条目 | 展示规则 |

| 外部 ROLL | 不在 `dropTable`，由其它系统额外判定 | `沙煞酒` | 不计入表内 ROLL，单列“外部 ROLL” |

## 六、第三章锻造设计（已落地）

> 本节记录第三章黄沙锻造台当前实现口径，已接入 `EquipmentData_Forge.lua` / `ForgeSystem.lua` / `SwordForgeUI.lua` / `NPCDialog.lua` / 第三章第一寨地图交互。

### 1. 设计目标

- 解决黄沙武器重复掉落后的转换需求，让玩家可以用多余黄沙武器换取缺口武器。
- 参考第一章 `虎王试炼炉`、第二章 `乌堡锻造台` 的独立站点模式：第三章新增专属锻造台交互。
- 本配方要求玩家先装备一把黄沙武器，锻造成功后将当前装备武器原地转化为随机另一把黄沙武器。
- 不新增第三章装备图鉴条目；产物仍是既有黄沙武器，因此沿用现有图鉴、掉落与装备模板。

### 2. 配方定义

| 项目 | 设计值 |
| --- | --- |
| 配方名 | 黄沙换兵 |
| 用户口径 | `黄沙武器（必须装备） + 帝尊叁戒 + 沙海令 ×1000 + 金币100W = 随机另外一个黄沙武器` |
| 配置口径 | `当前装备的黄沙武器 ×1 + 帝尊叁戒 ×1 + 沙海令 ×1000 + 金币 1,000,000 -> 随机另一把黄沙武器 ×1` |

| 输出模式 | 原地替换当前装备栏武器，不保留原武器已有洗练 `forgeStat` |

| 交互路由 | `NPCDialog` 路由到 `SwordForgeUI.Show(npc, "huangsha_forge")` |

### 5. 运行规则

1. 玩家在第三章第一寨 `65,59` 的黄沙锻造台打开锻造界面。
2. 玩家必须先装备任意一把黄沙武器，当前装备武器即为投入武器。
3. 系统检查：
   - 当前装备栏武器属于黄沙武器池；
   - 背包中存在 `帝尊叁戒`；
   - 沙海令不少于 `1,000`；
   - 金币不少于 `1,000,000`。
4. 成功后扣除：背包 `帝尊叁戒` ×1、`沙海令` ×1000、金币 `1,000,000`；当前装备黄沙武器不进背包，而是在装备栏原地转化。
5. 系统从剩余 4 把黄沙武器中等权随机 1 把，将当前装备武器替换为该武器的完整特殊装备属性。
6. 输出绝不等于投入武器；如果候选池异常为空，应中止打造，不扣材料。

### 8. 不采用方案

| 方案 | 不采用原因 |
| --- | --- |
| 直接复用第五章龙神熔炉固定 `equip_weapon + special_equipment` 配方 | 该链路只支持“指定一把输入武器 -> 指定一个目标装备”，不能单配方处理“任意黄沙武器 -> 随机另一把黄沙武器” |
| 背包投入黄沙武器并产物放入背包 | 不符合“当前武器转化为其他随机武器（要求必须装备）”的新口径 |
| 为 5 把黄沙武器拆成 5 个固定配方 | 可以较快实现，但会出现 5 个入口按钮；每个配方仍需新增随机排除自身的 generator，体验不如单配方 |
| 输出包含投入武器 | 违背“随机另外一个黄沙武器”的核心规则 |
| 新增装备模板或新图鉴 | 本配方只是既有黄沙武器互转，不应扩大第三章图鉴与属性总量 |
