# 重击机制与金丹被动调整方案

## 1. 文档目的

本文档用于给其他 AI 或开发者评估、拆解并执行战斗改动。

本次只确定以下四项：

1. 重击基础公式从 `ATK + HeavyHit` 改为 `DEF + HeavyHit`
2. 金丹及以上新增职业被动
3. 戮仙被动从攻击系数计算改为防御系数计算
4. 其余相关设定保持不变，禁止执行时自行扩展

本文档以当前代码为唯一基准，不以旧设计文档为准。

## 3. 最终改动口径

### 3.1 重击公式

所有“重击家族”统一改为：

```lua
HeavyBase = floor(player:GetTotalDef() + player:GetTotalHeavyHit())
HeavyFinal = floor(HeavyBase * (1 + player:GetConstitutionHeavyDmgBonus()))
```

适用范围：

1. 普通重击
2. 裂地重击
3. 龙象功
4. 多区域重击技能
5. 多区域重击技能的连击重算

保持不变：

- 重击概率不变
- 根骨重击伤害加成不变
- 仍然是真伤
- 仍然可暴击
- 事件触发链不变

### 3.2 金丹被动

从 `jindan_1` 开始生效，对应：

- `scripts/config/GameConfig.lua:443`

最终赋值如下（线性公式，最终值追加，不被%二次放大）：

```lua
-- 罗汉：根骨 → 防御（重击公式已改为 DEF 基底，一石二鸟）
finalDef += floor(totalConstitution * 0.40)

-- 太虚 / 剑修：悟性 → 攻击力
finalAtk += floor(totalWisdom * 0.50)

-- 镇岳：体魄 → 生命值
finalMaxHp += floor(totalPhysique * 3.00)
```

### 3.3 戮仙被动

戮仙按用户最终口径执行：

```lua
raw = player:GetTotalDef() * 1.0
```

等价于：

```lua
raw = player:GetTotalDef()
```

保持不变：

- 触发概率 `10%`
- 冷却 `1.0s`
- 仍然是额外普攻，不改为真伤
- 仍然走 `HitResolver.Hit(...)`
- 仍然可暴击
- 仍然可触发普攻特效
- 仍然保留 `isChain=true` 防递归

## 4. 金丹被动的实现要求

### 4.1 生效境界

只在 `jindan_1` 及以上生效。

建议判断方式：

- 使用 `player.realm`
- 对照 `GameConfig.REALMS[player.realm].order`
- 当 `order >= 7` 时生效

对应配置：

- `lianqi_1 = order 1`
- `zhuji_1 = order 4`
- `jindan_1 = order 7`

### 4.2 赋值位置

必须做“最终值追加”，不要回写基础值。

推荐位置：

- 攻击聚合完成后追加太虚被动
- 防御聚合完成后追加罗汉被动
- 生命聚合完成后追加镇岳被动

原因：

- 如果把罗汉被动提前塞进 `defBase` 再去吃现有 `根骨 -> DEF%`，会产生额外放大
- 如果把镇岳被动塞进 `hpBase` 再去吃 `体魄 -> HP%`，会导致多次放大
- 本方案明确要求是“最终值追加”

## 5. 重击改动后的伤害变化

以下为按代码口径推导后的重点结论。

说明：

- 当前 `preset_dps_calc.py` 不是完全代码口径
- 它漏算了部分通用重击链路
- 以下结论以代码逻辑重算为准

### 5.1 不含金丹被动，仅重击公式改动

#### 太虚

- 三章重击 DPS：`239.6 -> 188.6`，`-21.3%`
- 四章重击 DPS：`1158.4 -> 939.8`，`-18.9%`

#### 罗汉

- 三章重击+龙象：`2498.1 -> 2604.6`，`+4.3%`
- 四章重击+龙象：`14398.7 -> 17256.1`，`+19.8%`

#### 镇岳

- 三章重击 DPS：`173.7 -> 145.5`，`-16.2%`
- 四章重击 DPS：`842.1 -> 720.4`，`-14.5%`

### 5.2 含金丹被动

#### 罗汉重点结论

罗汉是本次改动的最大受益者，因为：

1. 根骨本来就转 DEF%
2. 根骨本来就转重击率
3. 根骨本来就转重击伤害
4. 新重击底数又改为 DEF
5. 金丹被动再额外增加 DEF

因此罗汉会出现同属性多次收益。

重算结果：

- 三章重击+龙象：`2498.1 -> 3045.8`，相对旧版 `+8.9%`
- 四章重击+龙象：`14398.7 -> 20963.3`，相对旧版 `+23.4%`

罗汉总 DPS 变化：

- 三章：`5727.2 -> 5975.0`，`+4.3%`
- 四章：`33004.9 -> 36977.7`，`+12.0%`

单次重击/龙象期望伤害：

- 三章：`5082.1 -> 5532.2`
- 四章：`22281.4 -> 27491.3`

### 5.3 前两章风险说明

如果执行时把新重击公式做成“全阶段立刻生效”，而不是只从金丹后进入新体系，则罗汉前两章会先变弱：

- 一章重击+龙象：`-14.2%`
- 二章重击+龙象：`-8.3%`

因此本方案建议：

- 重击公式可以全局改
- 但评估时要明确前两章罗汉会有短期回调

如果不接受前期回调，则应进一步讨论“重击公式是否也从金丹后切换”。

本方案当前不包含该额外机制。

## 6. 戮仙改动后的伤害结论

本次只按用户最终口径：

- 戮仙用 `DEF * 1.0`
- 不额外乘 `1.2`
- 其他设定不动

### 6.1 结论

#### 三章

- 太虚单次戮仙：下降
- 罗汉单次戮仙：上升
- 镇岳单次戮仙：下降

#### 四章

- 太虚单次戮仙：下降
- 罗汉单次戮仙：显著上升
- 镇岳单次戮仙：下降

罗汉近似口径：

- 三章单次戮仙：`1382.8 -> 1551.7`
- 四章单次戮仙：`6060.8 -> 9211.7`

说明：

- 戮仙本质上是额外普攻
- 它不是重击
- 也不是龙象
- 它的改动不会覆盖重击公式，只是独立放大罗汉的防御收益

## 9. 建议的实现方式

建议提炼两个辅助接口，避免同一公式散落多处。

### 9.1 重击基础值

建议新增：

```lua
function Player:GetHeavyHitBaseValue()
    return math.floor(self:GetTotalDef() + self:GetTotalHeavyHit())
end
```

然后所有重击家族统一读这个接口。

### 9.2 金丹被动启用判断

建议新增：

```lua
function Player:IsJindanPassiveUnlocked()
    local realmData = GameConfig.REALMS[self.realm]
    return realmData and (realmData.order or 0) >= 7
end
```

## 11. 线上同步要求

本仓内可见的线上副本伤害处理 `scripts/network/DungeonHandler.lua` 只接收最终伤害上报，并未单独复算戮仙或重击公式。

因此：

1. 本仓代码改动后，客户端/运行包侧会直接改变最终伤害上报值
2. 如果线上存在仓外独立战斗逻辑或战斗服，必须同步同口径公式
3. 若线上只做最终伤害汇总，则无需额外改“服务端重击公式”，但要同步运行包

## 12. 最终执行口径

执行时必须严格按以下值：

```lua
-- 重击
HeavyBase = DEF + HeavyHit

-- 金丹被动（最终值追加，不被%二次放大）
Luohan:  finalDef   += floor(totalConstitution * 0.40)
Taixu:   finalAtk   += floor(totalWisdom * 0.50)
Zhenyue: finalMaxHp += floor(totalPhysique * 3.00)

-- 戮仙
LuxianRaw = DEF * 1.0
```

除此之外，不允许额外调整任何倍率、概率、暴击规则或真伤规则。
