## 详细说明

### Task 1：RedeemHandler `serverCloud` 判空保护

**问题根因**：`RegisterRedeemCodesFromConfig()` 内部直接调用 `serverCloud:Set()`，若服务端云变量未初始化（`serverCloud = nil`），会导致 nil 指针崩溃，进而阻断服务端启动。

**修复位置**：`scripts/network/RedeemHandler.lua` → `RegisterRedeemCodesFromConfig()` 函数顶部

**修复内容**：
```lua
function M.RegisterRedeemCodesFromConfig()
    -- P0 止血：serverCloud 未就绪时跳过注册，不阻断服务端启动
    if not serverCloud then
        print("[Server][Redeem][WARN] serverCloud not available, skip RegisterRedeemCodesFromConfig.")
        return
    end
    -- ... 原有逻辑
end
```

**效果**：`serverCloud` 不可用时打印 WARN 日志并提前返回，不再崩溃。

### Task 2：LootSystem 惰加载

**问题根因**：`BlackMerchantHandler.lua` 和 `XianyuanChestHandler.lua` 在文件顶层使用硬依赖 `require("systems.LootSystem")`，若 LootSystem 存在语法错误或运行时异常，会导致整个 Handler 加载失败，进一步导致服务端启动中断。

**修复模式**：缓存式惰加载（Cached Lazy Load）

#### BlackMerchantHandler.lua

删除顶层 `require`，改为：
```lua
-- 删除：local LootSystem = require("systems.LootSystem")

local _lootSystemCache_ = nil
local function GetLootSystem()
    if _lootSystemCache_ then return _lootSystemCache_ end
    local ok, mod = pcall(require, "systems.LootSystem")
    if ok and mod then
        _lootSystemCache_ = mod
        return mod
    end
    print("[BlackMerchant][ERROR] LootSystem load failed: " .. tostring(mod))
    return nil
end
```

调用点（2处）改造：
- **Buy handler**：`local LS = GetLootSystem()` + nil 判断，失败时返回"系统繁忙"错误响应
- **WAL 补偿**：`LS and LS.CreateSpecialEquipment(...)` 安全调用

#### XianyuanChestHandler.lua

同样模式，唯一调用点（CompleteOpen handler）：
```lua
local LS = GetLootSystem()
if not LS then
    SendError(connection, SaveProtocol.S2C_XianyuanChest_Reward, "系统繁忙，请稍后再试")
    return
end
items = LS.GenerateXianyuanReward(cfg.tier, cfg.quality, cfg.attr)
```

### Task 3：server_main.lua 四模块改 pcall 加载

**问题根因**：`RedeemHandler`、`BlackMerchantHandler`、`XianyuanChestHandler`、`SkinShopHandler` 为可选业务模块，任一模块加载失败会导致 server_main.lua 整体 require 失败，服务端无法启动。

**改造范围（四处）**：

#### ① 顶层 require → pcall

```lua
local _rhOk, RedeemHandler = pcall(require, "network.RedeemHandler")
if not _rhOk then
    print("[Server][WARN] RedeemHandler load failed: " .. tostring(RedeemHandler))
    RedeemHandler = nil
end
-- 其余三个模块同样改造
```

#### ② 事件订阅加 if 守卫

```lua
if RedeemHandler then
    RateLimitedSubscribe(SaveProtocol.C2S_RedeemCode, "HandleRedeemCode")
    RateLimitedSubscribe(SaveProtocol.C2S_GMCreateCode, "HandleGMCreateCode")
else
    print("[Server][WARN] RedeemHandler unavailable, skipping Redeem subscriptions.")
end
-- BlackMerchantHandler / SkinShopHandler / XianyuanChestHandler 同样守卫
```

#### ③ RegisterRedeemCodesFromConfig 调用加守卫

```lua
if RedeemHandler then
    RedeemHandler.RegisterRedeemCodesFromConfig()
end
```

#### ④ 14 个全局 handler 包装函数加判空

示例：
```lua
function HandleGMCreateCode(eventType, eventData)
    if RedeemHandler then RedeemHandler.HandleGMCreateCode(eventType, eventData) end
end
function HandleBlackMerchantQuery(eventType, eventData)
    if BlackMerchantHandler then BlackMerchantHandler.HandleQuery(eventType, eventData) end
end
```

涵盖函数列表：
- RedeemHandler：`HandleGMCreateCode`、`HandleRedeemCode`（2个）
- BlackMerchantHandler：`HandleBlackMerchantQuery`、`HandleBlackMerchantBuy`、`HandleBlackMerchantRefresh`、`HandleBlackMerchantSell`、`HandleBlackMerchantSpecialBuy`（5个）
- SkinShopHandler：`HandleSkinShopQuery`、`HandleSkinShopBuy`（2个）
- XianyuanChestHandler：`HandleXianyuanChestQuery`、`HandleXianyuanChestOpen`、`HandleXianyuanChestComplete`、`HandleXianyuanChestClaim`（4个）

**结合已有改造**（DungeonHandler / EventHandler 之前已用 pcall，本次保持不变）

### Task 4：bootSessionId 补充到启动日志

**目的**：每次服务端启动生成唯一 ID，方便在日志中关联同一实例的所有初始化信息，区分重启前后的日志段落。

**生成方式**：时间戳 + 随机四位数（`{timestamp}_{rand}`）

**修复位置**：`scripts/server_main.lua` → `Start()` 函数开头

**修复内容**：
```lua
-- P0 止血：生成本次启动唯一 ID，便于日志关联同一实例的所有 boot 信息
local _bootTs = os.time and os.time() or 0
math.randomseed(_bootTs + os.clock() * 1000)
local bootSessionId = tostring(_bootTs) .. "_" .. tostring(math.random(1000, 9999))

print("[Server-BOOT][" .. bootSessionId .. "] Start() called at " .. tostring(_bootTs))
-- ...
print("[Server-BOOT][" .. bootSessionId .. "] Before SaveProtocol.RegisterAll()")
SaveProtocol.RegisterAll()
print("[Server-BOOT][" .. bootSessionId .. "] After SaveProtocol.RegisterAll()")
```

**日志格式变更**：

| 修改前 | 修改后 |
|--------|--------|
| `[Server-BOOT] Start() called at 1748...` | `[Server-BOOT][1748..._2341] Start() called at 1748...` |
| `[Server-BOOT] Before SaveProtocol.RegisterAll()` | `[Server-BOOT][1748..._2341] Before SaveProtocol.RegisterAll()` |
| `[Server-BOOT] After SaveProtocol.RegisterAll()` | `[Server-BOOT][1748..._2341] After SaveProtocol.RegisterAll()` |
