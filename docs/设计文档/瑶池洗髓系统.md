## 一、系统概述

**瑶池洗髓**是一个**角色级**养成系统，玩家在瑶池湖心亭将丹药化为洗髓灵液，积满后进行洗髓淬炼提升修炼等级。每级修炼提供 **+1% 独立最终增减伤**，最高 26 级（+26%）。

### 1.1 交互位置

| 属性 | 说明 |
|------|------|
| 位置 | 瑶池湖心亭（3×3 岛屿中心） |
| 交互类型 | 点击湖心亭交互点触发 |
| 入口条件 | 到达中洲（章节 101）即可 |

### 1.2 核心循环

```
丹药化为灵液 → 获得修炼点数 → 点数积满 → 洗髓淬炼（20秒读条）→ 修炼等级+1 → 增伤减伤+1%
```

### 1.3 修炼等级与境界绑定

**每个小段位只能提升 1 级修炼**，修炼等级上限 = 当前角色境界对应的小段位序号。

例如：
- 练气初期（order=1）→ 修炼上限 1 级
- 筑基后期（order=6）→ 修炼上限 6 级
- 渡劫巅峰（order=26）→ 修炼上限 26 级（满级）

玩家必须先突破到对应境界，才能继续提升修炼等级。

## 二、丹药与洗髓灵液

### 2.1 丹药说明

所有境界突破丹药**不可出售**。突破境界后多余的丹药，可在瑶池湖心亭化为洗髓灵液，用于修炼洗髓。

> *"天地灵药，不可轻弃。瑶池灵水可炼化多余丹药为洗髓灵液，以灵液蕴养肉身、洗炼筋骨。"*

### 2.2 丹药灵液转化表

每种丹药化为灵液后获得的修炼点数，与其灵韵价值挂钩（1 灵韵 = 1 修炼点数）：

| 丹药 | 灵韵价值 | 每颗修炼点数 | 获取途径 |
|------|---------|------------|---------|
| 练气丹 `qi_pill` | 10 灵韵 | 10 点 | 第一章炼制 |
| 筑基丹 `zhuji_pill` | 100 灵韵 | 100 点 | 第二章炼制/Boss |
| 金丹砂 `jindan_sand` | 100 灵韵 | 100 点 | 第二章 Boss 掉落 |
| 元婴果 `yuanying_fruit` | 100 灵韵 | 100 点 | 第三章获取 |
| 九转金丹 `jiuzhuan_jindan` | 500 灵韵 | 500 点 | 第四章炼制 |

### 2.3 操作规则

- 每次可选择投入任意种类、任意数量的丹药
- 投入后丹药转化为灵液（修炼点数），立即扣除丹药，**立即存档**
- 点数累计存储，不会因退出丢失
- 当累计点数 ≥ 当前等级所需点数时，可进行洗髓淬炼
- 当修炼等级已达当前境界上限时，无法进行淬炼（需先突破境界）
- **九转金丹及以上**（九转金丹、渡劫丹）化为灵液时，需**二次确认弹窗**：
  > "确定要将 N 颗九转金丹化为灵液吗？此操作不可逆。"

## 四、洗髓淬炼

### 4.1 淬炼流程

1. 玩家在湖心亭交互，打开洗髓面板
2. 投入丹药化为洗髓灵液（可多次投入累计）
3. 灵液满后，"洗髓淬炼"按钮亮起
4. 点击"洗髓淬炼"，展示配图，下方 **20 秒读条**（不可移动，可取消）
5. 读条完成，修炼等级 +1，清空当前等级已消耗点数，**立即存档**
6. 溢出的点数**保留**，自动累积到下一级

### 4.2 淬炼演出

- 点击"洗髓淬炼"后弹出**全屏淬炼画面**
- 画面上方：AI 生成的瑶池仙境沐浴配图（香艳仙气风格）
- 画面下方：读条进度条 + 剩余秒数
- 读条时长**固定 20 秒**，所有等级统一
- 读条完成后播放提升动效，显示等级变化

### 4.3 读条规则

- 读条期间不可移动、不可攻击
- 受到攻击不会中断（瑶池为安全区）
- 主动取消不消耗任何资源（丹药在投入时已消耗）

### 4.4 境界限制

- 修炼等级不得超过当前境界对应的上限
- 当达到境界上限时，面板显示"需突破 XX 境界"
- 突破境界后自动解锁下一级修炼

## 五、战斗效果

### 5.1 独立乘区增减伤（最终伤害）

修炼等级提供的增伤减伤是**独立乘区的最终伤害加成**，与现有装备减伤（`equipDmgReduce`，加算，上限 75%）分开计算，作用于伤害计算的最后一步。

**受伤公式（完整链路）**：
```
基础伤害 = CalcDamage(ATK, DEF)           -- ATK²/(ATK+DEF) × random(0.9,1.1)
暴击后伤害 = 基础伤害 × 暴击倍率            -- 如果暴击
装备减伤后 = 暴击后伤害 × (1 - min(装备减伤, 0.75))   -- 加算，上限75%
最终伤害 = 装备减伤后 × (1 - 修炼减伤)      -- 独立乘区，无上限（最高26%）
```

**示例**（修炼 10 级 + 装备减伤 50%）：
```
最终伤害 = 100 × (1 - 0.50) × (1 - 0.10) = 100 × 0.50 × 0.90 = 45
```

**攻击增伤公式**：
```
最终攻击伤害 = CalcDamage输出 × (1 + 修炼增伤)  -- 独立乘区，最高+26%
```

**关键区别**：
| 属性 | 类型 | 计算方式 | 上限 |
|------|------|---------|------|
| 装备减伤 `equipDmgReduce` | 加算区 | 所有装备减伤值相加 | 75% |
| 修炼减伤 `washLevel × 1%` | 独立乘区 | 与装备减伤相乘（串联） | 26% |
| 修炼增伤 `washLevel × 1%` | 独立乘区 | 作用于最终输出伤害 | 26% |

### 5.2 实现位置

在 `Player.lua` 的 `TakeDamage` 函数中（约 L626），于装备减伤之后插入修炼乘区：

```lua
-- 现有装备减伤（加算，上限 75%）— 已有逻辑
local dmgReduce = self.equipDmgReduce or 0
if dmgReduce > 0 then
    local reduction = math.min(dmgReduce, 0.75)
    damage = math.max(1, math.floor(damage * (1 - reduction)))
end

-- [新增] 修炼独立乘区减伤（无上限，最高 26%）
local washLevel = self:GetWashLevel()
if washLevel > 0 then
    local washReduce = washLevel * 0.01  -- 每级 1%
    damage = math.max(1, math.floor(damage * (1 - washReduce)))
end
```

攻击增伤在 `CalcDamage` 输出后叠加：

```lua
-- [新增] 修炼增伤
local washLevel = attacker:GetWashLevel()
if washLevel > 0 then
    damage = math.floor(damage * (1 + washLevel * 0.01))
end
```

### 5.3 属性校验

| 校验项 | 规则 | 校验时机 |
|--------|------|---------|
| washLevel 范围 | `0 ≤ washLevel ≤ 26` | 加载存档时 `math.max(0, math.min(26, level))` |
| washLevel ≤ 境界上限 | 不得超过当前境界 order | 淬炼前检查 + 加载存档时钳位 |
| washPoints ≥ 0 | 不允许负数 | 加载存档时 `math.max(0, points)` |
| 减伤叠加合理性 | 装备减伤(≤75%) × 修炼减伤(≤26%) 最高减免 81.5% | 仅文档记录，无需代码限制 |

**加载存档时的防御性校验**：
```lua
function Player:LoadWashData(saveData)
    local wash = saveData.yaochi_wash or { level = 0, points = 0 }
    self.washLevel = math.max(0, math.min(26, wash.level or 0))
    self.washPoints = math.max(0, wash.points or 0)
end
```

## 七、数据存储

### 7.1 存储结构

洗髓数据为**角色级**，存储在角色存档（SaveSlot）中：

```lua
-- 存档位置：saveData.yaochi_wash
saveData.yaochi_wash = {
    level = 0,           -- 当前修炼等级（0~26）
    points = 0,          -- 当前累计修炼点数
}
```

### 7.2 存储位置

角色存档 SaveSlot，通过 `CloudStorage` 按存档位持久化（与角色属性、装备等同级）。

**注意**：不使用 `account_atlas`（那是账号级仙图录）。洗髓是角色级系统，每个角色独立修炼。

### 7.3 存档迁移

在 `SaveMigrations.lua` 中新增迁移函数，确保旧存档加载时自动初始化：

```lua
-- 版本 N+1: 新增瑶池洗髓字段
MIGRATIONS[NEXT_VERSION] = function(data)
    if not data.yaochi_wash then
        data.yaochi_wash = { level = 0, points = 0 }
    end
    return data
end
```

### 7.4 Player 字段对接

`Player.lua` 加载存档时读取修炼数据：

```lua
function Player:LoadWashData(saveData)
    local wash = saveData.yaochi_wash or { level = 0, points = 0 }
    self.washLevel = wash.level
    self.washPoints = wash.points
end

function Player:GetWashLevel()
    return self.washLevel or 0
end

function Player:GetWashPoints()
    return self.washPoints or 0
end
```

3. 灵液满后，"洗髓淬炼"按钮亮起
4. 点击"洗髓淬炼"→ 弹出淬炼画面 → 20 秒读条 → 等级提升
5. 若已达境界上限，按钮显示"需突破 XX 境界"

## 十、淬炼配图资源

### 10.1 配图需求

| 属性 | 说明 |
|------|------|
| 风格 | 仙侠国风，香艳而不低俗，仙气飘渺 |
| 场景 | 瑶池仙境，灵气萦绕的湖泊，人物沐浴于灵液之中 |
| 色调 | 柔和的蓝紫色调 + 金色灵光点缀 |
| 尺寸 | 竖屏适配，建议 9:16 或 3:4 |
| 生成 | 使用 AI 生图工具（generate_image）生成 |

### 10.2 读条时长

**固定 20 秒**，所有等级统一，不随修炼等级变化。

```lua
local RITUAL_DURATION = 20  -- 秒，固定值
```
