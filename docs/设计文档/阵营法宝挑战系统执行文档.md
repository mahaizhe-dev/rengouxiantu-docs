## 一、当前结论

当前阵营法宝挑战为：

- `4` 个阵营
- `R1 ~ R9` 共 `9` 级声望
- 法宝覆盖 `T3 ~ T11`
- `T11` 在 UI 中显示为 `仙1阶`
- 每次入场固定消耗当前等级令牌 `10` 个
- 首通生成两件固定品质法宝二选一
- 重复通关生成两件随机品质法宝
- 额外奖励命中时作为第 `3` 个候选，最终仍只选一件

R9 固定规格：

| 项目 | 实现 |
|---|---|
| Boss 等级 | `140` |
| Boss 境界 | `renxian_1` / 人仙1阶 |
| Boss 类别 | `saint_boss` |
| 法宝 Tier | `T11` / 仙1阶 |
| 解锁令牌 | `zhexian_ling` / 谪仙令 |
| 解锁消耗 | `1000` |
| 每次入场 | `10` 谪仙令 |
| 首通品质 | `cyan` |
| 重复品质 | `blue ~ red` |
| 额外候选总概率 | `9.0%` |

## 四、奖励实现

### 4.1 首通

首通生成两件：

- 同阵营
- 同 Tier
- 同首通保底品质
- 独立副属性实例

R9 首通为两件 `T11 orange` 法宝二选一；重复通关仍在 `blue ~ red` 中随机。

### 4.2 重复通关

重复通关：

1. 按当前声望品质范围抽取法宝 A。
2. 独立抽取法宝 B。
3. 判定是否出现额外奖励。
4. 未命中时二选一；命中时法宝 A、法宝 B、额外道具三选一。

R9 每件法宝独立在 `blue ~ red` 中抽取，不会生成白、绿，也不会超过红。

## 六、历史稳定难度对比

| 内容 | 等级/类别 | 常规技能 | 阶段 | 场地百分比伤害 | 主要机制密度 |
|---|---|---:|---:|---:|---|
| R7 | 100 皇级 | 3 | 3 | `40%~60%` | 1 个阵营机制 |
| R8 | 120 圣级 | 3 | 3 | `40%~60%` | 1 个强化阵营机制 |
| 第五章四仙剑 | 120 圣级 | 4 | 3 | `45%` | 4 常规 + 阶段追加 |
| 第六章呱大人 | 140 人仙1圣级 | 4 | 3 | `42%` | 多段毒、减速、阶段追加 |
| 第六章魔君蚀玄 | 140 人仙1圣级 | 4 | 3 | `46%` | 多段裂痕、减速、短眩晕 |

R9 的控制原则：

- 不增加第四阶段。
- 不增加第二个全场百分比技能。
- 不增加召唤物、寻路单位、移动安全区或真正的囚笼边界。
- 每名 BOSS 的 R9 新技能只在 P2/P3 生效；凌战同时保留原有 P2 拉扯。
- R9 常驻强化与 R8 一致：攻速 `1.4`、移速 `1.4`、CDR `0.6`。
- R9 新技能在常驻 CDR 后的实际冷却为 `8.4~9.0s`。
- 四名 BOSS 的转场技能伤害统一为 `50%` 最大生命值。
- R2-R9 阶段血线统一为 P2 `60%`、P3 `30%`；R1 保持单阶段。
- R9 二阶段与 R8 一致：不额外提高攻击，移速 `1.4`、普攻间隔 `0.6`。
- R9 三阶段与 R8 一致：攻击 `2.5`、移速 `1.6`、普攻间隔 `0.35`。

R8 与 R9 使用完全相同的阶段和狂暴系数。R9 因等级、境界、基础生命、攻防和 P2 新技能更高，整体难度自然高于 R8，不再通过单独降低 R9 通用阶段参数进行补偿。

## 八、性能与风险

### 性能

- 每个 R9 BOSS 只多一个技能定义和一个技能计时器，成本可忽略。
- 新技能复用现有 `monster_skill` 事件、形状判定和状态效果。
- 没有新增召唤物、寻路、碰撞体、常驻地面对象或额外空间索引。
- 玉甲更新沿用已有单次怪物列表扫描；挑战场内通常只有一个 BOSS。
- 玉甲视觉只绘制一个圆环与倒计时，属于 O(1) 绘制。

### 风险

1. R9 基础 HP 约为 R8 的 `2.9` 倍、攻防约为 `2.1` 倍。
   - 常驻狂暴为 `1.4/1.4/0.6`，三阶段攻击/间隔为 `2.5/0.35`；沈墨额外保留50%血海与无上限血印，并将血影追命提高到3倍攻击。
2. 陆青云结界永久存在，长时间战斗仍可能压缩场地。
   - R9 不增加数量、半径和生成频率。
3. 云裳玉甲此前未生效，本轮属于真实行为变化。
   - 已显著降低旧配置中的减伤和反伤，并提供持续视觉。
4. R8 历史占位技能仍存在。
   - 本轮不改历史战斗，只保证 R9 不再依赖占位技能。
5. BOSS 死亡到玩家领取奖励之间曾存在残留伤害窗口。
   - 现已在胜利事件前停止战斗并清理挑战状态，避免胜利后再次触发失败退出。
