## 一、概述

角色创建完成后，玩家坠入虚空，接受五道"天问"——以极简写意的哲学二选一，决定初始属性加成。

**设计理念**：不是"你遇到一只兔子怎么办"的具体场景题，而是天道对灵魂的叩问。没有对错，只有道途。

### 核心参数

| 项目 | 值 |
|------|---|
| 触发时机 | 新角色：创角完成后自动触发；老角色：两界村 NPC 手动触发 |
| 题目数量 | 5 道 |
| 每题选项 | 2 选 1 |
| 路径总数 | 32 条（2⁵） |
| 唯一属性组合 | 30 种 |
| 仙缘加成 | 每次 +5 点 |
| 基础属性加成 | 攻击 +5 或 生命 +30 |
| 数据层级 | **角色级**（同账号多角色独立） |
| 重置入口 | 两界村·玄微 NPC |
| 重置费用 | 灵韵 ×500 |

## 五、表现设计（实现规格）

> **核心原则**：极简、写意、仪式感。不是 UI 界面，是天人对话。
> **技术方案**：全程 NanoVG 渲染，不使用 UI 库组件。**复用主 NanoVG 上下文**（不新建独立 Context），在 `NanoVGRender` 事件中以全屏强模态覆盖绘制。问心激活期间，先画满屏虚空黑幕遮盖底层场景，再在其上绘制所有问心内容，退出时淡出黑幕露出底层。这样避免多 Context 的生命周期管理和渲染排序问题。

### 5.1 状态机

问心全流程由一个线性状态机驱动，每个状态有明确的持续时间和过渡条件：

```
ENTER_VOID          坠入虚空（画面从当前场景渐暗到纯黑）
   ↓
VOID_IDLE           虚空静默（纯黑 + 星尘背景就绪）
   ↓
QUESTION_INTRO      引语逐字浮现
   ↓
OPTIONS_APPEAR      两个选项从两侧凝聚
   ↓
WAIT_SELECT         等待玩家点击（无限等待）
   ↓
SELECTED            选中反馈动画
   ↓
FADE_OUT            全部元素淡出，回归虚空
   ↓
├─ 还有下一问 → QUESTION_INTRO（循环 5 次）
└─ 第五问完成 → FINALE_CONVERGE
   ↓
FINALE_LIGHT        光柱注入动画
   ↓
EXIT_VOID           画面渐亮，退出问心
```

### 5.2 状态时序表

| 状态 | 持续时间 | 缓动函数 | 说明 |
|------|---------|---------|------|
| `ENTER_VOID` | 1.5s | `easeInCubic` | 画面 alpha 从 0→1 覆盖黑幕；同时星尘粒子开始生成 |
| `VOID_IDLE` | 1.0s | — | 纯等待，让玩家感受虚空；仅首次进入时播放，后续题目跳过 |
| `QUESTION_INTRO` | 逐字 0.12s/字 + 尾停 0.6s | `easeOutCubic`（每字 alpha） | 引语文字逐字浮现；"——"后的破折号作为整体一次性出现 |
| `OPTIONS_APPEAR` | 0.8s | `easeOutBack` | 两选项从屏幕两侧（左/右各偏移 120px）滑入最终位置，同时 alpha 0→1 |
| `WAIT_SELECT` | 无限 | — | 等待点击；选项有呼吸光晕（2.5s 周期 `sin` 波） |
| `SELECTED` | 1.2s | — | 分三段：选中项 bloom 爆发(0~0.4s)，未选项淡出(0.2~0.8s)，引语淡出(0.4~1.0s) |
| `FADE_OUT` | 0.8s | `easeInCubic` | 选中项残光收缩消散；全局回归纯黑 |
| `FINALE_CONVERGE` | 2.0s | `easeInOutCubic` | 五道残光从屏幕五个记忆点向中心汇聚 |
| `FINALE_LIGHT` | 1.5s | `easeOutExpo` | 中心光柱从细线爆发至全屏高度，亮度脉冲 |
| `EXIT_VOID` | 1.2s | `easeOutCubic` | 黑幕 alpha 1→0，露出游戏场景 |

**单题总耗时**（不含等待）：约 `0.12×4字 + 0.6 + 0.8 + 1.2 + 0.8 ≈ 3.9s`（引语 4 字时）
**五题总耗时**（不含等待）：约 `1.5 + 1.0 + 3.9×5 + 2.0 + 1.5 + 1.2 ≈ 26.7s`

### 5.5 逐字浮现动画（引语）

由于 `nvgTextGlyphPositions` 未暴露到 Lua 侧，使用 **逐字符渲染** 方案：

```lua
--- 将 UTF-8 字符串拆分为字符数组
local function utf8Chars(str)
    local chars = {}
    local i = 1
    while i <= #str do
        local byte = string.byte(str, i)
        local charLen = 1
        if byte >= 0xF0 then charLen = 4
        elseif byte >= 0xE0 then charLen = 3
        elseif byte >= 0xC0 then charLen = 2
        end
        chars[#chars + 1] = string.sub(str, i, i + charLen - 1)
        i = i + charLen
    end
    return chars
end

--- 绘制逐字浮现文本
--- @param elapsed  number  当前状态已经过时间
--- @param text     string  要显示的引语（如 "道法万千——"）
--- @param cx       number  中心 X
--- @param cy       number  中心 Y
local function drawIntroText(elapsed, text, cx, cy)
    local chars = utf8Chars(text)
    local charInterval = 0.12          -- 每字间隔
    local charFadeDur  = 0.25          -- 单字 alpha 淡入时长

    -- 先测量总宽度以实现居中
    nvgFontSize(vg, 28)
    nvgTextLetterSpacing(vg, 6)
    local totalW = 0
    local charWidths = {}
    for i, ch in ipairs(chars) do
        local adv = nvgTextBounds(vg, 0, 0, ch)
        charWidths[i] = adv
        totalW = totalW + adv
    end
    -- 字间距补偿：(n-1) 个间距已包含在 letterSpacing 中，
    -- nvgTextBounds 返回值已含 letterSpacing，此处直接累加即可

    local startX = cx - totalW * 0.5
    local x = startX

    for i, ch in ipairs(chars) do
        local charStart = (i - 1) * charInterval
        local t = math.max(0, math.min(1, (elapsed - charStart) / charFadeDur))
        -- easeOutCubic
        local u = 1 - t
        local alpha = 1 - u * u * u

        if alpha > 0.001 then
            -- 微弱上浮效果：从下方 8px 浮入
            local offsetY = (1 - alpha) * 8

            nvgFillColor(vg, nvgRGBA(
                C_INTRO_TEXT[1], C_INTRO_TEXT[2], C_INTRO_TEXT[3],
                math.floor(alpha * 220)
            ))
            nvgText(vg, x, cy + offsetY, ch, nil)
        end

        x = x + charWidths[i]
    end
end
```

**关键细节**：
- 每个字符独立 alpha + Y 偏移，产生"从虚无中凝聚"的效果
- `charInterval = 0.12s` 使 4 字引语共需 `0.12×3 + 0.25 = 0.61s`
- 破折号"——"视为 2 个字符，但同时出现（共享同一个 `charStart`）

### 5.6 选项出现与呼吸动画

**出现动画**（`OPTIONS_APPEAR` 状态，0.8s）：

```lua
local function drawOptions(elapsed, optA, optB, optAx, optBx, cy)
    local t = math.min(1, elapsed / 0.8)
    -- easeOutBack: 轻微过冲，增加"凝聚弹性"
    local c1 = 1.70158
    local c3 = c1 + 1
    local u = t - 1
    local eased = 1 + c3 * u * u * u + c1 * u * u

    -- 从两侧滑入
    local slideOffset = 120 -- 初始偏移像素
    local axCurrent = optAx - slideOffset * (1 - eased)  -- 左选项从更左滑入
    local bxCurrent = optBx + slideOffset * (1 - eased)  -- 右选项从更右滑入
    local alpha = math.floor(eased * 240)

    nvgFontSize(vg, 40)
    nvgTextLetterSpacing(vg, 8)
    nvgTextAlign(vg, NVG_ALIGN_CENTER + NVG_ALIGN_MIDDLE)

    -- 选项 A
    nvgFillColor(vg, nvgRGBA(C_OPTION_NORMAL[1], C_OPTION_NORMAL[2], C_OPTION_NORMAL[3], alpha))
    nvgText(vg, axCurrent, cy, optA, nil)

    -- 选项 B
    nvgFillColor(vg, nvgRGBA(C_OPTION_NORMAL[1], C_OPTION_NORMAL[2], C_OPTION_NORMAL[3], alpha))
    nvgText(vg, bxCurrent, cy, optB, nil)
end
```

**呼吸光晕**（`WAIT_SELECT` 状态，持续脉动直到点击）：

```lua
local function drawOptionGlow(time, x, y, text)
    -- 呼吸周期 2.5s，柔和正弦波
    local breath = 0.5 + 0.5 * math.sin(time * math.pi * 2 / 2.5)

    -- 文字后方的 bloom 光晕（先画光晕再画文字）
    local glowRadius = 60 + breath * 20
    local glowAlpha = 0.04 + breath * 0.06

    nvgBeginPath(vg)
    nvgCircle(vg, x, y, glowRadius)
    local grad = nvgRadialGradient(vg, x, y, 10, glowRadius,
        nvgRGBAf(C_OPTION_GLOW[1]/255, C_OPTION_GLOW[2]/255, C_OPTION_GLOW[3]/255, glowAlpha),
        nvgRGBAf(C_OPTION_GLOW[1]/255, C_OPTION_GLOW[2]/255, C_OPTION_GLOW[3]/255, 0))
    nvgFillPaint(vg, grad)
    nvgFill(vg)

    -- 文字本体
    nvgFillColor(vg, nvgRGBA(C_OPTION_NORMAL[1], C_OPTION_NORMAL[2], C_OPTION_NORMAL[3], 240))
    nvgFontSize(vg, 40)
    nvgTextAlign(vg, NVG_ALIGN_CENTER + NVG_ALIGN_MIDDLE)
    nvgText(vg, x, y, text, nil)
end
```

### 5.7 点击检测

选项点击判断使用 **矩形热区**（不依赖 UI 系统）：

```lua
--- 判断点击是否命中选项
--- @return "A" | "B" | nil
local function hitTestOptions(mx, my, optAx, optBx, cy)
    local hitW, hitH = 180, 70    -- 热区尺寸（逻辑像素），比文字范围更大以提升手感
    -- 选项 A
    if mx >= optAx - hitW/2 and mx <= optAx + hitW/2
       and my >= cy - hitH/2 and my <= cy + hitH/2 then
        return "A"
    end
    -- 选项 B
    if mx >= optBx - hitW/2 and mx <= optBx + hitW/2
       and my >= cy - hitH/2 and my <= cy + hitH/2 then
        return "B"
    end
    return nil
end
```

在 `HandleUpdate` 中检测 `input:GetMouseButtonPress(MOUSEB_LEFT)` 后调用此函数。
坐标转换：`mx, my = input.mousePosition.x / dpr, input.mousePosition.y / dpr`（模式 B 分辨率下）。

### 5.8 选中反馈动画（`SELECTED` 状态，1.2s）

选中后分三层并行动画：

#### 层 1：选中项 Bloom 爆发（0 ~ 0.4s）

```lua
local function drawSelectBloom(elapsed, x, y)
    local t = math.min(1, elapsed / 0.4)
    -- easeOutExpo: 快速爆发后缓慢衰减
    local eased = 1 - math.pow(2, -10 * t)

    local radius = 30 + eased * 200          -- 从 30px 扩散到 230px
    local alpha = 0.5 * (1 - t)              -- 从 0.5 衰减到 0

    nvgBeginPath(vg)
    nvgCircle(vg, x, y, radius)
    local grad = nvgRadialGradient(vg, x, y, radius * 0.1, radius,
        nvgRGBAf(C_SELECT_BLOOM[1]/255, C_SELECT_BLOOM[2]/255, C_SELECT_BLOOM[3]/255, alpha),
        nvgRGBAf(C_SELECT_BLOOM[1]/255, C_SELECT_BLOOM[2]/255, C_SELECT_BLOOM[3]/255, 0))
    nvgFillPaint(vg, grad)
    nvgFill(vg)
end
```

#### 层 2：选中项文字闪白后恢复金色（0 ~ 0.6s）

```lua
local function drawSelectedText(elapsed, x, y, text)
    local t = math.min(1, elapsed / 0.6)
    -- 前 0.15s 闪白，之后渐变为金色
    local flashT = math.min(1, elapsed / 0.15)
    local r = math.floor(255 - flashT * (255 - C_SELECT_FLASH[1]))
    local g = math.floor(255 - flashT * (255 - C_SELECT_FLASH[2]))
    local b = math.floor(255 - flashT * (255 - C_SELECT_FLASH[3]))

    nvgFontSize(vg, 40)
    nvgTextAlign(vg, NVG_ALIGN_CENTER + NVG_ALIGN_MIDDLE)
    nvgFillColor(vg, nvgRGBA(r, g, b, 255))
    nvgText(vg, x, y, text, nil)
end
```

#### 层 3：未选项文字淡出（0.2s ~ 0.8s）

```lua
local function drawUnselectedFade(elapsed, x, y, text)
    local fadeStart = 0.2
    local fadeDur = 0.6
    local t = math.max(0, math.min(1, (elapsed - fadeStart) / fadeDur))
    -- easeInCubic: 缓慢开始，加速消失
    local eased = t * t * t
    local alpha = math.floor((1 - eased) * 180)

    if alpha > 0 then
        -- 同时上浮飘散
        local driftY = eased * -30

        nvgFontSize(vg, 40)
        nvgTextAlign(vg, NVG_ALIGN_CENTER + NVG_ALIGN_MIDDLE)
        nvgFillColor(vg, nvgRGBA(C_UNSELECT_FADE[1], C_UNSELECT_FADE[2], C_UNSELECT_FADE[3], alpha))
        nvgText(vg, x, y + driftY, text, nil)
    end
end
```

#### 层 4：引语淡出（0.4s ~ 1.0s）

引语在选中后延迟 0.4s 开始淡出，0.6s 完成。淡出方式同未选项（alpha 衰减 + 微弱上浮 15px）。

### 5.9 水墨云气背景层（PNG 纹理）

在纯色虚空与星尘粒子之间，加入一层极淡的水墨云气纹理，弥补纯 NanoVG 渐变无法表达的晕染质感。

#### 资源规格

| 文件 | 路径 | 尺寸 | 说明 |
|------|------|------|------|
| `void_ink_cloud.png` | `assets/Textures/void_ink_cloud.png` | 1024×1024 | 深灰/深紫色调水墨纹理，大面积留黑，仅边缘有隐约墨痕；无缝平铺 |

**制作要求**：
- 色调：以 `C_VOID`（10, 8, 18）为底，墨痕颜色约 `(30, 25, 45)` ~ `(50, 40, 70)`
- 留黑 ≥ 70%：大部分区域是纯透明/纯黑，仅局部有云气晕染
- 格式：PNG，支持 alpha 通道；墨痕区域 alpha 约 80~150（最终渲染时再乘以 `nvgGlobalAlpha`）
- 无缝拼接：四边可平铺（供 `NVG_IMAGE_REPEATX + NVG_IMAGE_REPEATY` 使用）

#### 渲染实现

绘制顺序：**虚空底色 → 水墨云气 → 星尘粒子 → 文字/选项**

```lua
-- 加载（Start 中执行一次）
local inkImg = nvgCreateImage(vg, "Textures/void_ink_cloud.png",
    NVG_IMAGE_REPEATX + NVG_IMAGE_REPEATY)

-- 渲染（每帧，在星尘粒子之前）
local function drawInkCloudBg(time, cx, cy, screenW, screenH)
    nvgSave(vg)

    -- 极慢旋转：约 7 分钟一圈，产生"气流感"
    nvgTranslate(vg, cx, cy)
    nvgRotate(vg, time * 0.015)
    nvgTranslate(vg, -cx, -cy)

    -- 极低不透明度，仅增加底层质感
    nvgGlobalAlpha(vg, 0.08)

    -- 覆盖范围略大于屏幕（旋转时不露边）
    local margin = 256
    local paint = nvgImagePattern(vg,
        -margin, -margin,
        1024, 1024,
        0, inkImg, 1.0)

    nvgBeginPath(vg)
    nvgRect(vg, -margin, -margin, screenW + margin * 2, screenH + margin * 2)
    nvgFillPaint(vg, paint)
    nvgFill(vg)

    nvgGlobalAlpha(vg, 1.0)
    nvgRestore(vg)
end
```

**关键参数**：
- `nvgGlobalAlpha(vg, 0.08)`：几乎不可见，仅在纯黑底上增加微弱质感变化
- `time * 0.015`：极慢旋转，玩家几乎感觉不到在转，但长时间观察会发现云气在"流动"
- `margin = 256`：旋转时四角不会露出空白

### 5.10 星尘粒子背景

整个问心过程中，背景始终有缓慢漂浮的微弱星尘粒子，营造"虚空非空"的意境：

```lua
local MAX_PARTICLES = 60
local particles = {}    -- { x, y, vx, vy, alpha, radius, twinklePhase }

local function initParticles(screenW, screenH)
    for i = 1, MAX_PARTICLES do
        particles[i] = {
            x = math.random() * screenW,
            y = math.random() * screenH,
            vx = (math.random() - 0.5) * 8,    -- 极慢水平漂移 ±4 px/s
            vy = -math.random() * 6 - 2,        -- 缓慢上浮 2~8 px/s
            alpha = math.random() * 0.3 + 0.05, -- 基础 alpha 0.05~0.35
            radius = math.random() * 1.5 + 0.5, -- 半径 0.5~2.0 px
            twinklePhase = math.random() * 6.28, -- 闪烁相位随机
        }
    end
end

local function updateAndDrawParticles(dt, time, screenW, screenH)
    for i = 1, MAX_PARTICLES do
        local p = particles[i]
        p.x = p.x + p.vx * dt
        p.y = p.y + p.vy * dt

        -- 循环回绕
        if p.y < -10 then p.y = screenH + 10; p.x = math.random() * screenW end
        if p.x < -10 then p.x = screenW + 10 end
        if p.x > screenW + 10 then p.x = -10 end

        -- 闪烁（正弦波调制 alpha）
        local twinkle = 0.5 + 0.5 * math.sin(time * 1.2 + p.twinklePhase)
        local finalAlpha = p.alpha * twinkle

        nvgBeginPath(vg)
        nvgCircle(vg, p.x, p.y, p.radius)
        nvgFillColor(vg, nvgRGBA(
            C_STAR_DIM[1] + math.floor((C_STAR_BRIGHT[1] - C_STAR_DIM[1]) * twinkle),
            C_STAR_DIM[2] + math.floor((C_STAR_BRIGHT[2] - C_STAR_DIM[2]) * twinkle),
            C_STAR_DIM[3] + math.floor((C_STAR_BRIGHT[3] - C_STAR_DIM[3]) * twinkle),
            math.floor(finalAlpha * 255)
        ))
        nvgFill(vg)
    end
end
```

**设计意图**：
- 粒子极其细小（0.5~2px）且暗淡，不抢文字焦点
- 缓慢上浮 + 闪烁 → "虚空中有微弱灵气流动"之感
- 60 个粒子足够营造氛围但不构成视觉噪音

### 5.11 终幕：残光汇聚 + 光柱注入

第五问选择完成后，不做任何结算画面。直接进入终幕两段动画：

#### 阶段 1：残光汇聚（`FINALE_CONVERGE`，2.0s）

五道题的选中位置各保留一个"残光记忆点"。五个光点从各自位置向屏幕中心汇聚：

```lua
-- 记忆点位置在每题选中时记录
-- selectedPositions[i] = { x = ..., y = ... }

local function drawConverge(elapsed, selectedPositions, cx, cy)
    local t = math.min(1, elapsed / 2.0)
    -- easeInOutCubic
    local eased
    if t < 0.5 then
        eased = 4 * t * t * t
    else
        local u = -2 * t + 2
        eased = 1 - u * u * u / 2
    end

    for i = 1, 5 do
        local sp = selectedPositions[i]
        local px = sp.x + (cx - sp.x) * eased
        local py = sp.y + (cy - sp.y) * eased

        -- 光点逐渐变亮、缩小
        local radius = 12 * (1 - eased * 0.7)    -- 从 12 缩到 3.6
        local bloomR = 40 * (1 - eased * 0.5)     -- bloom 范围收缩

        -- bloom
        nvgBeginPath(vg)
        nvgCircle(vg, px, py, bloomR)
        local grad = nvgRadialGradient(vg, px, py, radius, bloomR,
            nvgRGBAf(1.0, 0.88, 0.55, 0.3 + eased * 0.3),
            nvgRGBAf(1.0, 0.88, 0.55, 0))
        nvgFillPaint(vg, grad)
        nvgFill(vg)

        -- 核心光点
        nvgBeginPath(vg)
        nvgCircle(vg, px, py, radius)
        nvgFillColor(vg, nvgRGBA(255, 235, 190, math.floor(180 + eased * 75)))
        nvgFill(vg)
    end
end
```

#### 阶段 2：光柱注入（`FINALE_LIGHT`，1.5s）

五光合一后，从中心向上下延伸为一道光柱：

```lua
local function drawLightPillar(elapsed, cx, cy, screenW, screenH)
    local t = math.min(1, elapsed / 1.5)
    -- easeOutExpo: 爆发式展开
    local eased = 1 - math.pow(2, -10 * t)

    -- 光柱尺寸
    local pillarW = 4 + eased * 12          -- 宽度 4→16 px
    local pillarH = eased * screenH * 1.2   -- 高度从 0 到超出屏幕

    -- 外层 bloom（BoxGradient 实现柔光边缘）
    local bloomW = pillarW + 60
    local bloomH = pillarH
    local feather = 40

    nvgBeginPath(vg)
    nvgRect(vg, cx - bloomW/2, cy - bloomH/2, bloomW, bloomH)
    local grad = nvgBoxGradient(vg, cx - pillarW/2, cy - pillarH/2, pillarW, pillarH,
        0, feather,
        nvgRGBAf(C_PILLAR_BLOOM[1]/255, C_PILLAR_BLOOM[2]/255, C_PILLAR_BLOOM[3]/255, 0.35 * (1 - t * 0.5)),
        nvgRGBAf(C_PILLAR_BLOOM[1]/255, C_PILLAR_BLOOM[2]/255, C_PILLAR_BLOOM[3]/255, 0))
    nvgFillPaint(vg, grad)
    nvgFill(vg)

    -- 核心光柱
    nvgBeginPath(vg)
    nvgRect(vg, cx - pillarW/2, cy - pillarH/2, pillarW, pillarH)
    nvgFillColor(vg, nvgRGBA(
        C_PILLAR_CORE[1], C_PILLAR_CORE[2], C_PILLAR_CORE[3],
        math.floor(255 * (1 - t * 0.3))     -- 略微淡出
    ))
    nvgFill(vg)

    -- 在 t > 0.6 后光柱开始脉冲闪烁（暗示能量注入完成）
    if t > 0.6 then
        local pulse = 0.5 + 0.5 * math.sin((t - 0.6) * 25)  -- 快速脉冲
        nvgGlobalAlpha(vg, pulse * 0.3)
        nvgBeginPath(vg)
        nvgRect(vg, 0, 0, screenW, screenH)
        nvgFillColor(vg, nvgRGBA(255, 240, 210, 40))
        nvgFill(vg)
        nvgGlobalAlpha(vg, 1.0)
    end
end
```

## 六、触发机制与兼容

### 统一入口：`DaoQuestionSystem.TryEnter(player)`

**不要把触发逻辑硬绑在创角成功回调里**。所有入口统一走一个判定函数，由它决定是否进入问心：

```lua
--- 统一入口判定（所有触发场景都调用这个函数）
--- @param player Player
--- @param source "create_char" | "npc_first" | "npc_reset"
--- @return boolean entered  是否成功进入问心
function DaoQuestionSystem.TryEnter(player, source)
    local answers = player.daoAnswers

    if source == "create_char" or source == "npc_first" then
        -- 首次问心：daoAnswers 必须为空或非法
        if DaoQuestionSystem.HasCompleted(player) then
            return false  -- 已答过，拒绝重复触发
        end
        DaoQuestionSystem.BeginSession(player, false)  -- 免费
        return true

    elseif source == "npc_reset" then
        -- 重置问心：必须已答过 + 灵韵足够
        if not DaoQuestionSystem.HasCompleted(player) then
            return false  -- 从未答题，应走 npc_first
        end
        if player.lingYun < 500 then
            return false  -- 灵韵不足（UI 层提示 "你的灵韵尚浅"）
        end
        DaoQuestionSystem.BeginSession(player, true)   -- 标记为重置模式
        return true
    end

    return false
end
```

**调用点**（谁调、何时调）：

| 场景 | 调用时机 | 调用方式 |
|------|---------|---------|
| 新角色创角 | 场景加载完成后的首帧 `HandleUpdate` | `TryEnter(player, "create_char")` |
| 老角色首次 | NPC 玄微对话按钮点击 | `TryEnter(player, "npc_first")` |
| 重置问心 | NPC 玄微「重新问心」按钮点击 | `TryEnter(player, "npc_reset")` |

> **为什么不绑在 `CreateCharacter` 回调里？**
> 创角回调执行时场景可能尚未就绪（NanoVG 上下文、渲染管线、资源加载），硬绑回调会产生时序依赖。
> 改为"场景就绪后首帧检测 `daoAnswers` 为空则进入"，把时序问题消解为状态判定。

### NPC 玄微的 UI 行为

| 条件 | NPC 行为 |
|------|---------|
| `HasCompleted(player)` 返回 `false` | 显示「问心」按钮，点击调用 `TryEnter(player, "npc_first")` |
| `HasCompleted(player)` 返回 `true` | 显示「重新问心」按钮，点击调用 `TryEnter(player, "npc_reset")` |
| 灵韵不足（重新问心时） | 提示"你的灵韵尚浅，且去历练" |

### 老角色兼容

老存档加载后 `dao*` 字段缺失，默认为 0——老角色无问心加成，正常游玩不受影响。可随时找玄微免费做首次问心。

### 重置时的事务语义

重置涉及"扣灵韵 + 清旧答案 + 写新答案"三步，必须按事务语义处理，防止吞钱或白嫖：

**核心原则**：灵韵扣除和答案写入在同一个原子操作中完成，发生在五问**全部答完之后**，而非点击「重新问心」时。

```lua
--- 重置问心会话管理
--- BeginSession 阶段：仅标记进入问心状态，不扣费、不清数据
--- CommitSession 阶段：五问答完后，原子执行 扣费→清旧→写新
--- AbortSession 阶段：中途退出，什么都不改

function DaoQuestionSystem.BeginSession(player, isReset)
    -- 仅设置会话状态，不动任何持久数据
    self.session = {
        player   = player,
        isReset  = isReset,
        answers  = {},       -- 本次会话的临时答案（逐题追加）
    }
end

function DaoQuestionSystem.CommitSession()
    local s = self.session
    if not s then return false end

    -- 校验答案完整性
    local valid = DaoQuestionSystem.ValidateAnswers(s.answers)
    if not valid then
        Log:Error("DaoQuestion: CommitSession failed, invalid answers")
        self.session = nil
        return false
    end

    local player = s.player

    -- ★ 原子操作开始 ——————————————————
    -- 1. 扣灵韵（仅重置时）
    if s.isReset then
        if player.lingYun < 500 then
            -- 理论上 TryEnter 已检查过，这里是防御性兜底
            Log:Error("DaoQuestion: CommitSession failed, insufficient lingYun")
            self.session = nil
            return false
        end
        player.lingYun = player.lingYun - 500
    end

    -- 2. 清旧加成
    DaoQuestionSystem.ClearDaoFields(player)

    -- 3. 写新加成
    DaoQuestionSystem.ApplyAnswers(player, valid)

    -- 4. 持久化（触发存档）
    SaveSystem.MarkDirty()
    -- ★ 原子操作结束 ——————————————————

    self.session = nil
    return true
end

function DaoQuestionSystem.AbortSession()
    -- 中途退出：丢弃会话，不扣灵韵、不改属性
    if self.session then
        Log:Info("DaoQuestion: session aborted, no changes made")
    end
    self.session = nil
end
```

**时序图**：

```
点击「重新问心」
    ↓
TryEnter() → 检查灵韵 ≥ 500（仅检查不扣）→ BeginSession(isReset=true)
    ↓
进入五问流程（表现与首次一致，答案写入 session.answers 临时表）
    ↓
├─ 五问全部完成 → CommitSession()
│    → 扣灵韵 500 → 清旧 dao* → 写新 dao* → 触发存档
│
└─ 中途退出 → AbortSession()
     → 不扣灵韵、不改属性、什么都不变
```

**防御的故障场景**：

| 风险 | 旧方案（先扣后答） | 新方案（答完再扣） |
|------|------------------|------------------|
| 答到一半崩溃/断线 | 💀 灵韵已扣，属性未更新（吞钱） | ✅ session 丢失，灵韵不变 |
| 主动退出对话 | 💀 灵韵已扣，需要补偿逻辑 | ✅ AbortSession()，零副作用 |
| 答完后存档失败 | 💀 灵韵扣了，属性没持久化 | ⚠️ 同一帧内连续写入，失败概率极低；如需更强保证可加写前快照 |
| 灵韵在答题过程中被其他系统花掉 | 不影响（已扣） | ✅ CommitSession 再次检查 `lingYun >= 500`，不足则失败 |

### daoAnswers 合法性校验

所有读取 `daoAnswers` 的地方（`HasCompleted`、`ApplyAnswers`、反序列化后）必须经过统一校验函数，拒绝脏数据：

```lua
local VALID_VALUES = { A = true, B = true }

--- 校验 daoAnswers 是否合法
--- @param raw any  从存档或 GM 命令传入的原始值
--- @return table|nil  合法则返回规范化后的 table，否则返回 nil
function DaoQuestionSystem.ValidateAnswers(raw)
    -- 必须是 table
    if type(raw) ~= "table" then return nil end

    -- 必须恰好 5 个元素（不多不少）
    local count = 0
    for _ in pairs(raw) do count = count + 1 end
    if count ~= 5 then return nil end

    -- 每个元素必须是连续整数键 1~5，值为 "A" 或 "B"
    local result = {}
    for i = 1, 5 do
        local v = raw[i]
        if type(v) ~= "string" or not VALID_VALUES[v] then
            return nil
        end
        result[i] = v
    end
    return result
end
```

**调用时机**：

| 场景 | 处理方式 |
|------|---------|
| 反序列化（`DeserializePlayer`） | `ValidateAnswers(data.daoAnswers)`，非法则置 `{}` + `dao*` 全清零 |
| 存档迁移（v18→v19） | 同上，对已有 `daoAnswers` 做校验 |
| GM 命令 `/dao_set` | 先校验，非法则拒绝并报错 |
| `ApplyAnswers` 入口 | 内部 assert，防御性兜底 |
| `HasCompleted` 判定 | `ValidateAnswers(player.daoAnswers) ~= nil` |

**典型脏数据场景**（均会被拦截）：
- 老存档 `daoAnswers` 为 `nil` 或非 table → 视为未答题
- GM 误写 `daoAnswers = {"A","B","C","A","A"}` → `"C"` 非法，整组拒绝
- 手动篡改少于/多于 5 题 → 长度不对，拒绝
- 非连续键 `{[1]="A", [3]="B"}` → 遍历缺失 `raw[2]`，拒绝

**反序列化中的集成**：

```lua
-- SaveSerializer.DeserializePlayer() 中，替换原来的简单 or：
local rawAnswers = data.daoAnswers or {}
local validAnswers = DaoQuestionSystem.ValidateAnswers(rawAnswers)
if validAnswers then
    player.daoAnswers = validAnswers
    -- dao* 字段正常读取
    player.daoWisdom       = data.daoWisdom       or 0
    player.daoFortune      = data.daoFortune      or 0
    player.daoConstitution = data.daoConstitution or 0
    player.daoPhysique     = data.daoPhysique     or 0
    player.daoAtk          = data.daoAtk          or 0
    player.daoMaxHp        = data.daoMaxHp        or 0
else
    -- 答案非法 → 全部清零，等同于从未答题
    player.daoAnswers      = {}
    player.daoWisdom       = 0
    player.daoFortune      = 0
    player.daoConstitution = 0
    player.daoPhysique     = 0
    player.daoAtk          = 0
    player.daoMaxHp        = 0
    Log:Warning("DaoQuestion: invalid daoAnswers in save, reset to empty")
end
```

## 十一、FAQ

**Q: 玩家能看到每道题对应什么属性吗？**
A: 不能。问心过程和 NPC 均不暗示属性对应关系。答完后属性面板可见加成数值（如"问心悟性 +5"），但不标注来自第几题。天道不解释。

**Q: 可以中途退出问心吗？**
A: 创角首次问心不可跳过（5 题必须答完）。NPC 重置时如果中途退出，保留旧属性不变、不扣灵韵。

**Q: 老账号不做问心会怎样？**
A: 正常游玩，`dao*` 全部为 0，无加成也无惩罚。可随时找玄微免费做首次问心。

**Q: "四脉均衡"和"极致双修"哪个更强？**
A: 没有强弱之分。均衡适合泛用，极致适合配合职业打极端流派。这是道途，不是考试。

**Q: 为什么 Q5 是攻击/生命而不是仙缘属性？**
A: Q1-Q4 已完美覆盖四仙缘（各出现 2 次）。Q5 引入基础属性维度，让最终一问"生与死"的哲学重量直接映射到最本质的战斗选择——伤害还是存活。

**Q: 重置费用 500 灵韵是否合理？**
A: 初章通关约可获得 2000-3000 灵韵（待确认具体经济数据），500 灵韵约为初章灵韵收入的 15-25%，属于"有代价但可承受"。如需调整，修改 `DaoQuestionSystem` 中的常量即可。

**Q: 问心数据会不会在角色间串？**
A: 不会。`dao*` 字段存储在 `save_{slot}` 的 `player` 对象中，与 `daoTreeWisdom`、`wineSlots` 等角色级数据同级，完全隔离。
