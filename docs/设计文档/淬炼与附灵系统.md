## 二、Tab1：萃取

### 2.1 规则

- **输入**：同一套装的 **3 件** T9 灵器装备（**必须同套装**） + **100 灵韵**
- **输出**：该套装对应的**附灵玉·壹**（确定性，非随机）

| 输入（3件同套装） | 输出 |
|----------------|------|
| 3× 血煞 T9 | 血煞附灵玉·壹（`lingyu_xuesha_lv1`）|
| 3× 青云 T9 | 青云附灵玉·壹（`lingyu_qingyun_lv1`）|
| 3× 封魔 T9 | 封魔附灵玉·壹（`lingyu_fengmo_lv1`）|
| 3× 浩气 T9 | 浩气附灵玉·壹（`lingyu_haoqi_lv1`）|

**T9 灵器套装判定**：`item.tier == 9 AND item.quality == "cyan" AND item.setId ∈ {xuesha,qingyun,fengmo,haoqi}`

**前置检查（任一不满足则禁止）**：
- 三槽均放入合法 T9 灵器套装件
- 三件 `setId` **完全相同**
- 玩家灵韵 ≥ 100

提交的 3 件装备**永久销毁**（**仅限背包中的装备**，装备栏中已穿戴的装备不可提交）。

### 2.2 执行逻辑（纯客户端）

```lua
local setId = slot1.setId  -- 三件相同，取第一件的 setId
local outputId = "lingyu_" .. setId .. "_lv1"

-- 1. 销毁三件装备（从背包/装备栏移除）
-- 2. player.lingYun -= 100
-- 3. InventorySystem.AddConsumable(outputId, 1)
-- 4. SaveSystem:Save()
-- 5. 播放炼化动效 + 提示获得对应附灵玉
```

### 2.3 UI 流程

```
打开淬炼师 → Tab：萃取
  ↓
三个装备槽位（点击从**背包**中选择，装备栏已穿戴的装备不可选）
  ↓
验证规则：
  - 每槽：显示"✓ T9灵器套装"或"✗ 不符合条件"
  - 套装一致性：三件不同套装时显示"✗ 需要相同套装"
  - 底部预览："萃取产出：[血煞附灵玉·壹]"（根据当前三件更新）
  - 费用：100 灵韵（不足时变红）
  ↓
[确认萃取]（全部合法才可点击）
  ↓
确认弹窗："销毁这3件血煞装备并消耗100灵韵，换取1枚血煞附灵玉·壹？"
  ↓
执行 → 炼化特效 → 弹出"获得 血煞附灵玉·壹 ×1"
```

## 三、Tab2：附灵

### 3.1 规则

**目标装备**（必须满足）：
- **仅限装备栏中已穿戴的装备**，背包装备不可被附灵
- **无自然 `setId`**（非套装装备）：只有非套装装备可被附灵
- 任何原本有 `setId` 的装备（无论是否四大套装）**均无法附灵**

> 设计意图：附灵是为非套装装备注入套装属性，而非强化已有套装装备。

**费用**：1× 对应附灵玉·壹 + **100 灵韵**

**结果**（确定性）：
- 使用 血煞附灵玉·壹 → `item.enchantSetId = "xuesha"`
- 使用 青云附灵玉·壹 → `item.enchantSetId = "qingyun"`
- 使用 封魔附灵玉·壹 → `item.enchantSetId = "fengmo"`
- 使用 浩气附灵玉·壹 → `item.enchantSetId = "haoqi"`

**重新附灵**：已有 `enchantSetId` 的装备可再次消耗另一枚附灵玉+100灵韵覆盖。

**附灵后立即存档**。

### 3.2 套装计件效果

`enchantSetId` 让装备在计算套装效果时**额外贡献一个套装的件数**：

```
item.setId       = nil         → 不贡献任何原有套装
item.enchantSetId = "qingyun"  → 青云套装 +1件（附灵贡献）
```

例：玩家装备 5 件青云套装 + 1 件附灵了"青云"的非套装防具 → 视为 6 件青云。

> ⚠️ `InventorySystem` 套装计件需修改（见第六节）。

### 3.3 执行逻辑（纯客户端）

```lua
-- setIdMap：consumableId → enchantSetId
local SET_ID_MAP = {
    lingyu_xuesha_lv1  = "xuesha",
    lingyu_qingyun_lv1 = "qingyun",
    lingyu_fengmo_lv1  = "fengmo",
    lingyu_haoqi_lv1   = "haoqi",
}

local enchantSetId = SET_ID_MAP[selectedConsumableId]
-- 1. 校验 item.setId == nil
-- 2. 校验 CountConsumable(selectedConsumableId) >= 1
-- 3. 校验 player.lingYun >= 100
-- 4. ConsumeConsumable(selectedConsumableId, 1)
-- 5. player.lingYun -= 100
-- 6. item.enchantSetId = enchantSetId
-- 7. SaveSystem:Save()
-- 8. 播放附灵动效（MUST）
-- 9. 刷新套装计件 + 属性面板
```

### 3.4 UI 流程

```
打开淬炼师 → Tab：附灵
  ↓
左侧：装备列表（仅显示**装备栏**中非套装的已穿戴装备，setId==nil）
  - 已附灵的装备显示附灵标记（可覆盖）
  ↓
右侧上：选择附灵玉（从背包中持有的附灵玉选择，显示图标+套装名+数量）
右侧中：目标装备信息 + 当前附灵状态
右侧下：费用 × 附灵玉1枚 + 100灵韵
  ↓
[附灵] 按钮
  - 无附灵玉 或 灵韵不足 → 置灰 + 提示
  ↓
执行 → 播放动效 → 显示"[青云套装] 附灵成功"
  ↓
刷新装备显示（Tooltip 出现附灵行）
```

### 3.5 视觉效果（MUST）

| 阶段 | 时长 | 内容 |
|------|------|------|
| 蓄力 | ~0.5s | 附灵玉图标飘向装备格，玉石旋转发光（对应套装颜色） |
| 注灵 | ~0.8s | 装备格流光特效（NanoVG），颜色渐变为套装主色 |
| 揭晓 | ~0.5s | 装备格闪光，弹出结果标签（套装名 + 颜色） |

参考：`BreakthroughCelebration.lua`

## 四、装备 Tooltip 附灵显示

`EquipTooltip.lua` 中新增：

```
[装备名]
──────────────────
主属性: 攻击 +120
副属性: 暴击 +5%
洗练属性: 速度 +8    ← 原有
附灵：青云套装       ← 新增（金色/橙色高亮，与套装行同色系）
```

**注意**：原有套装装备（有 setId）不会有 enchantSetId，无需变动其 Tooltip。只有非套装装备被附灵后才显示此行。

## 五、黑市配置

四种附灵玉均可上架黑市，定价一致：

| 属性 | 值 |
|------|-----|
| 黑商收购价（`buy_price`，玩家卖出得） | **20 仙石** |
| 黑商出售价（`sell_price`，玩家买入花费） | **40 仙石** |
| 库存上限 | `BMConfig.MAX_STOCK = 5`（各种独立计） |

**BMConfig.ITEMS 中新增**（新分类 `lingyu`）：
