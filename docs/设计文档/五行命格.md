# 五行命格

> **版本**: v0.25  
> **日期**: 2026-06-03  
> **状态**: 讨论稿  
> **当前命名建议**: 对外统一使用“五行命格”，仙箓作为旧称/研发代号保留  
> **设计定位**: 章节 BOSS 掉落的独立副装备系统

---

## 一、概述

### 1.1 系统目标

五行命格是一条独立于常规装备、法宝、神器之外的副养成线，核心体验参考暗黑 2 护身符，但做如下本土化调整：

- **来源固定**: 每个命格固定对应一个 BOSS
- **命名固定**: 命格名固定带五行和 BOSS 名；若附带套装，则额外显示四象套装名
- **构筑明确**: 玩家最多装备 15 个命格，按五行分栏组合属性
- **叠加明确**: 同属性命格允许同时装备，但同命格 ID 不可重复
- **定位明确**: 命格只做“添头”，提供持续构筑收益，不替代主装备体系

### 1.2 设计原则

1. **独立背包，独立装备栏，入口并入装备页**
   命格有独立背包和独立装备栏，UI 入口放在装备栏切页中。
2. **每枚命格只加 1 条属性**
   不带副属性，不带灵性/圣性，不做多词条污染。
3. **按命格 ID 判重**
   同名命格不能重复装备；不同命格即使同属性，也允许同时装备。
4. **五行分栏装备**
   金木水火土各固定 3 个命格位，形成 `5 × 3` 的装备结构。
5. **主属性互相竞争，次级属性分开搭配**
   五行先承载主属性轴，再挂 1 条次级属性，避免次级属性干扰主属性追求。
6. **四象套装独立**
   套装是额外附加标签，不是每件命格必有。

### 1.3 与现有系统的关系

| 系统 | 关系 | 说明 |
|------|------|------|
| 常规装备 | 并行 | 不占用现有装备槽 |
| 法宝/专属 | 并行 | 不复用 `treasure` / `exclusive` 槽位 |
| 背包系统 | 独立 | 不共用主背包格子 |
| 装备栏系统 | 接入 | 作为装备栏的独立切页 |
| 掉落系统 | 强关联 | 通过章节 BOSS 掉落表接入 |
| 套装系统 | 独立 | 命格有独立四象套装标签和套装效果 |
| 存档系统 | 需扩展 | 建议新增独立 `mingge` 数据块 |

## 二、核心规则

### 2.1 解锁与获取

- **系统解锁境界**: `金丹初期`
- **解锁后规则**: 系统解锁后，可回刷更低等级的历史 BOSS 获取对应命格
- **来源**: 章节 BOSS
- **掉落归属**: 每个 BOSS 固定掉落自己的本命命格
- **掉落随机项**: 先 roll 品质，再生成命格，然后判定是否附带四象套装

### 2.2 装备规则

| 项目 | 规则 |
|------|------|
| 最大装备数 | 15 |
| 是否分部位 | 分五行部位，金木水火土各 3 个命格位 |
| 判重规则 | 同 `命格ID` 禁止重复装备 |
| 同属性叠加 | 允许 |
| 跨 T 级同属性叠加 | 允许 |
| 套装共存 | 允许 |

### 2.3 “同一种装备”的定义

- **同一种命格 = 同一个命格 ID**
- **不是按属性判重**
- **不是按 T 级判重**

示例：

| 装备 | 属性 | 是否可同时装备 |
|------|------|----------------|
| 金命·沙万里（T1） | 天诛伤害 | 可 |
| 金命·焚天蜃龙（T2） | 天诛伤害 | 可 |
| 金命·沙万里（T1 紫） + 金命·沙万里（T1 青） | 天诛伤害 | 不可 |

### 2.4 命格属性规则

- 每枚命格只加 **1 条属性**
- 不生成副属性
- 不生成灵性/圣性
- 不做洗练词条系统
- 数值有随机波动
- 每枚命格固定归属 1 个五行部位
- 每枚命格**可能**附带 1 个四象套装标签

## 三、命名与掉落归属

### 3.1 命名规则

统一命名格式：

```text
{五行}命·{BOSS名}
```

若命格附带四象套装，则扩展为：

```text
{五行}命·{BOSS名}·{套装名}
```

示例：

- `金命·沙万里`
- `金命·沙万里·青龙`
- `火命·烈焰狮王·白虎`
- `水命·蜃妖王·玄武`

### 3.2 本命命格规则

- 一只 BOSS 对应 1 个命格 ID
- 该命格固定对应 1 条属性
- 该命格固定归属于 1 个五行部位
- 该命格默认无套装，生成时有概率附带 1 个四象套装

### 3.3 特殊来源规则

- `流沙之子·外域` 与 `流沙之子·中域` 掉落 **同一个命格**
- 双来源同命格时：
  - 共用同一个 `命格ID`
  - 共用同一个命格名
  - 共用同一个微缩头像资源

## 四、五行属性设计

### 4.1 属性池

当前命格属性固定为 15 条：

- 攻击（`atk`）
- 防御（`def`）
- 生命上限（`maxHp`）
- 生命回复（`hpRegen`）
- 击杀回血（`killHeal`）
- 重击（`heavyHit`）
- 暴击率（`critRate`）
- 暴击伤害（`critDmg`）
- 移动速度（`moveSpeed`）
- 天诛（`tianzhuChance`）
- 天诛伤害（`tianzhuDamage`）
- 福缘（`fortune`）
- 悟性（`wisdom`）
- 根骨（`constitution`）
- 体魄（`physique`）

### 4.2 五行归属

| 五行 | 属性 | 对应字段 | 主属性轴 | 次级属性 |
|------|------|---------|---------|---------|
| 金 | 暴击伤害、天诛伤害、击杀回血 | `critDmg`, `tianzhuDamage`, `killHeal` | 暴击伤害 + 天诛伤害 | 击杀回血 |
| 木 | 生命上限、体魄、重击 | `maxHp`, `physique`, `heavyHit` | 生命上限 + 体魄 | 重击 |
| 水 | 暴击率、天诛、生命回复 | `critRate`, `tianzhuChance`, `hpRegen` | 暴击率 + 天诛 | 生命回复 |
| 火 | 攻击、悟性、移动速度 | `atk`, `wisdom`, `moveSpeed` | 攻击 + 悟性 | 移动速度 |
| 土 | 防御、根骨、福缘 | `def`, `constitution`, `fortune` | 防御 + 根骨 | 福缘 |

### 4.3 分配原则

- 主属性互相竞争，决定该五行的主追求方向
- 次级属性分开搭配，不与同流派主属性绑死
- 同属性命格允许多件共存，但必须来自不同命格

## 五、品质、roll 与数值规则

### 5.1 品质

当前先锁定 3 档品质：

| 品质 | 标识 | 说明 |
|------|------|------|
| 紫 | `purple` | 最低可用品质 |
| 橙 | `orange` | 进阶品质 |
| 青 | `cyan` | 高端品质 |

补充说明：

- 当前版本固定为 **紫 / 橙 / 青** 3 档品质
- 当前版本 **不开放第 4 档品质**
- 当前版本 **不开放初始品质升级**

### 5.2 roll 区间

命格总 roll 轴固定为 `1 ~ 6000`，三段品质均匀切分：

| 品质 | roll 区间 |
|------|----------|
| 紫 | `1 ~ 2000` |
| 橙 | `2001 ~ 4000` |
| 青 | `4001 ~ 6000` |

规则说明：

- 品质不是窄池稀有段，而是完整大区间
- 同品质内部仍然保留较大的 roll 浮动空间
- 最终属性值由“品质段 + 段内 roll 位置”共同决定

### 5.3 显示精度

命格数值显示规则与装备保持一致：

- **生命回复**: 保留 `1` 位小数
- **全部百分比属性**: 保留 `1` 位小数
- **其余属性**: 只显示整数

对应到当前属性：

- 保留 `1` 位小数：生命回复、暴击率、暴击伤害、移动速度、天诛、天诛伤害
- 显示整数：攻击、防御、生命上限、击杀回血、重击、福缘、悟性、根骨、体魄

### 5.4 T 级结构

当前奖励池按 `T1 / T2 / T3` 三档规划：

| T 级 | 规模 | 定位 |
|------|------|------|
| T1 | 15 个 BOSS 命格 | 中前期主线命格池 |
| T2 | 15 个 BOSS 命格 | 后期与第五章命格池 |
| T3 | 7 个 BOSS 命格 | 120 级终盘高追求命格池 |

说明：

- `T1` 与 `T2` 各自完整覆盖 15 条属性
- `T3` 不强求覆盖 15 条属性，优先承载最高追求属性

### 7.5 结构说明

- 除 **生命上限 / 生命回复** 外，其余属性统一遵循：
  - `T2` 第一档下限 = `T1` 第二档上限

- 生命上限按 **攻击/防御的 5 倍量级** 投放
- 生命回复按 **生命上限 ÷ 10** 设计，并保留 1 位小数
- 重击按 **攻击/防御的 3 倍** 投放
- 暴击率从 `1.0%` 起
- 暴击伤害按暴击率约 `2.5 倍` 设计
- 天诛从 `1.0%` 起
- 天诛伤害按天诛约 `2.5 倍` 设计
- 移动速度与暴击率同档

## 八、掉落与四象套装

### 8.1 掉落流程

1. 击杀对应 BOSS
2. 以 `5%` 概率掉落该 BOSS 的本命命格
3. 先 roll 品质
4. 再生成该件命格
5. **仅最高品质（当前为青色）** 才有 `20%` 概率附带四象套装；紫色、橙色不出套装

### 8.2 掉落规则

- 每个 BOSS 只掉落自己的本命命格
- 每个 BOSS 一次最多掉 1 枚命格
- 命格属性固定，品质与数值随机
- 命格掉落后与装备一样，**会在地上显示掉落物**
- 命格掉落物与装备同规则，**宠物不会自动拾取**
- 套装不是固定掉落，而是生成后的额外随机标签
- 当前版本 **不做“为命格增加套装 / 更改套装”**
- 如后续需要，按 **独立模块** 设计，不并入五行命格首发系统

### 8.3 四象套装

四象套装固定为：

1. 青龙
2. 白虎
3. 朱雀
4. 玄武

### 8.4 套装触发规则

- 同一五行的 3 个格子中，若 3 枚命格套装名完全一致，则触发该五行的 3 件套效果
- 没有套装的命格不参与套装判定
- 五行之间独立结算，每个五行最多触发 1 个套装效果
- 同一种四象套装允许在不同五行重复出现，并分别形成独立套装效果
- 同名四象套装效果 **允许叠加**
- 理论上最多可同时获得 `5` 个套装效果

### 8.5 四象效果口径

| 套装 | 3 件套效果 |
|------|------------|
| 青龙 | 攻速 `+8%` |
| 白虎 | 暴击伤害 `+20%` |
| 朱雀 | 天诛伤害 `+20%` |
| 玄武 | 宠物同步率 `+8%` |

## 九、UI 与交互

### 9.1 入口规则

五行命格使用：

- **独立背包**
- **独立装备栏**
- **装备栏切页入口**

玩家切到命格切页时：

- 左侧/上方切换为命格装备栏
- 右侧切换为命格独立背包
- 不显示主背包内容

命格掉落交互规则：

- 命格掉落物沿用装备掉落的地面显示表现
- 玩家可手动拾取命格
- 与装备同规则，宠物不会自动拾取命格
- 命格相关的出售、筛选、锁定等操作仅作用于 **命格独立背包**
- 不与主背包 / 道具背包共用列表、入口或批量操作

### 9.2 页面布局

| 当前切页 | 左侧/上方栏位 | 右侧背包 |
|---------|---------------|---------|
| 常规装备页 | 常规装备栏 | 主背包 |
| 命格页 | 命格装备栏 | 命格独立背包 |

### 9.3 命格背包建议

| 项目 | 建议 |
|------|------|
| 背包大小 | 60 |
| 排序 | 品质降序 -> T级降序 -> 五行 -> BOSS名 -> 数值降序 |
| 锁定 | 支持 |
| 批量出售 | 支持；仅作用于命格独立背包，交互逻辑与装备页的批量出售一致，但不与主背包 / 道具背包混用 |
| 过滤 | 五行 / 套装 / 属性 / T级 / 是否已装备 |

### 9.4 命格装备栏建议

- 按五行分成 `金 / 木 / 水 / 火 / 土` 五栏
- 每一栏固定 `3` 个命格位
- 已装备同名命格时，拖入新件触发替换确认
- 每一栏显示当前是否已激活对应四象 3 件套

## 十、养成与回收

### 10.1 首发不建议做的内容

- 升星
- 强化
- 二次词条
- 形状/大小格玩法
- 命格洗练 / 洗值
- 命格合成升 T级

### 10.2 首发建议做的回收

- 命格可直接 **出售获得灵韵**
- 出售收益按 **T级 × 品质系数** 计算

品质系数：

- 紫：`1`
- 橙：`2`
- 青：`3`

补充说明：

- `T1`：紫 `1` / 橙 `2` / 青 `3`
- `T2`：紫 `2` / 橙 `4` / 青 `6`
- `T3`：紫 `3` / 橙 `6` / 青 `9`
- 以此类推，后续更高 T 级继续按同一公式递推
- 当前版本不引入命格专用回收货币
- 出售直接结算为主货币 `灵韵`
- 当前版本命格 **无法洗练**
