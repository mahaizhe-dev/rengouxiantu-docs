## 1. 文档目的

本文档只描述当前代码中**已经存在且按现状实际运行**的宠物系统规则，不讨论目标设计，也不代替审计文档。

## 2. 系统概览

### 2.1 基本定位

- 宠物是**默认伴生系统**，角色创建后会直接生成宠物实例。
- 当前宠物实体名固定为`小黄`。
- 当前代码形态下，单角色只维护**1只宠物**。
- 宠物长期跟随玩家，参与战斗、成长、突破、技能、外观与拾取。

### 2.2 核心行为摘要

- 宠物攻击目标取自`玩家当前目标`，不会独立索敌。
- 宠物有等级、经验、阶级、技能槽、外观选择。
- 宠物基础属性由`固定基础值 + 等级成长 + 主人同步属性 + 技能修正`共同构成。
- 宠物外观分为：
  - 基础外观：随宠物阶级解锁
  - 高级外观：账号级解锁，属性加成作用于**角色**而不是宠物

## 3. 成长规则

### 3.1 阶级与等级上限

| 阶级 | 名称 | 等级上限 | 基础同步率 |
| --- | --- | ---: | ---: |
| 0 | 幼犬 | 20 | 20% |

| 5 | 圣犬 | 120 | 70% |

| 7 | 天犬 | 160 | 100% |

说明：

- 宠物突破后提高等级上限。
- 7阶后无下一阶，视为满阶。

### 3.2 基础属性与成长

宠物1级基础值：

- 生命：40
- 攻击：5
- 防御：2

每级固定成长：

- 生命：+20
- 攻击：+2
- 防御：+2

### 3.3 属性公式

宠物三项核心属性的实际计算公式为：

`(基础值 + 固定成长 × (等级 - 1) + 主人总属性 × 同步率) × (1 + 技能百分比加成) + 蕴灵类每级固定加成`

其中：

- 主人生命取`GetTotalMaxHp()`
- 主人攻击取`GetTotalAtk()`
- 主人防御取`GetTotalDef()`
- 技能的百分比型、每级型加成均参与该计算

### 3.4 同步率规则

当前同步率 = `当前阶级基础同步率 + 主人福缘同步加成`

主人福缘同步加成为：

`floor(总福缘 / 25) × 2% + 装备套装 petSyncRate 加成`

结论：

- 宠物同步率不仅取决于宠物阶级，也取决于角色福缘与部分装备效果。
- 当前代码里**没有额外同步率上限钳制**。

### 3.5 升级规则

- 宠物只能在**存活状态**下喂食。
- 达到当前阶级等级上限后无法继续获得有效升级，必须先突破。
- 喂食后会自动连续检查升级，允许一次喂食跨多级。
- 每次升级后会：
  - 重新计算属性
  - 宠物生命回满

## 5. 战斗与跟随规则

### 5.1 跟随规则

- 宠物默认状态为`follow`。
- 跟随速度：`5.0`
- 跟随启动距离：`1.5`
- 跟随停止距离：`0.75`
- 与主人距离超过`10格`时，宠物会直接瞬移到玩家身边。

### 5.2 攻击规则

- 宠物只攻击`玩家当前 target`。
- 宠物攻击距离：`1.2格`
- 普攻固定攻击间隔：`1.2秒`
- 若当前有目标，宠物进入`attack`状态；否则回到`follow`

### 5.3 死亡与复活

- 宠物死亡后进入`dead`状态
- 复活时间：`30秒`
- 复活后：
  - 重新计算属性
  - 生命回满

### 5.4 宠物伤害公式

基础伤害公式沿用全局公式：

`DAMAGE = floor((ATK^2 / (ATK + DEF)) × 随机波动)`

其中随机波动范围为：

- `0.9 ~ 1.1`

最终流程为：

1. 先判定忽视防御
2. 再按公式计算基础伤害
3. 再判定暴击
4. 再乘以加伤倍率
5. 最低伤害保底为1
6. 结算吸血

### 5.5 伤害修正规则

- 忽视防御：触发时目标防御按0处理
- 暴击倍率：`1.5 + critDmgPct / 100`
- 加伤：最终伤害再乘`1 + bonusDmgPct / 100`
- 吸血：按最终造成伤害的百分比回复生命
- 连击：只对普攻生效，触发后追加一次独立攻击
- 闪避：宠物受击前先判定，成功则本次伤害为0

## 6. 固有技能与宠物行为特性

### 6.1 固有被动：犬魂狂暴

触发条件：

- 宠物死亡时

效果：

- 主人获得10秒狂暴
- 攻速+30%
- 移速+30%

规则细节：

- 该效果作用于玩家，不作用于宠物
- 采用刷新计时逻辑，不叠层

### 6.2 固有行为：拾荒伴侣

当前实际规则：

- 宠物可捡范围：`2.5格`
- 掉落物落地后至少`2秒`，宠物才会去捡
- 扫描频率：每`0.5秒`扫描一次附近掉落
- 追逐单个掉落物超时：`3秒`
- 宠物**不会捡装备**
- 宠物不会捡已经被玩家磁吸的掉落
- 宠物拾取失败后，会将该掉落加入`30秒黑名单`

宠物拾取时的结算规则：

- 经验、金币、灵韵：直接结算给玩家
- 消耗品：按背包实际可写入数量拾取
- 装备：完全跳过

## 7. 主动技能：灵噬

### 7.1 解锁条件

按当前代码实际，灵噬在**1阶即可解锁**。

当前阶级命名下，1阶名称是`凶犬`，因此实际规则是：

- 宠物突破到`凶犬`后解锁灵噬

### 7.2 技能参数

- 技能ID：`spirit_bite`
- CD：`12秒`
- 易伤效果：目标受到所有伤害`+8%`
- 易伤持续时间：`4秒`

### 7.3 各阶伤害倍率

| 宠物阶级 | 灵噬伤害倍率 |
| --- | ---: |
| 1 | 100% ATK |
| 2 | 120% ATK |
| 3 | 140% ATK |
| 4 | 160% ATK |
| 5 | 180% ATK |
| 6 | 200% ATK |
| 7 | 220% ATK |

### 7.4 释放逻辑

- 只有在宠物已有目标且在攻击距离内时才会自动尝试释放
- 灵噬优先于普攻判定
- 释放后重置CD
- 灵噬同样会吃到：
  - 忽视防御

  - 吸血

## 8. 技能系统规则

### 8.1 技能槽

- 技能槽总数固定为`10格`
- 布局为`2 x 5`
- 当前代码下10格全部默认开放，无额外解锁条件

### 8.2 学习规则

- 只能从空技能格学习
- 只能学习`1级技能书`
- 同一技能ID只允许存在一份
- 学习时消耗对应技能书`1本`
- 学会后立即重算宠物属性

运行时补充：

- 即便异常情况下出现同技能重复插入，属性结算时也会按`skill.id`去重，不重复叠加

### 8.3 升级规则

技能最高等级为`4级`。

升级到各目标等级时，当前成本如下：

| 目标等级 | 路径A：普通升级 | 路径B：精研升级 |
| --- | --- | --- |
| 2 | 2级书 x1，20%成功 | 2级书 x1 + 100灵韵，100%成功 |
| 3 | 3级书 x1，10%成功 | 3级书 x1 + 1000灵韵，100%成功 |
| 4 | 4级书 x1，10%成功 | 4级书 x1 + 10000灵韵，100%成功 |

规则细节：

- 普通升级失败会消耗技能书
- 精研升级必定成功
- 升级成功后立即重算宠物属性

### 8.4 删除规则

- 删除技能需要二次确认
- 删除后不返还任何材料
- 删除后立即重算宠物属性

### 11.4 外观加成生效规则

这是当前代码里非常关键的一条现状规则：

- **高级外观的加成作用于角色，不作用于宠物**
- **加成按“账号已解锁的全部高级外观”汇总，不按当前装备中的那一件外观单独结算**

也就是说：

- 装备哪一件高级外观，主要影响显示
- 角色属性加成来自`账号已解锁高级外观的总和`

固定值类加成总封顶：

- 宠物成长数据与外观选择是角色核心存档的一部分
- 高级外观解锁状态是账号装扮数据的一部分

## 14. 代码现状备注

以下内容不是“目标设计”，而是当前代码表现下需要特别记住的现状：

1. 灵噬实际解锁条件是`1阶`，当前阶级名对应`凶犬`，不是更后面的阶级。
2. 高级外观的属性加成是“全部已解锁叠加”，不是“当前装备哪件就只吃哪件”。
3. `攻速强化`与`减伤强化`虽然有完整配置、技能书和UI，但当前战斗逻辑未真正使用。
4. 宠物技能重复学习在UI层被限制，在运行时属性结算层也会再次按`skill.id`去重。
5. 本文档只记录当前生效规则；合理性、风险、优化项请结合《宠物系统代码基准审计文档》一起看。
