> 基于代码实际分析，非设计稿。最后更新：2026-04-27

---

## 一、系统概述

阵营挑战系统是"切磋"玩法，玩家在地图上找到**阵营使者 NPC**，花费**令牌**解锁并挑战对应等级的 BOSS。击败后获得**专属装备**和**永久属性丹药**奖励。

### 涉及文件

| 文件 | 职责 |
|------|------|
| `config/ChallengeConfig.lua` | 阵营/等级/奖励/费用配置 |
| `config/MonsterTypes_ch3.lua` | 全部 16 个挑战怪物的数值定义 |
| `config/EquipmentData.lua` | 6 件专属装备的属性定义 |
| `config/SkillData.lua` | 3 个专属技能（血海翻涌/浩气领域/神塔镇压）定义 |
| `systems/ChallengeSystem.lua` | 核心逻辑：解锁、进入副本、退出、奖励发放 |
| `systems/SkillSystem.lua` | 专属技能绑定/解绑（槽位 4）+ `multi_zone_heavy` 技能类型 |
| `systems/BossMechanics.lua` | BOSS 专属机制（玉甲回响等） |
| `systems/InventorySystem.lua` | 装备穿戴时触发技能绑定 |
| `ui/ChallengeUI.lua` | 挑战面板 + 胜利弹窗 |
| `config/zones/chapter2/camp_a.lua` | Ch2 使者 NPC 放置 |
| `config/zones/chapter3/fort_9.lua` | Ch3 使者 NPC 放置 |
| `config/zones/midland/qingyun.lua` | 青云城挑战掌门 NPC 放置 |

## 四、挑战流程

```
玩家点击使者NPC
    ↓
打开 ChallengeUI（按NPC章节过滤显示 T1-T4 或 T5-T8）
    ↓
├── 未解锁 → 显示"提交 N 令牌解锁"按钮
│   └── 前置条件：上一级已通关（T1 无前置，但也需付费解锁）
│       └── 扣令牌 → 标记 unlocked → 立即存档
│
└── 已解锁 → 显示"挑战（10 令牌）"按钮
    ↓
扣除 10 令牌入场费
    ↓
进入虚空竞技场（覆盖层模式）
    ├── 备份当前世界（地图、怪物、NPC、掉落、玩家位置、相机）
    ├── 生成 8×8 独立小地图（6×6 可行走 + 四周墙壁）
    ├── 玩家传送到中心偏下，血量回满
    └── 生成挑战怪物在中心偏上
        ↓
    战斗进行中
        ├── 玩家死亡 → 由死亡界面提供"退出挑战"按钮
        └── 怪物死亡 → 弹出胜利弹窗
            ↓
        胜利弹窗内容：
            ├── 首通 → 显示首通奖励（装备/丹药）
            └── 非首通 → 判定重复掉落（概率）
                ├── 命中 → 显示额外掉落装备
                └── 未命中 → 显示"未触发额外掉落"
            ↓
        点击"确认"
            ├── 发放奖励（装备进背包/丹药应用永久属性）
            ├── 标记 completed
            └── 退出副本 → 恢复世界状态 → 血量回满
```

## 五、专属装备体系

### 5.1 装备三阶

每个阵营有 3 件专属装备，对应 3 个阶位：

| 阶位 | 血煞盟 | 浩气宗 | 品质 | Tier | 首通来源 |
|------|--------|--------|------|------|---------|
| 初 | 血海图·初 | 浩气印·初 | 紫 | T3 | 挑战 T1 |
| 破 | 血海图·破 | 浩气印·破 | 紫 | T5 | 挑战 T3 |
| 绝 | 血海图·绝 | 浩气印·绝 | 橙 | T7 | 挑战 T6 |

### 5.2 装备属性

所有专属装备占 **exclusive 槽位**（第 10 槽），主属性为**悟性 (wisdom)**。

#### 血海图系列

| 属性 | 初 (T3紫) | 破 (T5紫) | 绝 (T7橙) |
|------|----------|----------|----------|
| 悟性 (主属性) | 19 | 39 | 78 |
| 防御力 | +2.4 | +4.8 | +9.24 |
| 击杀回血 | +6 | +12 | +23.1 |
| 暴击伤害 | +8% | +12.5% | +20.9% |
| 技能 | 血海翻涌 | 血海翻涌 | 血海翻涌 |

#### 浩气印系列

| 属性 | 初 (T3紫) | 破 (T5紫) | 绝 (T7橙) |
|------|----------|----------|----------|
| 悟性 (主属性) | 19 | 39 | 78 |
| 防御力 | +2.4 | +4.8 | +9.24 |
| 生命回复 | +1.0/s | +2.0/s | +3.85/s |
| 暴击率 | +1.6% | +2.5% | +4.18% |
| 技能 | 浩气领域 | 浩气领域 | 浩气领域 |

### 5.3 装备机制

- 穿戴到 exclusive 槽位后，`InventorySystem.RecalcEquipStats()` 检测 `skillId`
- 自动调用 `SkillSystem.BindExclusiveSkill(skillId)` 绑定技能到**技能栏第 4 槽**
- 卸下时调用 `SkillSystem.UnbindExclusiveSkill(skillId)` 解绑
- 获取高阶装备后替换低阶，技能不变但 `tierScaling` 数值随阶提升

### 5.4 属性计算公式

```
主属性 = BASE_MAIN_STAT × TIER_MULTIPLIER × QUALITY_MULTIPLIER
副属性 = BASE_SUB × SUB_TIER_MULTIPLIER [× 橙品1.1倍]

TIER_MULTIPLIER: T3=3.0, T5=6.0, T7=10.5
QUALITY_MULTIPLIER: 紫=1.3, 橙=1.5
```

## 十、副本技术实现（覆盖层模式）

进入挑战时不做场景切换，而是**替换整个地图数据**：

1. **备份**当前世界完整状态（tiles、width/height、monsters、npcs、lootDrops、bossKillTimes、玩家位置、相机位置）
2. **生成** 8×8 独立地图（6×6 `CAMP_DIRT` 地板 + 四周 `WALL` 墙壁）
3. **清空**世界实体（怪物/NPC/掉落物）
4. **传送**玩家到中心偏下 (4.5, 5.5)，**生成** BOSS 在中心偏上 (4.5, 3.5)
5. **相机** snap 到新位置，避免平滑过渡跳动
6. 战斗结束后**恢复**备份的完整状态，血量回满

## 二、法宝属性规则

### 2.1 主属性

每件法宝的主属性**只有一条**，类型固定，数值由 Tier 和品质决定。

**战斗属性法宝**（攻击/防御/生命）沿用标准公式：
```lua
MainStat = BASE_MAIN_STAT × TIER_MULTIPLIER[tier] × qualityMultiplier
```

**仙缘属性法宝**（悟性/福缘/根骨/体魄）使用专用公式，确保主副属性恒定 5× 比值：
```lua
MainStat = SubStatValue × 5
-- 即：floor(tier × qualityMultiplier) × 5
-- 白品 T3=15, T5=25, T7=35, T9=45
```

### 8B.5 法宝专属技能：神塔镇压（`qingyun_suppress`）

| 属性 | 值 |
|------|-----|
| 类型 | `multi_zone_heavy`（多区域重击） |
| 冷却 | 45 秒 |
| 范围 | 2.0 格 |
| 区域数 | 4 个正方形，2×2 网格排列 |
| 每个区域大小 | 1.0 × 1.0 格 |
| 最大目标 | 8 |
| 伤害倍率 | 1.0×（**固定，不分阶**） |
| 伤害类型 | **重击**（无视防御） |
| 可暴击 | ✅ |
| 特效色 | 青翠绿 (80, 200, 120, 255) |

**伤害公式**：
```
heavyDmg = floor(ATK + heavyHit) × damageMultiplier × (1 + constitutionHeavyDmgBonus)
```
- `heavyHit`：来自法宝主属性根骨加成（根骨 → 重击值）
- `constitutionHeavyDmgBonus`：根骨提供的重击伤害加成百分比（根骨/25 × 1%）
- 重击无视目标防御，直接扣血
- 重击本身已极强（无视防御 + 根骨双重加成），不需要高倍率

**基准 DPS 计算**（基准属性：ATK=1500, HeavyHit=1000, 根骨=300, 敌方DEF=1000）：
```
重击基础伤害 = floor(1500 + 1000) = 2500
根骨加成 = 300/25 × 1% = 12%
加成后 = 2500 × (1 + 0.12) = 2800
× damageMultiplier(1.0) = 2800
CD = 45s → DPS ≈ 62（不计暴击）

含暴击（100%暴击率, 暴击倍率7.5x）：
2800 × 7.5 = 21000
CD = 45s → DPS ≈ 467
```

**multi_zone_heavy 技能类型说明**：
- 以目标方向为正前方，在目标周围生成 2×2 网格的 4 个正方形区域
- 每个区域内的怪物独立判定命中，使用 hitSet 去重防止多区域重叠的怪物被多次命中
- 全部 4 区域共享一次暴击判定（要么全暴击，要么全不暴击）
- 支持连击计数（combo）

**表现效果**：青云宝塔在区域中心凝现（淡入+放大），短暂蓄力发光后炸裂，8 片塔身碎片四散飞射 + 双层冲击波扩散 + 中心白绿闪光，2×2 区域底框同步闪亮后淡出

#### 技能不分阶（§8.4）

法宝技能不随声望等级成长，所有声望等级统一使用固定数值（damageMultiplier=1.0, CD=45s）。

### 8C.4 专属机制：封魔印（sealMark）

**机制说明**：厉无极周期性对玩家施加封魔印层数，达到上限时爆发伤害。玩家需要**持续输出**来清除封魔印层数，鼓励积极进攻的战斗风格。

**核心循环**：
1. 每 `stackInterval` 秒自动给玩家叠加 1 层封魔印
2. 玩家每次攻击命中 BOSS 清除 `clearPerHit`（=1）层
3. 当层数达到 `maxStacks`（=5）时，爆发 `burstDamagePercent` × 玩家最大 HP 的伤害，然后层数清零重新开始

**设计意图**：与沈墨的血煞印记（持续 DOT 惩罚）和云裳的玉甲回响（等待窗口期输出）形成差异化——封魔印鼓励**不间断攻击**，停止攻击 = 印记爆发。

**实现文件**：
- `systems/combat/BossMechanics.lua`（`UpdateSealMark` / `ClearSealMark` + EventBus monster_hurt 订阅）
- `entities/Monster.lua`（初始化 `self.sealMark` 配置）
- `entities/Player.lua`（`sealMarkStacks` / `sealMarkTickTimer` / `sealMarkConfig` 状态字段）
- `systems/CombatSystem.lua`（Update 中调用 `UpdateSealMark`）
- `systems/ChallengeSystem.lua`（退出时调用 `ClearSealMark`）

#### 封魔印参数递增表

| 声望等级 | stackInterval (秒) | maxStacks | burstDamagePercent | clearPerHit |
|---------|-------------------|-----------|-------------------|-------------|
| 1 (T3) | 5.0 | 5 | 20% MaxHP | 1 |
| 2 (T4) | 4.5 | 5 | 22% MaxHP | 1 |
| 3 (T5) | 4.0 | 5 | 25% MaxHP | 1 |
| 4 (T6) | 3.5 | 5 | 25% MaxHP | 1 |
| 5 (T7) | 3.0 | 5 | 25% MaxHP | 1 |
| 6 (T8) | 2.5 | 5 | 25% MaxHP | 1 |
| 7 (T9) | 2.0 | 5 | 25% MaxHP | 1 |

> **难度递增设计**：主要通过缩短 `stackInterval` 提升压力（5.0s→2.0s），高声望等级下玩家必须维持更高的攻击频率才能避免爆发。`burstDamagePercent` 在 R3+ 统一为 25%，避免一击秒杀但保持惩罚力度。

### 8C.5 法宝专属技能：封魔阵·增伤（`fengmo_seal_array`）

| 属性 | 值 |
|------|-----|
| 类型 | `damage_amp_zone`（增伤区域） |
| 冷却 | 20 秒 |
| HP 代价 | 当前 HP 的 10%（低于 10% HP 时无法施放） |
| 范围 | 2.0 格半径 |
| 持续时间 | 10 秒 |
| 增伤效果 | 区域内怪物受到最终伤害 +10% |
| 特效色 | 暗紫 (160, 80, 200, 255) |

**技能机制**：
- 施放时扣除当前 HP 的 10%（向下取整，最少扣 1 点）
- 在脚下生成增伤区域，区域内所有怪物受到的**最终伤害**增加 10%
- 增伤判定在 `Monster:TakeDamage` 中执行（在易伤 debuff 之后、淬炼加成之前）
- 同类增伤区域不叠加（多个区域取第一个命中的）

**与体魄属性的协同**：
- 体魄提供 MaxHP 加成（每 5 点 +1% MaxHP）→ HP 池更大 → HP 代价绝对值可接受
- 体魄提供 HP 回复（每 1 点 +0.3 HP/秒）→ 更快回复 HP 代价
- 体魄提供血怒加成（每 25 点 +1% 触发率/伤害）→ 增伤区域内血怒暴发更强

#### 技能不分阶（§8.4）

法宝技能不随声望等级成长，所有声望等级统一使用固定数值（CD=20s, 增伤=10%, HP代价=10%）。

### 10.2 旧存档迁移方案

**策略**：系统首次开启时做一次性映射，根据旧版章节解锁状态自动开启新版声望等级。

```lua
-- 迁移逻辑（只执行一次）
function MigrateLegacyChallenge(save)
    if save.version >= 3 then return end  -- 已迁移
    if save.treasure_challenge then return end  -- 已有新数据

    save.treasure_challenge = {}

    -- 仅对血煞、浩气做迁移（旧版只有这两个阵营）
    for _, faction in ipairs({"xuesha", "haoqi"}) do
        local treasureKey = faction == "xuesha" and "xuehaitu" or "haoqiyin"
        local tc = {}

        local oldProgress = save.progress and save.progress[faction]
        local hasChapter2 = oldProgress ~= nil  -- 有旧进度 = Ch2 已解锁
        local hasChapter3 = false
        if oldProgress then
            -- 检查是否有 T5+ 的解锁记录（T5 属于 Ch3）
            for k, v in pairs(oldProgress) do
                if tonumber(k) >= 5 and v.unlocked then
                    hasChapter3 = true
                    break
                end
            end
        end

        if hasChapter2 then
            -- Ch2 已解锁 → 开启声望 1,2,3（T3,T4,T5 使用乌堡令）
            tc["1"] = { unlocked = true, completed = false }
            tc["2"] = { unlocked = true, completed = false }
            tc["3"] = { unlocked = true, completed = false }
        end
        if hasChapter3 then
            -- Ch3 已解锁 → 额外开启声望 4,5（T6,T7 使用沙海令）
            tc["4"] = { unlocked = true, completed = false }
            tc["5"] = { unlocked = true, completed = false }
        end

        if next(tc) then
            save.treasure_challenge[faction] = { [treasureKey] = tc }
        end
    end

    -- 初始化精华库存（5 种通用精华）
    save.essence_counts = save.essence_counts or {
        lipo_essence = 0,      -- 力魄精华
        panshi_essence = 0,    -- 磐石精华
        yuanling_essence = 0,  -- 元灵精华
        shihun_essence = 0,    -- 噬魂精华
        lingxi_essence = 0,    -- 灵息精华
    }

    -- 旧丹药迁移（凝X丹系列）
    -- 血煞丹 → 凝力丹（ATK），浩气丹 → 凝元丹（maxHP）
    save.ningliDanCount = save.xueshaDanCount or 0    -- 血煞丹 → 凝力丹
    save.ningyuanDanCount = save.haoqiDanCount or 0   -- 浩气丹 → 凝元丹
    save.ningjiaDanCount = save.ningjiaDanCount or 0   -- 新增，默认 0
    save.ninghunDanCount = save.ninghunDanCount or 0   -- 新增，默认 0
    save.ningxiDanCount = save.ningxiDanCount or 0     -- 新增，默认 0

    -- 清理旧字段
    save.xueshaDanCount = nil
    save.haoqiDanCount = nil

    -- 旧精华迁移（如有）→ 全部丢弃，旧精华存在时间极短
    if save.essence_counts.zhanpo_essence then
        save.essence_counts.zhanpo_essence = nil
    end
    if save.essence_counts.lingquan_essence then
        save.essence_counts.lingquan_essence = nil
    end

    save.version = 3
end
```

**迁移规则摘要**：

| 旧存档状态 | 新版开启声望等级 | 说明 |
|-----------|----------------|------|
| 有旧版进度（Ch2 已解锁） | 声望 1、2、3 | 对应乌堡令段（T3/T4/T5） |
| 旧版 T5+ 有解锁记录（Ch3 已解锁） | 额外开启声望 4、5 | 对应沙海令段（T6/T7） |
| 无旧版进度 | 不开启 | 新玩家正常走解锁流程 |

> - 仅血煞和浩气做迁移（旧版只有这两个阵营），青云/封魔为新阵营，从零开始
> - 迁移只标记 `unlocked`，不标记 `completed`（玩家需要重新打一遍以获得新版奖励）
> - 旧版进度 `progress` 字段保留不动，后续版本移除旧挑战时一并清理
> - **丹药迁移**：血煞丹→凝力丹（次数直接转移），浩气丹→凝元丹（次数直接转移）
> - **精华迁移**：旧精华（战魄/灵泉）全部丢弃（存在时间极短），新 5 种精华从 0 开始
