## 1. 需求摘要

本次只做音乐与掉落音效系统整理，范围如下：

1. 每个章节有且仅有一首 BGM，缺失章节需要补齐。
2. 登录界面继续使用第一章 BGM。
3. 系统菜单增加两个开关：
   - 音乐开关
   - 掉落音开关
4. 开关状态需要持久化，不能每次重启恢复默认。
5. 新增掉落音功能，仅在“稀有道具成功获得”时触发。
6. 掉落音先只做三类：
   - 神器碎片专属音效
   - 特殊装备掉落音效
   - 黑市其他道具掉落音效
7. 中洲城（chapter 101）已有独立 BGM，走同一套 `PlayChapterBGM` 逻辑，无需额外处理。

## 3. 目标行为定义

### 3.1 BGM

- `chapter 1 ~ chapter 5` 各自拥有一首 BGM。
- 登录界面固定使用第一章 BGM。
- 进入章节、切换章节、回登录界面时，BGM 按现有流程自动切换。
- 音乐开关关闭时：
  - 不播放新音乐
  - 当前 BGM 立即停止
- 音乐开关打开时：
  - 立即恢复当前场景对应 BGM

### 3.2 掉落音

- 掉落音只在“成功拿到物品”时触发，不在怪物掉落落地时触发。
- 仅对稀有掉落触发。
- 后触发的掉落音直接覆盖前一个（stop + play），不做队列排队。
- 如果同一批奖励同时命中多个类别，按优先级只播最高级别一个。

优先级固定为：

1. 神器碎片
2. 特殊装备
3. 黑市其他道具

### 3.3 开关持久化

- 新档默认：
  - `musicEnabled = true`
  - `dropVoiceEnabled = true`
- 老档兼容：
  - 缺字段时自动补默认值
- 玩家切换开关后：
  - 本次运行立刻生效
  - 后续保存进档
  - 重新进入角色后状态不丢失

## 4. 实施原则

### 4.1 不要继续把音频逻辑堆在 `main.lua`

远端 AI 需要把音乐和掉落音逻辑抽到独立系统模块，例如：

- `scripts/systems/AudioSystem.lua`

建议模块职责只保留：

- `Init`
- `PlayChapterBGM(chapterId)`
- `StopBGM`
- `SetMusicEnabled(enabled)`
- `IsMusicEnabled()`
- `SetDropVoiceEnabled(enabled)`
- `IsDropVoiceEnabled()`
- `PlayRareDropCue(cueType)`
- `ApplySavedSettings(settings)`
- `ExportSettings()`

### 4.2 不要把掉落音钩子埋进通用背包写入函数

禁止直接在以下函数里无差别播放掉落音：

- `InventorySystem.AddConsumable`
- `InventorySystem.AddItem`

原因：

- 商店购买
- GM 发物品
- 兑换码奖励
- 存档恢复
- 系统补发

这些路径都会用到通用入包函数，若直接埋钩子会导致误播。

### 4.3 不要直接复制受版权保护的经典旋律片段

需求里提到“参考经典掉落游戏音乐、恭喜发财一小段”，这里要明确：

- 代码侧只做 3 个 cue 位接线。
- 资源侧使用“项目方自备授权素材”或“后续人工替换音频文件”。
- 远端 AI 不负责生成或嵌入疑似侵权音频内容。

## 5. 数据方案

### 5.1 推荐方案

音频开关直接存入 `player` 存档结构，最小改动即可完成持久化。

在 `SerializePlayer / DeserializePlayer` 增加：

```lua
audioSettings = {
    musicEnabled = true,
    dropVoiceEnabled = true,
}
```

推荐理由：

- 改动面最小
- 不必扩顶层 `coreData` 新字段
- 现有角色长期状态已经都在 `player` 内，风格一致

### 5.2 兼容要求

- 老档无 `audioSettings` 时自动补默认值。
- `audioSettings` 非 table 时也回退默认值。
- 反序列化时要立即同步到运行时音频系统，而不是只还原字段不生效。

### 5.3 登录界面边界

这里有一个必须提前定清的边界：

- 如果音频设置只存在角色存档里，那么进入登录界面时，角色还没加载，理论上还不知道该用哪个开关状态。

因此建议远端 AI 采用二段式方案：

1. 角色存档中保留权威配置。
2. 同时维护一个“最近一次音频设置镜像”，供登录界面优先读取。

可接受的简化方案：

- 第一阶段先只做角色存档持久化。
- 登录界面始终默认播放第一章音乐。
- 进入角色后再应用该角色的实际设置。

如果要完全符合用户直觉，则应增加账号级镜像设置。

## 6. 分类规则

掉落音分类必须是“显式规则”，不要靠文案字符串模糊匹配。

### 6.1 神器碎片

判定规则：

- `consumableId` 包含 `_fragment_`

应覆盖：

- `rake_fragment_*`
- `bagua_fragment_*`
- `tiandi_fragment_*`
- `zhentu_fragment_*`

### 6.2 特殊装备

判定规则：

- `item.isSpecial == true`

补充兜底：

- 或者 `equipId` 在 `EquipmentData.SpecialEquipment` 中存在

### 6.3 黑市其他道具

判定规则：

- 物品存在于 `BlackMerchantConfig.ITEMS`
- 但不属于“神器碎片”
- 也不属于“特殊装备”

注意：

- 这里是“黑市其他道具”，不是“所有消耗品”。
- 只能按 `BlackMerchantConfig` 白名单判定，避免普通掉落材料误播。

## 11. 验收清单

### 11.1 音乐

- 登录界面启动后播放第一章 BGM。
- 第一章到第五章切换时，各章节能切到对应 BGM。

### 11.2 持久化

- 新档默认：音乐开，掉落音开。
- 旧档无字段时：自动补默认值，不报错。
- 在系统菜单关闭任一开关后保存退出，再次进入同一角色，状态保持不变。

### 11.3 掉落音

- 获得神器碎片时，播放“神器碎片”音。
- 获得特殊装备时，播放“特殊装备”音。
- 获得黑市白名单中的其他稀有道具时，播放“黑市其他道具”音。
- 同一波奖励同时包含多个类别时，只播一次最高优先级音效。
- 背包满导致未成功获得时，不播。
- 普通材料、普通消耗品、普通装备不播。

### 11.4 非目标链路

- 商店购买不播掉落音。
- 黑市购买不播掉落音。
- GM 发物品不播掉落音。
- 读档恢复不播掉落音。

### 12.1 登录界面静音问题

处理建议：

- 第二阶段如有需要，再做账号级音频设置镜像。

远端 AI 需要保证：

- 地面拾取链路和挑战直发链路各自只在自己的成功结算点播放。

### 15.2 掉落音播放策略：覆盖模式

原文档"同一批奖励只播一次"已修正为：

- **后触发覆盖前触发**：新的掉落音触发时，立即 `Stop()` 当前正在播放的掉落音，然后 `Play()` 新音效。
- 不做队列、不做延迟排队。
- 同一批内多类别仍取最高优先级只播一个（优先级规则不变）。
- 但两批之间若间隔极短，后一批直接覆盖前一批的音效。

实现要点：
```lua
-- AudioSystem 内维护单一 dropSource_
function AudioSystem.PlayRareDropCue(cueType)
    if not dropVoiceEnabled_ then return end
    if dropSource_:IsPlaying() then
        dropSource_:Stop()
    end
    local sound = cache:GetResource("Sound", DROP_CUE_MAP[cueType])
    if sound then
        dropSource_:Play(sound)
    end
end
```

### 15.3 掉落音音量规范

| 参数 | 值 | 说明 |
|------|-----|------|
| BGM gain | 0.35 | 已有，不变 |
| 掉落音 gain | 0.65 | 需明显但不盖过 BGM |
| 掉落音 SoundType | `"Effect"` | 与 BGM 的 `"Music"` 类型隔离 |

### 15.4 AudioSystem.Init 时机

```
Start()
  → Scene 创建
  → AudioSystem.Init()          ← 此时创建 BGM/掉落音的 SoundSource
  → 登录界面渲染
  → PlayBGM(1)                  ← 登录音乐
  → ...
  → 存档加载完成
  → AudioSystem.ApplySavedSettings(settings)  ← 应用角色存档中的开关状态
```

关键：`Init()` 在存档加载之前调用，默认开关为 `true`；存档加载后再覆盖为角色实际设置。
