# 情人节世界掉落活动（设计方案）

> 基准代码：当前已落地的端午活动（`dragonboat_2026`）
>
> 参考文档：`端午世界掉落活动（设计方案）.md` / `端午世界掉落活动（代码执行文档）.md` / `节日活动架构记录.md` / `万界黑商.md` / `宠物外观系统.md`
>
> 本轮审查口径：以 `scripts/` 中端午活动实装代码为准，文档只作为辅助参考；凡文档与代码不一致，以代码链路结论收口。
>
> 日期口径：当前时间为 2026-07-06。若按真实节日排期，情人节活动默认使用 `valentine_2027`；如果运营决定做 2026 补档活动，执行前统一把 `valentine_2027` 替换为 `valentine_2026`，但必须保证 eventId 与排行榜 key 不复用旧活动。

## 1. 基准结论

当前端午活动已经落地为完整的双开启物、双宝箱、积分排行、开奖记录、保底、BOSS 击杀里程碑、精品粽食品与宠物皮肤大奖结构。情人节活动不重构主链，继续复用这套骨架，但必须把端午代码中的硬编码节日文案和精品粽专用逻辑同步替换。

本轮变化不是单纯换文案，至少包含 6 个明确改动：

1. 替换两种抽奖开启物。
2. 不再产出 `仙界精品粽`，但保留精品粽黑市交易。
3. 端午两种抽奖开启物从黑市下架。
4. 保留 BOSS 击杀里程碑任务，但情人节从 0 重新计算。
5. 大奖皮肤改为根骨宠物皮肤 `pet_premium_gengu`。
6. 新增一种情人节专属食品，效果为永久 `悟性 +1`，定位对标 `仙界精品粽`。

关键安全口径：

1. 不删除端午历史数据，不清空端午排行榜、开奖记录、保底、里程碑 key。
2. 不清理玩家背包中的端午道具。
3. 不重置全局 `bossKills`，只使用情人节独立里程碑基线实现“重新开始计算”。
4. 新食品不得复用 `pillFortune` / `premiumZongEaten`，必须新增悟性食品专属字段。

## 2. 端午收尾原则

情人节上线时，端午按以下口径收尾：

1. `dragonboat_2026` 活动关闭，不再作为当前 `ACTIVE_EVENT`。
2. `dragonboat_colored_rope` 和 `dragonboat_sachet` 不再产出、不再开箱、不再进入当前活动奖池。
3. `dragonboat_colored_rope` 和 `dragonboat_sachet` 从 `BlackMerchantConfig.ITEMS` 下架；玩家库存仍保留，可按普通道具出售换金币。
4. `xianjie_premium_zong` 不再通过世界掉落、宝箱、里程碑、兑换码产出。
5. `xianjie_premium_zong` 保留在黑市：
   - 收购价 `10` 仙石
   - 出售价 `20` 仙石
   - 库存上限 `50`
6. `xianjie_premium_zong` 的食用链路、存档字段、角色面板显示继续保留，不做迁移或降级。
7. 端午兑换码 `DUANWU2026` 必须作废失效，且不能继续通过配置注册或云端旧状态发放 `dragonboat_colored_rope` / `dragonboat_sachet`；GM 测试按钮可保留，但不得进入普通玩家可触发路径。

### 2.1 端午代码审查结论

本节基于当前代码全量审查，不只引用端午设计文档。情人节执行时必须逐项对齐：

| 链路 | 端午现状 | 情人节一致性要求 |
| --- | --- | --- |
| 活动配置 | `scripts/config/EventConfig.lua` 的 `ACTIVE_EVENT.eventId = "dragonboat_2026"`，双开启物、奖池、保底、排行榜 key 全在同一配置块内 | 整块切换为 `valentine_2027`，不能复用 `evt_dragonboat_*` key |
| 世界掉落 | `scripts/systems/loot/rewards.lua` 第三段读取 `premiumZongDropRates` 后硬编码发放 `xianjie_premium_zong` | 必须改成通用 `rareFoodDrop.itemId`，否则情人节仍会掉粽子 |
| 奖池发奖 | `EventHandler.HandleOpenFudai` 根据 `openBoxes`、`smallBoxPool`、`bigBoxPool` 发奖，并写 `event_data[eventId].pity` | 小/大宝箱槽位 ID、保底 `targetId`、`jackpotIds` 必须同步换成情人节食品和根骨皮肤 |
| 开箱错误文案 | `EventHandler.lua` 无开启物时硬编码“辟邪香囊 / 端午彩绳” | 改为从 `GameConfig.EVENT_ITEMS[boxCfg.itemId].name` 取名 |
| 活动 UI 文案 | `EventFudaiTab.lua` 硬编码“端午彩绳”“辟邪香囊”“仙界精品粽”“悟性皮肤”；`EventMilestoneTab.lua` 顶部说明硬编码“达标可领取辟邪香囊” | 改为从活动配置和奖励配置动态取名，或至少替换为情人节文案 |
| 活动 NPC | `EventExchangeUI.lua` 默认 NPC 名、对话、头像仍是“粽仙使者 / 端午活动宝箱 / Textures/npc_zongxian.png” | `EventConfig.ACTIVE_EVENT.npc` 或默认值必须切换为情人节 NPC 与资源 |
| 黑市 | `BlackMerchantConfig.lua` 当前仍上架 `dragonboat_colored_rope`、`dragonboat_sachet`、`xianjie_premium_zong` | 下架两种端午开启物，保留精品粽，新增情人节三件商品 |
| 兑换码 | `RedeemCodes.lua` 存在 `DUANWU2026`，奖励彩绳 ×5 + 香囊 ×1；`RegisterRedeemCodesFromConfig()` 当前写入 `enabled = true`，未读取 `entry.disabled` / `entry.enabled` | 作废端午码时必须先修正注册逻辑或直接更新云端 `redeem_DUANWU2026.enabled=false`，再新增情人节同款码 |
| 里程碑 | `MilestoneLogic.lua` 与 `EventHandler.lua` 使用独立 key `evt_ms_<eventId>_<slot>`，`baseline` 在角色加载时初始化 | 新 `eventId` 会自然重新建基线，不得清空全局 `bossKills`，也不得复用端午 key |
| 食品使用 | 精品粽走 `batch_use.lua` 专用分支、`ConsumeConsumable()` 黑市锁、`SavePersistence.Save()` 立即存档 | 相思红豆糕必须复制完整使用合同，不能只在 `GameConfig` 加物品 |
| 食品存档 | `premiumZongEaten` 为权威，驱动 `pillFortune` 和 `pillCounts.zong`；`SaveSerializer` / `SaveLoader` / `SaveWriteService` 都有对齐校验 | 相思红豆糕新增独立权威次数字段，驱动 `pillWisdom` 和 `pillCounts.redbean_cake` |
| 皮肤 | `pet_premium_wuxing.hidden=false` 且已进入 `SkinShopConfig.SKIN_IDS`；`pet_premium_gengu.hidden=true` 且未入商店 | 根骨皮肤需 `hidden=false` 并加入商店列表，大奖重复补偿仍按 5000 灵韵 |
| 测试 | `scripts/tests/test_event_system.lua` 直接断言 `dragonboat_2026`、`dragonboat_colored_rope`、`dragonboat_sachet`、`xianjie_premium_zong`、`premiumZongDropRates`、`pet_premium_wuxing` | 情人节执行时必须同步改为 Valentine 断言，并新增兑换码作废/新码、食品存档、黑市上下架测试 |

情人节活动继续采用端午骨架：

1. BOSS 世界掉落 2 种开启物。
2. 两种开启物分别对应小宝箱 / 大宝箱。
3. 保留积分排行、全服开奖记录、双保底机制。
4. 不开启兑换系统。
5. 保留 10 档 BOSS 击杀里程碑任务，每档奖励 1 个大宝箱开启物。
6. 新增情人节专属食品 `相思红豆糕`，可食用、可黑市交易、可进入奖池。

## 4. 情人节物品设计

### 4.1 两种抽奖开启物

| 物品 ID | 名称 | 定位 | 来源 | 黑市价格 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `valentine_red_thread` | 同心红线 | 小宝箱开启物 | BOSS 世界掉落 | `1 / 2` | 常规产出，替代端午 `dragonboat_colored_rope` |
| `valentine_pair_jade` | 鸳鸯玉佩 | 大宝箱开启物 | 高阶 BOSS 世界掉落 / 里程碑奖励 | `6 / 12` | 高级开启物，替代端午 `dragonboat_sachet` |

### 4.2 专属食品

| 物品 ID | 名称 | 类型 | 来源 | 黑市价格 | 黑市库存 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| `valentine_xiangsi_redbean_cake` | 相思红豆糕 | 特殊丹药消耗品 | 高阶 BOSS 稀有独立掉落 / 红线宝箱大奖 / 玉佩宝箱稀有奖励 | `10 / 20` | `50` | 食用后永久 `悟性 +1`，单角色最多食用 `10` 个，可上架黑市 |

定位说明：

1. `同心红线` 维持高频、低价值、活动辨识度强的常规开启物定位。
2. `鸳鸯玉佩` 维持高阶开启物定位，主要来自高阶 BOSS 与里程碑任务。
3. `相思红豆糕` 对标 `仙界精品粽`，但属性从 `福缘 +1` 改为 `悟性 +1`。
4. `相思红豆糕` 与 `仙界精品粽` 是两个独立食品，不共享食用次数，不共享存档字段。
5. 采用“红豆寄相思”的传统意象，情人节辨识度高于通用糕点；视觉上做成红豆灵糕/心结灵纹，不走现代巧克力包装。

## 5. 世界掉落设计

### 5.1 两段开启物掉落

情人节前两段掉落继续复用端午实现方式，只做 ID 同构替换：

1. `dragonboat_colored_rope -> valentine_red_thread`
2. `dragonboat_sachet -> valentine_pair_jade`

第一段：按 `chapter + monster.category` 掉 `valentine_red_thread`。

第二段：按 `monsterOverrides > realmDropRates` 掉 `valentine_pair_jade`。

掉率沿用端午数值，不额外上调。

### 5.2 相思红豆糕独立掉落

当前代码第三段掉落是 `premiumZongDropRates` 并硬编码发放 `xianjie_premium_zong`。情人节必须把这段泛化，否则会继续产出粽子。

建议配置改为：

```lua
rareFoodDrop = {
    itemId = "valentine_xiangsi_redbean_cake",
    rates = {
        yuanying_1 = 0.0005, yuanying_2 = 0.0005, yuanying_3 = 0.0005,
        huashen_1  = 0.0008, huashen_2  = 0.0008, huashen_3  = 0.0008,
        heti_1     = 0.0010, heti_2     = 0.0010, heti_3     = 0.0010,
        dacheng_1  = 0.0012, dacheng_2  = 0.0012, dacheng_3  = 0.0012, dacheng_4 = 0.0012,
        dujie_1    = 0.0015, dujie_2    = 0.0015, dujie_3    = 0.0015, dujie_4   = 0.0015,
        ch5_sword_zhu  = 0.0020,
        ch5_sword_xian = 0.0020,
        ch5_sword_lu   = 0.0020,
        ch5_sword_jue  = 0.0020,
    },
}
```

判定规则：

1. 仅 BOSS 参与，不含普通怪和精英怪。
2. 仅 `realm >= yuanying_1` 的 BOSS 参与。
3. 四仙剑保留 typeId 覆盖口径。
4. 与红线 / 玉佩互不排斥，允许同一只 BOSS 同时掉落开启物和相思红豆糕。
5. 活动配置中不得再保留 `premiumZongDropRates`，避免误产出粽子。

### 7.3 重新开始计算的实现口径

不得清空或回写全局 `bossKills`。推荐继续使用当前服务端独立里程碑 key：

```lua
evt_ms_valentine_2027_<slot> = {
    baseline = 活动开始时该角色的全局 bossKills,
    claimed = {
        ["300"] = true,
    },
}
```

活动击杀数：

```lua
eventBossKills = math.max(0, currentBossKills - baseline)
```

执行细节：

1. 活动启用后，玩家首次加载角色时若情人节里程碑 key 不存在，应立即初始化 baseline。
2. 查询和领取时都读取该独立 key。
3. 当前击杀数取 `max(saveData.bossKills, ClientBossKills)`，沿用现有“服务端基线 + 客户端实时击杀”口径，避免等待存档导致的领取延迟。
4. 若 key 缺失，领取接口不现场建基线，直接提示“里程碑数据未就绪，请重新进入活动面板”，避免玩家绕过初始化。
5. 端午 `evt_ms_dragonboat_2026_<slot>` 不迁移、不清理、不复用。

## 9. 相思红豆糕使用链路

`相思红豆糕` 对标 `仙界精品粽`，但必须独立建字段和校验。

### 9.1 存档字段

建议新增：

1. `player.pillWisdom`
2. `player.premiumRedbeanCakeEaten`

推荐关系：

1. 每食用 1 个 `相思红豆糕`
2. `player.premiumRedbeanCakeEaten += 1`
3. `player.pillWisdom += 1`
4. 两者上限均为 `10`

可选镜像：

```lua
player.pillCounts.redbean_cake = player.premiumRedbeanCakeEaten
```

不能复用：

1. 不能复用 `pillFortune`，那是精品粽的福缘字段。
2. 不能复用 `premiumZongEaten`，那是精品粽的次数权威字段。
3. 不能写进 `daoTreeWisdom` / `wineWisdom` / `artifactCh4Wisdom`，这些都有独立来源语义。

### 9.2 属性结算

`Player:GetTotalWisdom()` 需要纳入：

```lua
+ (self.pillWisdom or 0)
```

角色信息面板需要显示：

1. `悟性` 总值中实际生效。

### 9.3 使用逻辑

建议沿用精品粽的专用使用链路：

1. `scripts/systems/inventory/batch_use.lua` 新增 `valentine_xiangsi_redbean_cake` 分支。
2. 使用时通过 `ConsumeConsumable()` 扣除，自动受黑市锁保护。
3. 按 `10 - premiumRedbeanCakeEaten` 夹紧本次可食用数量。
4. 成功后更新 `premiumRedbeanCakeEaten`、`pillWisdom`、`pillCounts.redbean_cake`。
5. 成功后立即请求保存，避免服务端整包回写覆盖刚食用的永久收益。
6. UI 提示：
   - `食用 1 份相思红豆糕，悟性永久+1`
   - `已食用 X/10`
   - `已达食用上限`

### 9.4 服务端校验

需要在 `SaveWriteService` / `SaveLoader` 中做只读或对齐式保护：

1. `premiumRedbeanCakeEaten` 上限 `10`。
2. `pillWisdom` 上限 `10`。
3. `pillWisdom` 必须等于 `premiumRedbeanCakeEaten`。
4. `pillCounts.redbean_cake` 必须等于 `premiumRedbeanCakeEaten`。
5. 校验逻辑不得删除字段或截断其他 `pillCounts` 内容。

## 11. 兑换码设计

### 11.1 端午兑换码作废

当前 `scripts/config/RedeemCodes.lua` 中存在端午兑换码：

```lua
{
    code = "DUANWU2026",
    rewards = {
        { type = "item", id = "dragonboat_colored_rope", amount = 5 },
        { type = "item", id = "dragonboat_sachet", amount = 1 },
    },
    maxUses = 100,
    perUser = 1,
}
```

正式切换情人节前必须让该码失效：

1. `RedeemCodes.lua` 中保留历史条目，但标记为 `enabled = false` 或 `disabled = true`，备注改为 `[已注销] 端午活动：彩绳x5+香囊x1`。
2. 修正 `scripts/network/RedeemHandler.lua` 的 `RegisterRedeemCodesFromConfig()`：当前注册时写死 `enabled = true`，必须改为读取配置，例如 `enabled = (entry.enabled ~= false and entry.disabled ~= true)`。
3. 如果线上云端已经存在 `redeem_DUANWU2026.enabled=true`，需要通过 GM/后台一次性把云端码状态改成 `enabled=false`；只改本地配置不会自动让已注册云端旧码失效。
4. 不删除该条目，不删除历史兑换记录，不清空 `redeem_global_DUANWU2026` quota。

### 11.2 情人节同款兑换码

新增同款活动码：
