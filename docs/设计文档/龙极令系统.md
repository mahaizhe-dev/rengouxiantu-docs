## 一、概述

### 1.1 设计目标

在龙神圣器打造界面（DragonForgeUI）新增两种打造功能：

| 功能 | 产物 | 材料核心 | 定位 |
|------|------|---------|------|
| **龙极令打造** | 龙极令（攻击型法宝，T9 灵器固定） | 4 龙鳞 + 1000 太虚令 | 法宝体系的攻击力向补充，拥有独有的扇形 AOE 技能 |
| **灵器打造** | 随机 T9 灵器装备 | 4 龙鳞 + 10 万金币 | 龙鳞材料的消耗出口，提供高品质装备获取途径 |

### 1.2 三种打造的核心区别 🔴

**龙神圣器打造界面包含三类打造，操作模式和数据来源完全隔离：**

| 打造类型 | 操作模式 | 配方数据 | 消耗数据 | 产物去向 |
|---------|---------|---------|---------|---------|
| 龙极武器（Tab 1-5） | **装备栏** — 必须装备灵器武器才能打造 | `DRAGON_FORGE_RECIPES` | `DRAGON_FORGE_COST` | 替换当前装备的灵器武器 |
| 龙极令（Tab 6） | **背包** — 材料从背包取，产物放入背包 | `LONGJI_FORGE_RECIPE` | `LONGJI_FORGE_COST` | `InventorySystem.AddItem()` 放入背包 |
| 灵器铸造（Tab 7） | **背包** — 材料从背包取，产物放入背包 | 无（固定规则） | `LINGQI_FORGE_COST` | `InventorySystem.AddItem()` 放入背包 |

**关键差异**：
- 龙极武器：操作对象是**已装备的灵器武器**，打造后直接替换装备栏中的武器
- 龙极令 & 灵器铸造：操作完全在**背包内**完成，材料从背包消耗，产物放入背包空位
- 三种打造的配方和消耗数据结构**完全独立**，不共享 cost 对象，防止串数据

### 1.3 龙极令在法宝体系中的位置

```
法宝体系
├── 仙缘属性法宝（阵营挑战获得，T3-T9 渐进成长）
│   ├── 血海图（血煞盟·殷无咎）→ 悟性 wisdom,  xianyuan 公式, base=5
│   ├── 浩气印（浩气宗·苏浩然）→ 福缘 fortune,  xianyuan 公式, base=5
│   ├── 青云塔（青云门·云无涯）→ 根骨 constitution, xianyuan 公式, base=5
│   └── 封魔盘（封魔殿·凌战）→ 体魄 physique,  xianyuan 公式, base=5
│
└── 攻击型法宝（第四章铸造，T9 灵器一步到位）
    └── 龙极令（四龙材料铸造）→ 攻击力 atk, standard 公式, base=2   ← 新增
```

**定位差异**：
- 仙缘法宝提供仙缘属性（间接影响多项战斗属性）+ 各自独有技能
- 龙极令提供直接的攻击力加成 + 龙息扇形 AOE 技能
- 全部法宝共用 `exclusive` 槽位，同时只能装备 1 件

### 1.4 攻击力法宝基础值依据

根据阵营挑战系统现状文档 §二·法宝主属性基础值表（第 469 行）：

> | **法宝** | **atk** | **2** | **6** | **12** | **21** | **34** |

攻击法宝 `mainStatBase = 2`，使用 standard 公式。

| 属性 | 规则 |
|------|------|
| **等级 Tier** | T9（固定） |
| **品质** | **灵器 (cyan) 固定**（必出灵器品质） |
| **主属性** | 攻击力 atk |
| **主属性公式** | standard 标准公式：`base × TIER_MULTIPLIER[9] × qualityMult` |
| **主属性数值** | `2 × 17.0 × 1.8 = 61.2 → 61`（取 floor） |
| **副属性** | 按灵器品质规则随机生成 3 条（从标准副属性池抽取） |
| **仙缘附属性** | 1 条随机仙缘属性（灵器品质标准行为） |
| **技能** | 龙息（见 §三） |
| **产物去向** | 放入背包（`InventorySystem.AddItem()`） |

### 2.4 主属性数值定位

```
T9 灵器品质 攻击力对比：

  武器 (base=5):  5 × 17.0 × 1.8 = 153
  戒指 (base=3):  3 × 17.0 × 1.8 = 91.8 → 91  (×2 = 182)
  龙极令 (base=2): 2 × 17.0 × 1.8 = 61.2 → 61

  合计攻击力上限: 153 + 182 + 61 = 396 (武器+双戒+龙极令)
```

龙极令攻击力（61）约为单戒指的 67%，约为武器的 40%，是法宝槽位对攻击力的补充而非主力。

## 三、专属技能：龙息

### 3.1 技能概述

| 属性 | 值 |
|------|-----|
| 技能 ID | `dragon_breath` |
| 技能名 | 龙息 |
| 图标 | `"🐲"` |
| 类型 | `xianyuan_cone_aoe` |
| 伤害属性 | 仙缘四属性之和（wisdom + fortune + constitution + physique） |
| 伤害系数 | `damageCoeff = 2.0` |
| 伤害公式 | `rawDmg = floor(xianyuan_sum × damageCoeff)` → `CalcDamage(rawDmg, monster.def)` |
| 形状 | 前方扇形 (`shape = "cone"`) |
| 扇形角度 | 90° (`coneAngle = 90`) |
| 作用范围 | 2.5 瓦片 (`range = 2.5`) |
| 最大目标数 | 8 (`maxTargets = 8`) |
| 冷却时间 | 15 秒 (`cooldown = 15`) |
| 暴击 | 可暴击 (`canCrit = true`) |
| 技能绑定 | 槽位 4（与所有法宝技能一致） |

### 3.2 设计思路

- **参考**：魔兽世界经典 AOE 技能「龙息术」— 前方扇形喷吐，范围瞬间伤害
- **伤害来源**：取仙缘四属性之和（而非单一属性），鼓励全面发展仙缘属性
- **走 CalcDamage**：与套装 AOE（SetAoeSystem）相同的伤害管线，不走真实伤害，受目标防御影响
- **可暴击**：通过暴击率/暴击伤害进一步放大输出
- **90° 扇形**：比血海翻涌（120°）窄但射程更远（2.5 vs 1.8），需要更精确的瞄准

### 3.3 视觉效果

- **颜色**：金色渐变扇形 AOE（从中心亮金到边缘暗金渐变）
- **粒子**：粒子从玩家位置**向外扩散**，模拟龙息喷吐效果
- **effectColor**：`{255, 200, 50, 255}`（主色金龙色）→ 边缘渐变 `{200, 120, 20, 180}`
- **实现要点**：使用 NanoVG 扇形渐变填充 + 粒子系统向外发射，粒子生命周期 0.3-0.5s

### 3.4 DPS 预估

以典型 T9 角色数值（四仙缘各 60，总和 240）为基准：

```
rawDmg = floor(240 × 2.0) = 480
CalcDamage(480, monster.def) 
  → 若 DEF=200：480² / (480+200) = 338.8 → ×(0.9~1.1) ≈ 305~373
  → 若 DEF=100：480² / (480+100) = 397.2 → ×(0.9~1.1) ≈ 357~437

DPS = 单次伤害 / CD = ~350 / 15 ≈ 23.3 (单目标)
多目标 DPS = 23.3 × 命中数（最多 8）
```

**对比参考**：
- 血海翻涌（T3 数值）：即时 60%ATK + DoT 10%×4s = 1.0×ATK / 12s ≈ 0.083×ATK/s
- 龙息：单发 ~350 / 15s ≈ 23.3/s，但作用于更多目标
- 定位：低频高爆发 AOE，适合清怪和多目标场景

### 3.5 SkillData 模板

```lua
dragon_breath = {
    id = "dragon_breath",
    name = "龙息",
    icon = "🐲",
    description = "龙威喷吐，向前方90°扇面释放龙息，对区域内敌人造成仙缘伤害",
    type = "xianyuan_cone_aoe",
    unlockLevel = 1,
    cooldown = 15.0,
    range = 2.5,
    shape = "cone",
    coneAngle = 90,
    maxTargets = 8,
    -- 伤害计算
    damageAttribute = "xianyuan_sum",    -- 取四仙缘属性之和
    damageCoeff = 2.0,                   -- 伤害系数
    usesCalcDamage = true,               -- 走 CalcDamage(rawDmg, def)，非真实伤害
    canCrit = true,                      -- 可暴击
    effectColor = {255, 200, 50, 255},   -- 金色主色
    effectColorEdge = {200, 120, 20, 180}, -- 边缘渐变色
    particleDirection = "outward",       -- 粒子向外扩散
    isEquipSkill = true,
    sourceSlot = "exclusive",
    -- 不分阶（与现有法宝技能一致）
    tierScaling = nil,
    descriptionProvider = function(skill, maxHp, gourdTier)
        return string.format(
            "向前方90°扇面喷吐龙息，对最多%d个敌人造成仙缘伤害（仙缘之和×%.1f），可暴击。CD: %.0f秒",
            skill.maxTargets, skill.damageCoeff, skill.cooldown
        )
    end,
},
```

### 3.6 伤害管线实现要点

龙息引入了新的 `damageAttribute = "xianyuan_sum"`，需要在技能伤害处理中新增：

```lua
-- 属性获取器扩展（SkillSystem 或 CombatSystem 中）
local SKILL_ATTRIBUTE_GETTERS = {
    -- 单属性（复用 SetAoeSystem 的 ATTRIBUTE_GETTERS）
    wisdom       = "GetTotalWisdom",
    fortune      = "GetTotalFortune",
    physique     = "GetTotalPhysique",
    constitution = "GetTotalConstitution",
    -- 组合属性（新增）
    xianyuan_sum = function(player)
        return player:GetTotalWisdom()
             + player:GetTotalFortune()
             + player:GetTotalPhysique()
             + player:GetTotalConstitution()
    end,
}
```

伤害计算流程：
```
1. 获取 xianyuan_sum = wisdom + fortune + constitution + physique
2. rawDmg = floor(xianyuan_sum × damageCoeff)
3. actualDmg = CalcDamage(rawDmg, monster.def)
4. if canCrit and roll_crit: actualDmg = floor(actualDmg × critMultiplier)
5. monster:TakeDamage(actualDmg, player)
```

## 四、灵器打造

### 4.1 打造入口

- **NPC**：神真子（复用 DragonForgeUI）
- **操作方式**：DragonForgeUI 新增「灵器铸造」页签（Tab 7）
- **操作模式**：**背包内操作** — 材料从背包消耗，产物放入背包空位
- **可重复打造**：每次消耗完整配方，可无限次打造

### 4.2 打造配方

| 材料 | 数量 | 来源 |
|------|------|------|
| 封霜龙鳞 (`dragon_scale_ice`) | 1 | 封霜应龙掉落（1%） |
| 渊蛟龙骨 (`dragon_scale_abyss`) | 1 | 堕渊蛟龙掉落（1%） |
| 焚天龙焰 (`dragon_scale_fire`) | 1 | 焚天蜃龙掉落（1%） |
| 蚀骨龙牙 (`dragon_scale_sand`) | 1 | 蚀骨螭龙掉落（1%） |
| 金币 | 100,000 | — |

### 4.3 打造产出规则

| 属性 | 规则 |
|------|------|
| **等级 Tier** | T9（固定） |
| **品质** | 灵器 (cyan) 固定 |
| **装备槽位** | 随机 9 选 1（与 `STANDARD_SLOTS` 一致，含 necklace） |
| **Ring 处理** | 不区分 ring1/ring2，与掉落系统一致 |
| **套装概率** | 30% 套装灵器，70% 普通灵器 |
| **主/副属性** | 按槽位标准规则生成（复用现有 `GenerateRandomEquipment` / `GenerateSetEquipment`） |
| **产物去向** | 放入背包（`InventorySystem.AddItem()`） |

**槽位池**（与掉落系统完全一致，无葫芦/法宝/披风）：
```lua
STANDARD_SLOTS = {"weapon", "helmet", "armor", "shoulder", "belt", "boots", "ring1", "ring2", "necklace"}
```

### 4.4 套装灵器说明

当命中 30% 套装概率时：
- 从 4 套（血煞 / 青云 / 封魔 / 浩气）中随机选取
- 调用 `GenerateSetEquipment(monsterLevel, setId, slotOverride)` — 已硬编码 T9+cyan
- 产出的装备带有对应套装标记，可与其他同套装装备触发套装效果

当为普通灵器（70%）时：
- 走标准灵器生成流程，T9 + cyan 品质 + 随机槽位
- 标准灵器生成流程

### 4.5 设计意图

- **龙鳞材料消耗出口**：除龙极武器（500万金）和龙极令（1000太虚令）外，提供更低门槛的材料消耗方式
- **套装获取途径**：现有套装装备主要来源于特定掉落，灵器打造提供 30% 概率的额外途径
- **金币消耗**：10 万金币相比龙极武器（500 万）大幅降低，使材料本身成为核心门槛

## 八、存档相关处理

### 8.1 是否需要存档版本迁移？

**结论：不需要递增 `CURRENT_SAVE_VERSION`（当前 v18），不需要写迁移函数。**

原因分析：

| 新增内容 | 存档影响 | 是否需要迁移 |
|---------|---------|-------------|
| `FabaoTemplates.fabao_longjiling` | 纯新增模板条目，不修改已有模板 | ❌ 不需要 |
| `SkillData.dragon_breath` | 纯新增技能定义，不修改已有技能 | ❌ 不需要 |
| `LONGJI_FORGE_COST` / `LINGQI_FORGE_COST` | 纯新增配置常量 | ❌ 不需要 |
| 打造产出的装备/法宝 | 走现有序列化流程，字段结构不变 | ❌ 不需要 |
| 旧存档玩家 | 从未打造过龙极令/灵器 → 无新字段 → 正常加载 | ❌ 不需要 |

**关键依据**：
- 打造产物使用的 `CreateFabaoEquipment` / `GenerateRandomEquipment` / `GenerateSetEquipment` 生成的装备结构与现有装备完全一致
- `mainStatType = "atk"` + `mainStatFormula = "standard"` 组合在 `CreateFabaoEquipment` 的 else 分支已被支持
- 不修改任何已有存档字段，不改已有属性公式

### 8.2 打造后存档触发机制

打造完成后必须立即触发存档，防止断线导致材料已扣但产物未保存：

```lua
-- 打造成功后
EventBus.Emit("save_request")  -- 触发存档请求
-- SaveSystem 会在 SAVE_DEBOUNCE_INTERVAL（3秒）内合并多次请求
```

**与现有 DoForge() 一致**：龙极武器打造也是打造完立即 `EventBus.Emit("save_request")`。

### 8.3 背包满时的处理

打造前必须检查背包空位：

```lua
local freeSlots = InventorySystem.GetFreeSlots()
if freeSlots <= 0 then
    -- 提示"背包已满，请先清理背包"
    -- 禁用打造按钮
    return
end
```

**龙极武器不需要此检查**（因为直接替换装备栏），但龙极令和灵器铸造必须检查（产物进背包）。

### 8.4 材料扣除的原子性

扣除材料和生成产物必须在同一同步流程中完成：

```lua
-- 伪代码 — DoForgeLongji
local function DoForgeLongji()
    -- 1. 检查所有前置条件
    if not CheckMaterials(LONGJI_FORGE_COST) then return end
    if InventorySystem.GetFreeSlots() <= 0 then return end
    
    -- 2. 扣除材料（先扣材料再生成产物，与 DoForge 一致）
    DeductMaterials(LONGJI_FORGE_COST)
    
    -- 3. 生成产物
    local fabao = LootSystem.CreateFabaoEquipment("fabao_longjiling", 9, "cyan")
    InventorySystem.AddItem(fabao)
    
    -- 4. 存档
    EventBus.Emit("save_request")
    
    -- 5. UI 反馈
    ShowForgeSuccessPopup(fabao)
end
```

### 9.2 对数值平衡的影响

| 维度 | 分析 |
|------|------|
| **攻击力膨胀** | 龙极令 atk=61，相当于一个低 base 法宝的攻击力贡献。占用 exclusive 槽位后放弃了仙缘属性加成（4 个法宝各提供 wisdom/fortune/constitution/physique），属于 trade-off 选择而非纯增量 |
| **龙息 AOE DPS** | 单目标 ~23.3 DPS（CD15s），对比主手武器自动攻击 DPS 偏低。多目标场景（8目标）才有优势，但 90° 窄角+2.5范围限制了覆盖率 |
| **灵器打造对装备获取的影响** | 材料门槛极高（4 龙鳞各 1% 掉率），产出 T9 灵器品质与现有掉落上限一致，不引入超标装备。30% 套装概率提供定向获取途径但受材料稀缺度限制 |
| **太虚令消耗** | 1000 太虚令门槛较高，与八卦阵掉率匹配后约需 X 天积累（取决于八卦阵日均掉落量）。作为第四章深度内容的消耗锚点合理 |
| **金币通缩** | 灵器铸造每次消耗 10 万金币（远低于龙极武器 500 万），对经济系统影响可控 |

| 模块 | 影响范围 | 说明 |

### 9.4 回滚方案

如果上线后发现严重问题：

1. **数值过强**：热更调整 `damageCoeff`（龙息系数）、`LINGQI_FORGE_SET_CHANCE`（套装概率）
2. **材料过多/过少**：热更调整 `LONGJI_FORGE_COST.taixu_token`、`LINGQI_FORGE_COST.gold`
3. **紧急下架**：在 DragonForgeUI 中隐藏 Tab 6/7（数据结构不受影响，已打造的装备保留）

## 十二、风险评估

| 风险 | 级别 | 应对措施 |
|------|------|---------|
| `mainStatType="atk"` + standard 公式在 CreateFabaoEquipment 中可能未被正确处理 | **中** | 代码审查确认 else 分支存在且可处理，GM 测试验证数值 |
