# 角色与怪物碰撞轻量优化执行方案

> 文档状态：第二轮参数调整已实现（BOSS低幅参与、普通怪加快收敛 v5）
>
> 创建日期：2026-07-14
>
> 适用范围：主世界、挑战、试炼、镇狱塔及复用 `Player` / `Monster` 实体的本地战斗场景
>
> 核心原则：只缓解卡住和完全重叠，不追求严格物理、不追求绝对分散
>
> 优先级：性价比 > 手感改善幅度 > 物理准确性

---

## 1. 结论摘要

当前角色与怪物均未使用引擎刚体物理，而是采用 Lua 世界坐标和瓦片通行查询。

本次不更换移动系统，不接入完整物理引擎，不重做寻路，也不引入强制队形。首版只做两项小改动：

1. 玩家碰撞从四角矩形检测改为相同尺寸的轻量椭圆脚部检测。
2. 仅对正在与玩家或宠物交战的近身普通、精英及非副本权威 BOSS执行低频、限数量、限配对的弱分离；无法安全分散时允许保持部分甚至完全重叠。

首版明确接受以下结果：

- 怪物之间仍然可以重叠。
- 怪物在墙角、窄路、固定机制或大量聚集时可以完全重叠。
- 远处、待机和巡逻怪物完全保持当前行为。
- 不要求每只怪物头像、名字和身体都完整可见。
- 角色仍然会被墙阻挡，只减少绕角和单格入口处的生硬卡顿。
- 不保证所有极端角度、低帧率尖峰和异常地图结构都能自动脱困。

---

## 2. 需求边界

### 2.1 必须达到

1. 玩家绕单个阻挡格边缘移动时，比当前更不容易被尖角挂住。
2. 玩家进入一格宽通道时，对接近角度和横向偏差的容忍度略有提升。
3. 镇岳与其他职业使用相同碰撞规则，不制作职业专属通行能力。
4. 多只普通、精英或非副本权威 BOSS正在围攻玩家或宠物时，大多数开放地形下不再长期保持完全同坐标。
5. 怪物分离不得把怪物推出阻挡格、固定站位或关键技能位置。
6. 性能成本可测、可限、可快速关闭。

### 2.2 明确不要求

- 不要求角色获得自动寻路或自动贴墙导航。
- 不要求玩家在所有斜角都能无条件进入单格通道。
- 不要求角色碰撞轮廓与精灵每一帧完全一致。
- 不要求怪物形成整齐包围圈。
- 不要求怪物之间完全没有交叠。
- 不要求墙边怪物自动寻找新的空位。
- 不要求怪物为了分散而重新寻路。
- 不要求玩家与怪物之间产生实体推挤。
- 不处理远离玩家、待机或巡逻怪物的重叠。
- 不引入 Bullet、Box2D、NavMesh、Boids 或群体寻路系统。

## 3. 改动前实现基线

### 3.1 玩家碰撞

`scripts/entities/Player.lua` 改动前配置：

```lua
self.colHalfW = 0.3
self.colHalfH = 0.15
self.collisionOffsetY = 0.525
```

改动前移动流程：

1. 根据输入方向计算目标 `newX/newY`。
2. 用碰撞矩形的四个角调用 `gameMap:IsWalkable`。
3. 固定先测试 X，再使用更新后的 X 测试 Y。
4. 两轴都失败时累计 `stuckTimer`。

主要问题：

- 一格通道宽度为 `1.0`，碰撞宽度为 `0.6`，理论左右容差各为 `0.2` 格。
- 四角矩形在阻挡格边角处是完全尖锐的二值判定。
- 固定 X→Y 顺序会让不同接近方向产生不同结果。
- 普通墙角卡顿时当前位置仍然合法，现有 `Unstuck` 通常会立即返回。
- 镇岳逻辑碰撞体与其他职业相同，但精灵实体轮廓更宽，因此视觉上更容易感到挂墙。

### 3.2 怪物碰撞

`scripts/entities/Monster.lua` 当前移动只检测：

```lua
gameMap:IsWalkable(newX, newY)
```

怪物只拥有逻辑中心点，没有参与移动计算的实体半径。

`bodySize` 和 `MONSTER_RADIUS_SCALE` 只影响渲染，不参与移动和怪物间碰撞。因此多个怪物追逐同一目标时，完全相同坐标属于当前合法状态。

### 3.3 网络边界

项目虽然启用多人构建，但普通主世界移动和怪物 AI 在客户端 `main.lua` 中运行。

多人副本会周期上报玩家最终位置，当前并不存在需要同步修改的服务端角色移动碰撞副本。

首版怪物弱分离默认不处理 `isDungeonBoss`，避免未来多客户端本地模拟产生额外位置差异。

---

## 4. 总体方案

```text
玩家输入
  -> 计算目标位移
  -> 椭圆脚部占位检测
  -> 保持现有 X→Y 分轴滑行
  -> 更新玩家位置

怪物 AI 更新
  -> 保持现有追击/巡逻/攻击逻辑
  -> 每 0.1 秒收集近身 chase/attack 怪物
  -> 对封顶的小规模列表执行两两检查
  -> 累计小幅修正
  -> 目标位置可走才应用
  -> 分不开则保持原位
```

## 5. 玩家轻量圆角碰撞

### 5.1 首版参数

首版保持现有碰撞外接尺寸，只改变边角形状：

| 参数 | 当前 | 首版建议 | 说明 |
|---|---:|---:|---|
| 横向半径 | 0.30 | 0.30 | 总宽保持 0.60 |
| 纵向半径 | 0.15 | 0.15 | 总高保持 0.30 |
| Y 偏移 | 0.525 | 0.525 | 不改变区域判断和脚部锚点 |

只有椭圆圆角改善不足时，第二阶段才考虑轻微缩小到 `0.29 / 0.145`。首版不同时修改形状和尺寸，便于判断实际收益。

### 5.2 椭圆与瓦片检测

新增统一的玩家占位判断，例如：

```lua
CanOccupy(gameMap, playerX, playerY)
```

检测流程：

1. 得到碰撞中心 `cx = playerX`、`cy = playerY + collisionOffsetY`。
2. 按当前地图半格边界规则枚举椭圆覆盖的周边瓦片。
3. 对每个不可通行瓦片，求椭圆中心到瓦片矩形的最近点。
4. 使用归一化椭圆距离判断是否相交。

判定公式：

```text
(dx / radiusX)^2 + (dy / radiusY)^2 <= 1 + epsilon
```

实现要求：

- 瓦片 `(N,N)` 按闭合碰撞边界 `[N-0.5, N+0.5]` 处理。
- 地图点查询仍使用与 `GameMap:GetTile` 一致的 `floor(value + 0.5)` 规则；椭圆覆盖范围的下界使用 `ceil(min - 0.5)`、上界使用 `floor(max + 0.5)`，保证四个方向的精确相切都不会漏检。
- 使用很小的固定 `epsilon`，对精确相切采取保守阻挡，避免对角夹缝漏穿。
- 不使用 `sqrt`。
- 不创建临时 Vector 或 table。
- 不使用少量采样点伪装圆形。
- 瓦片可走性仍统一调用 `gameMap:IsWalkable`，保留封印阻挡层。
- 半径小于 0.5 时，单次占位通常只需检查最多 4 个瓦片。

### 5.3 首版保持现有分轴顺序

首版继续使用当前的：

```text
先 X，后 Y
```

原因：

- 本次主要问题是矩形尖角，椭圆化是最直接收益。
- 双顺序会引入两套候选位置、评分和稳定性处理。
- `GameMap:IsWalkable` 当前封印查询会创建字符串 key，不应在未证明必要前主动增加查询次数。

只有椭圆化后仍能稳定复现明显的方向偏置，才在第二阶段增加 Y→X 候选。

### 5.4 首版不做移动小步进

小步进有助于低帧率和高速移动，但会增加循环次数和实现面。

由于当前基础速度为每秒 4 格，首版先保留单帧位移模式。只有以下测试失败时再增加最多 2～4 个子步：

- 30 FPS 下单格入口明显比 60 FPS 更难进入。
- 高移速装备导致穿越可行窗口。
- `dt` 波动导致绕角结果明显不稳定。

如果后续增加，必须设置单帧最大子步数，禁止随异常 `dt` 无限增长。

### 5.5 首版改动范围

首版椭圆占位接入玩家普通移动和脱困判断；脱困的搜索策略保持不变。

不改变以下逻辑的策略：

- `Player:Unstuck` 仍沿原有方向、回退点和邻近瓦片中心搜索；仅把内部的可站立判断统一复用 `CanOccupy`，避免椭圆允许的合法墙角位置被旧 AABB 误判后传送。
- `Player:ApplyKnockback` 保持原终点检测策略，不新增路径扫掠。

原因：

- `Unstuck` 的搜索策略本身不属于本次目标，但它必须和普通移动使用同一占位定义，否则会把椭圆判定下的合法位置误判为非法。
- 击退可能一次跨越一格以上，只替换终点占位无法解决路径穿墙；若治理需要额外子步或扫掠，不属于本次低复杂度目标。

首版不修改：

- 玩家速度公式
- 输入归一化
- 角色动画
- 相机
- 地图瓦片定义
- `GameMap:IsWalkable` 的语义

## 6. 怪物弱分离

### 6.1 行为定义

怪物弱分离是一个可失败的视觉和移动改善层，不是碰撞约束。

它只表达：

> 当附近怪物过度接近，且存在低成本可行位置时，尝试让它们缓慢错开一点。

它不表达：

> 所有怪物必须保持最小距离。

以下均为合法结果：

- 两只怪仍有明显重叠。
- 多只怪只散开其中一部分。
- 墙角和窄路中的怪物完全不动。
- 超出处理预算的怪物维持原状。
- 固定怪、施法怪和副本权威 BOSS完全重叠。

### 6.2 执行频率与范围

首版建议：

| 参数 | 建议值 |
|---|---:|
| 更新间隔 | 0.10 秒 |
| 处理状态 | `chase`、`attack` |
| 处理范围 | 玩家周围 8 格 |
| 每轮最大怪物数 | 16 |
| 每轮最大配对数 | 128 |
| 普通/精英单次最大位移 | 0.030 格 |
| BOSS单次最大位移 | 0.005 格 |
| 重叠修正比例 | 50% |

说明：

- 不处理 `idle`、`patrol`、`return` 和远处怪物。
- 8 格与当前脱战范围一致，覆盖实际围攻中的怪物。
- 达到最大怪物数后，后续怪物本轮不参与。
- 每一个候选配对尝试都计入配对上限，不只统计实际重叠的配对。
- 达到最大配对数后立即停止本轮处理。
- `dt` 很大时每帧最多执行一轮，不通过 `while` 补跑历史轮次。
- 停止处理不会产生错误，只表示剩余怪物继续重叠。

### 6.3 分离距离

目标中心距离必须明显小于视觉直径，从设计上保留重叠：

| 类型 | 建议目标中心距离 | 预期结果 |
|---|---:|---|
| 普通怪 | 0.30 格 | 身体仍大面积重叠 |
| 精英 | 0.36 格 | 仍允许名字和头像交叠 |
| BOSS及以上 | 0.40 格 | 仅缓慢错开，仍保留明显交叠 |

不同类型配对时取较大值。`boss`、`king_boss`、`emperor_boss`、`saint_boss` 使用同一低幅规则；`isDungeonBoss` 继续排除。

首版不根据每个怪物贴图的真实透明边界计算半径，避免引入资源分析和配置维护成本。

### 6.4 修正方式

对于距离小于目标距离的怪物对：

1. 计算分离方向。
2. 距离为 0 时，根据两个怪物 ID 生成固定方向。
3. 取剩余重叠量的50%，再受单轮最大修正限制。
4. 按怪物权重累计到各自偏移。
5. 所有配对完成后统一应用，避免遍历顺序导致漂移。
6. 候选位置不可走时，直接放弃该怪物本轮修正。

首版不尝试：

- 改走另一侧
- 沿墙搜索
- 重新寻路
- 把修正转移给其他怪物
- 强制传送到最近空位

### 6.5 怪物权重

建议：

| 类型 | 可移动权重 |
|---|---:|
| 普通怪 | 1.0 |
| 精英 | 0.7 |
| BOSS及以上 | 0.2 |

固定和特殊对象不进入候选集，也不参与配对。BOSS会参与，但最终位移额外限制为每轮最多0.005格。

### 6.6 默认排除对象

- `not monster.alive`
- `monster.data.stationary`
- `monster.isTrainingDummy`
- `monster.isPillar`
- `monster.casting`
- `monster.burstState`
- `monster.isDungeonBoss`
- `monster.disableSoftSeparation`

如果后续发现某个特殊玩法依赖精确站位，应增加显式字段：

```lua
monster.disableSoftSeparation = true
```

不通过不断堆叠类型判断解决长期扩展。

### 6.7 不与玩家发生推挤

首版只处理怪物与怪物。

不得：

- 推动玩家
- 阻挡玩家移动
- 让怪物成为新的地图阻挡物
- 修改玩家攻击目标选择
- 修改怪物仇恨和攻击距离

## 7. 性能设计

### 7.1 玩家碰撞

改动前玩家移动每帧最多执行约 8 次点通行查询。

新方案保持单一 X→Y 分轴顺序，每次椭圆占位检查最多覆盖 4 个瓦片，因此每帧仍最多约 8 次通行查询，额外成本主要是少量最近点和归一化乘法。

高频路径不创建临时 Vector 或 table，地图查询上限与改动前处于同一量级。

### 7.2 怪物分离

禁止对全部怪物执行无边界 `O(N^2)` 两两检测。

朴素配对规模：

```text
100 只怪：4,950 对/轮
300 只怪：44,850 对/轮
```

首版只收集正在近身交战的怪物，并对这个严格封顶的小列表执行两两检测。

允许因为预算限制漏处理重叠，这是设计要求，不是性能降级故障。

最大 16 只怪物理论共有 120 对，低于 128 配对上限；每秒最多约 1,200 对，性能成本稳定且很低。

### 7.3 GC 约束

怪物分离高频路径不得为每一对怪物创建：

- `{ x, y }`
- 临时 Vector
- 字符串 pair key
- 闭包

应复用：

- 战斗怪物列表
- 怪物偏移数组

每轮结束必须：

- 清空上轮候选列表残留尾部引用。
- 清零本轮使用的 X/Y 偏移。
- 不让已被替换出 `GameState.monsters` 的旧怪物继续被缓冲区持有。

### 7.4 性能指标

使用现有 `PerfMonitor` 增加独立段：

```lua
PerfMonitor.StartSegment("monster_crowd")
PerfMonitor.EndSegment("monster_crowd")
```

建议目标设备验收：

| 指标 | 目标 |
|---|---:|
| `monster_crowd` p95 | 小于 0.30 ms |
| `frame` p95 增量 | 小于 0.50 ms |
| 单轮配对 | 不超过配置上限 |
| GC 尖峰 | 无新增可感知周期性尖峰 |

`monster_crowd` 只记录真正执行弱分离的轮次，并增加以下计数器：

- `monster_crowd_candidates`
- `monster_crowd_pair_checks`
- `monster_crowd_moves_applied`
- `monster_crowd_moves_blocked`

如果超出预算，处理顺序为：

1. 缩小处理范围。
2. 降低执行频率。
3. 降低配对上限。
4. 关闭系统。

不升级为更复杂的数据结构。

## 10. 风险与缓解

### 10.1 玩家视觉穿墙略有增加

原因：镇岳精灵轮廓本身较宽，即使碰撞外接尺寸不变，圆角区域也会比旧矩形更宽松。

缓解：

- 首版不缩小半径。
- 不做职业专属碰撞。
- 以脚部接触为准，不以肩膀和武器轮廓为准。
- 若视觉穿墙明显，可恢复旧四角检测。

### 10.2 对角夹缝漏穿

原因：错误的圆形采样方法可能漏掉相邻阻挡格。

缓解：

- 使用椭圆与瓦片矩形最近点检测。
- 增加四方向精确相切和对角阻挡格接触的回归测试。

### 10.3 怪物追击与分离互相拉扯

原因：AI 向目标移动，弱分离向侧面修正。

缓解：

- 低频执行。
- 普通/精英单次位移上限为0.030格，避免一步到位的瞬移感。
- BOSS及以上单次位移上限为0.005格。
- 攻击状态已有较大退出距离余量。
- 蓄力和 burst 状态不参与。
- 允许持续部分重叠，不提高分离强度追求静态平衡。

### 10.4 特殊玩法站位改变

缓解：

- 默认排除固定对象、塔柱、训练假人和副本权威 BOSS。
- 提供 `disableSoftSeparation` 显式字段。
- 普通 BOSS及以上只使用0.2移动权重和0.005格单轮限幅。

### 10.5 性能回退

缓解：

- 仅处理近身交战怪物。
- 固定执行间隔。
- 固定最大怪物数。
- 固定最大配对数。
- 高频路径零临时分配。
- 独立性能段和开关。

## 11. 验收标准

### 11.1 玩家碰撞

必须通过：

1. 正面撞墙不能穿过。
2. 沿水平和垂直墙移动时可以稳定滑行。
3. 绕单个阻挡格四个角移动时，持续完全停住的情况明显少于当前版本。
4. 一格通道从多个斜角接近时，成功进入的容错不低于当前版本。
5. 两个对角阻挡格只在角点相接时，玩家不能穿过夹缝。
6. 罗汉、太虚、镇岳使用完全相同的逻辑参数。

不作为失败：

- 玩家没有对准单格入口时仍被挡住。
- 极端贴角时需要松开并重新调整输入。
- 镇岳肩膀或武器与墙体有少量视觉重叠。
- 极低帧率尖峰下偶发一次停顿。

### 11.2 怪物弱分离

必须通过：

1. 固定测试场景中，6只普通怪从相同坐标进入攻击状态，在开放地形运行1秒后，至少4只怪与初始重叠组产生大于0.03格的中心偏移。
2. 分离后所有被移动怪物仍处于可通行位置。
3. 固定怪、塔柱、训练假人、施法怪和副本权威 BOSS不被移动。
4. 四种 BOSS类别均可参与，但单轮位移不超过0.005格。
5. 普通怪单轮位移不超过0.030格，不一步瞬移到目标间距。
6. 达到配对上限时正常结束，不影响怪物 AI。
7. 关闭开关后行为恢复当前版本。
8. 性能指标不超过本方案预算。
9. 远处、待机和巡逻怪物位置与当前版本一致。

以下结果明确允许：

- 6只怪中仍有2～3只完全重叠。
- 墙角中的怪物全部保持重叠。
- 窄路中的怪物只散开少量。
- 大量怪物超过配对预算后不再处理。
- BOSS与普通怪仍有明显视觉交叠。

## 15. 独立审查记录

### 15.1 审查结论

两份独立只读审查均认为初稿“修改后可执行”，主要意见是初稿仍然同时改变了过多变量，并对怪物问题做了超出真实需求的通用化。

审查后首版裁决：

> 首版只实施相同外接尺寸的玩家椭圆普通移动碰撞，以及近身交战普通/精英怪的封顶弱分离。

第二轮根据运行反馈，在不提高候选数和配对预算的前提下，增加非副本权威 BOSS低幅参与，并加快普通/精英的收敛速度。

### 15.2 已采纳的必须修改项

1. 玩家半径保持 `0.30 / 0.15`，不在首版同时缩小碰撞体。
2. 玩家首版保持现有 X→Y 分轴顺序，不增加双顺序评分。
3. 玩家首版不修改击退；脱困只复用新的占位判断，不改变方向搜索、回退点和邻近瓦片搜索策略。
4. 明确瓦片半格边界、候选枚举规则和保守相切判定。
5. 怪物只处理 `chase` / `attack` 状态且距离玩家不超过 8 格的对象。
6. 远处、待机、巡逻、回巢怪物完全不处理。
7. 删除空间网格，改为最大16只怪、最大128对的低频直接两两检测。
8. 首版排除所有 BOSS、固定怪、塔柱、训练假人、施法怪和副本权威怪。
9. 每个候选配对尝试都计入预算，达到上限立即停止。
10. 复用数组必须清理尾部旧引用，避免场景切换后持有旧怪物。
11. 增加候选数、配对数、成功移动和阻挡移动计数。
12. 把怪物验收改为固定的“6怪同点、开放地形”场景；第二轮将主要收敛时间从2秒缩短到1秒。

### 15.3 保留为后续可选项

- 玩家双顺序解析。
- 玩家半径轻微缩小。
- 玩家有限移动子步。
- 最大0.04格的转角辅助。
- 怪物名字渲染错层。

这些项目只有首版测试明确证明不足时才允许进入下一阶段。

### 15.4 明确拒绝的过度设计

- 远处怪物分离。
- 巡逻和回巢怪物分离。
- 无限幅的 BOSS分离，以及副本权威 BOSS分离。
- 空间网格、邻格遍历和复杂桶生命周期。
- 怪物硬碰撞和强制最小间距。
- 沿墙寻找替代位置。
- 怪物重新寻路或形成包围队形。
- 玩家自动吸附入口。
- 首版治理长距离击退穿墙。

### 15.5 剩余可接受风险

- 椭圆面积小于同外接尺寸矩形，墙角通过范围仍会略微扩大。
- `GameMap:IsWalkable` 的封印层查询会构造字符串 key，但首版没有增加轴顺序，因此调用量与当前处于同一量级。
- 0.1秒一次的小幅怪物修正可能产生轻微离散感，因此普通/精英保持0.030格限幅，BOSS使用更低的0.005格限幅。
- 极端聚怪、墙角和超过预算的怪物仍可能完全重叠，这是符合需求的设计结果。

### 15.6 实现独立复审与修复

代码独立复审未发现 P0 阻断问题，怪物弱分离的范围、状态筛选、预算和调用时机符合本规范；发现两项玩家碰撞一致性问题，已修复：

1. `Unstuck` 原先仍用四角 AABB，可能把椭圆判定下的合法墙角位置误判后传送；现在只复用 `CanOccupy`，保留原搜索策略。
2. 椭圆覆盖范围下界原先使用 `floor(min + 0.5)`，在左侧/上侧精确相切时可能漏检；现在改为 `ceil(min - 0.5)`，并增加四方向对称回归测试。

修复后玩家碰撞测试为7/7；第二轮参数调整后怪物弱分离测试为7/7。
